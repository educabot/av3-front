# T-3.3.5: Endpoints y handler para Topics

**Historia:** HU-3.3 — Topics — jerarquía curricular
**Tipo:** Backend
**Estado:** ⬜ Pendiente

---

## Descripción

Implementar los endpoints REST para CRUD de topics. El handler extrae `organization_id` del contexto JWT (inyectado por el middleware de autenticación). La respuesta del listado devuelve la estructura de árbol con hijos anidados.

## Implementación

### Endpoints

| Método | Ruta | Rol | Descripción |
|--------|------|-----|-------------|
| `POST` | `/api/v1/topics` | admin, coordinator | Crear topic (acepta `parent_id` nullable) |
| `GET` | `/api/v1/topics` | todos | Listar árbol completo de la org |
| `GET` | `/api/v1/topics?level=N` | todos | Filtrar topics por nivel |
| `PATCH` | `/api/v1/topics/:id` | admin, coordinator | Actualizar topic (nombre, descripción, parent_id) |
| `DELETE` | `/api/v1/topics/:id` | admin | Eliminar topic y sus descendientes |

### Handler (`src/infraestructure/http/handlers/topic_handler.go`)

```go
package handlers

type TopicHandler struct {
    createUC *admin.CreateTopicUseCase
    moveUC   *admin.MoveTopicUseCase
    provider providers.TopicProvider
}

// CreateTopic godoc
// @Summary Crear un topic
// @Accept json
// @Produce json
// @Param body body CreateTopicRequest true "Topic data"
// @Router /api/v1/topics [post]
func (h *TopicHandler) CreateTopic(c *gin.Context) {
    orgID := c.GetInt("organization_id") // del JWT middleware

    var req CreateTopicRequest
    if err := c.ShouldBindJSON(&req); err != nil {
        c.JSON(400, gin.H{"error": err.Error()})
        return
    }

    topic := &entities.Topic{
        OrganizationID: orgID,
        ParentID:       req.ParentID,
        Name:           req.Name,
        Description:    req.Description,
    }

    if err := h.createUC.Execute(topic); err != nil {
        if errors.Is(err, admin.ErrMaxLevelExceeded) {
            c.JSON(422, gin.H{"error": "Se excede el máximo de niveles permitidos"})
            return
        }
        c.JSON(500, gin.H{"error": err.Error()})
        return
    }

    c.JSON(201, topic)
}

// ListTopics godoc
// @Summary Listar topics (árbol o por nivel)
// @Produce json
// @Param level query int false "Filtrar por nivel"
// @Router /api/v1/topics [get]
func (h *TopicHandler) ListTopics(c *gin.Context) {
    orgID := c.GetInt("organization_id")

    if levelStr := c.Query("level"); levelStr != "" {
        level, err := strconv.Atoi(levelStr)
        if err != nil {
            c.JSON(400, gin.H{"error": "level debe ser un número"})
            return
        }
        topics, err := h.provider.ListByLevel(orgID, level)
        if err != nil {
            c.JSON(500, gin.H{"error": err.Error()})
            return
        }
        c.JSON(200, topics)
        return
    }

    tree, err := h.provider.ListTree(orgID)
    if err != nil {
        c.JSON(500, gin.H{"error": err.Error()})
        return
    }
    c.JSON(200, tree)
}
```

### Registro en `mapping.go`

```go
// En src/infraestructure/http/mapping.go
func (m *Mapping) RegisterTopicRoutes(r *gin.RouterGroup, h *handlers.TopicHandler) {
    topics := r.Group("/topics")
    {
        topics.POST("", m.RequireRole("admin", "coordinator"), h.CreateTopic)
        topics.GET("", h.ListTopics)
        topics.PATCH("/:id", m.RequireRole("admin", "coordinator"), h.UpdateTopic)
        topics.DELETE("/:id", m.RequireRole("admin"), h.DeleteTopic)
    }
}
```

## Notas

- `organization_id` se obtiene del contexto JWT — nunca del body del request, para garantizar aislamiento multi-tenant
- `DELETE` solo para admin porque elimina en cascada todo el subárbol
- `PATCH` con `parent_id` dispara el usecase de movimiento (T-3.3.4) que re-computa niveles
- El query param `level` es opcional: sin él devuelve el árbol completo, con él devuelve lista plana filtrada
- Las hojas del árbol devuelven `"children": []` (no null) para consistencia en el frontend
