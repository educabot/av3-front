# T-3.3.4: Usecase de creación con validación de niveles

**Historia:** HU-3.3 — Topics — jerarquía curricular
**Tipo:** Backend
**Estado:** ⬜ Pendiente

---

## Descripción

Implementar el usecase de creación y actualización de topics con validación de niveles máximos. La lógica principal: calcular el nivel automáticamente según el padre, y rechazar con 422 si se excede el máximo configurado en la organización. En caso de mover un topic (cambiar `parent_id`), re-computar niveles de todo el subárbol.

## Implementación

### Usecase de creación (`src/core/usecases/admin/create_topic.go`)

```go
package admin

import (
    "alizia/src/core/entities"
    "alizia/src/core/providers"
    "errors"
)

var ErrMaxLevelExceeded = errors.New("topic exceeds maximum allowed levels")

type CreateTopicUseCase struct {
    topicProvider providers.TopicProvider
    orgProvider   providers.OrganizationProvider
}

func NewCreateTopicUseCase(tp providers.TopicProvider, op providers.OrganizationProvider) *CreateTopicUseCase {
    return &CreateTopicUseCase{topicProvider: tp, orgProvider: op}
}

func (uc *CreateTopicUseCase) Execute(topic *entities.Topic) error {
    // 1. Calcular nivel
    if topic.ParentID == nil {
        topic.Level = 1
    } else {
        parent, err := uc.topicProvider.GetByID(*topic.ParentID)
        if err != nil {
            return err
        }
        topic.Level = parent.Level + 1
    }

    // 2. Validar contra máximo de la org
    org, err := uc.orgProvider.GetByID(topic.OrganizationID)
    if err != nil {
        return err
    }
    maxLevels := org.Config.TopicMaxLevels // default: 3
    if topic.Level > maxLevels {
        return ErrMaxLevelExceeded
    }

    // 3. Crear
    return uc.topicProvider.Create(topic)
}
```

### Usecase de movimiento (`src/core/usecases/admin/move_topic.go`)

```go
package admin

type MoveTopicUseCase struct {
    topicProvider providers.TopicProvider
    orgProvider   providers.OrganizationProvider
}

func (uc *MoveTopicUseCase) Execute(topicID int, newParentID *int, orgID int) error {
    topic, err := uc.topicProvider.GetByID(topicID)
    if err != nil {
        return err
    }

    // Calcular nuevo nivel
    var newLevel int
    if newParentID == nil {
        newLevel = 1
    } else {
        parent, err := uc.topicProvider.GetByID(*newParentID)
        if err != nil {
            return err
        }
        newLevel = parent.Level + 1
    }

    // Obtener max levels de la org
    org, err := uc.orgProvider.GetByID(orgID)
    if err != nil {
        return err
    }
    maxLevels := org.Config.TopicMaxLevels

    // Calcular profundidad del subárbol
    maxDepth := uc.getSubtreeDepth(topic)
    levelDiff := newLevel - topic.Level
    if maxDepth+levelDiff > maxLevels {
        return ErrMaxLevelExceeded
    }

    // Actualizar topic y re-computar niveles del subárbol
    topic.ParentID = newParentID
    topic.Level = newLevel
    if err := uc.topicProvider.Update(topic); err != nil {
        return err
    }

    return uc.recomputeSubtreeLevels(topicID, newLevel)
}

// recomputeSubtreeLevels recalcula recursivamente los niveles de los hijos
func (uc *MoveTopicUseCase) recomputeSubtreeLevels(parentID int, parentLevel int) error {
    children, err := uc.topicProvider.GetChildren(parentID)
    if err != nil {
        return err
    }
    for _, child := range children {
        child.Level = parentLevel + 1
        if err := uc.topicProvider.Update(&child); err != nil {
            return err
        }
        if err := uc.recomputeSubtreeLevels(child.ID, child.Level); err != nil {
            return err
        }
    }
    return nil
}

// getSubtreeDepth retorna el nivel máximo dentro del subárbol
func (uc *MoveTopicUseCase) getSubtreeDepth(topic *entities.Topic) int {
    children, err := uc.topicProvider.GetChildren(topic.ID)
    if err != nil || len(children) == 0 {
        return topic.Level
    }
    maxDepth := topic.Level
    for _, child := range children {
        depth := uc.getSubtreeDepth(&child)
        if depth > maxDepth {
            maxDepth = depth
        }
    }
    return maxDepth
}
```

## Notas

- El nivel máximo por defecto es 3 (configurable en `organizations.config.topic_max_levels`)
- La validación de niveles se hace ANTES de persistir para evitar estados inconsistentes
- `recomputeSubtreeLevels` es recursivo — para el MVP con árboles de <1000 nodos y ~3 niveles, la profundidad de recursión es trivial
- El error `ErrMaxLevelExceeded` se mapea a HTTP 422 (Unprocessable Entity) en el handler (ver T-3.3.5)
- No se permite mover un topic a un descendiente propio (ciclo) — agregar validación si se detecta como caso real
