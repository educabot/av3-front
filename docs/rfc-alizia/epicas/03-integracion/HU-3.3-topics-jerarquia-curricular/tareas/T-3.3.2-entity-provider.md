# T-3.3.2: Entity Topic y TopicProvider interface

**Historia:** HU-3.3 — Topics — jerarquía curricular
**Tipo:** Backend
**Estado:** ⬜ Pendiente

---

## Descripción

Definir la entidad `Topic` con tags GORM y la interfaz `TopicProvider` que abstrae las operaciones de persistencia. La entidad incluye una relación recursiva (`Children`) para poder construir el árbol en memoria.

## Implementación

### Entity (`src/core/entities/topic.go`)

```go
package entities

import "time"

type Topic struct {
    ID             int       `json:"id" gorm:"primaryKey"`
    OrganizationID int       `json:"organization_id" gorm:"not null"`
    ParentID       *int      `json:"parent_id"`
    Name           string    `json:"name" gorm:"not null;size:255"`
    Description    string    `json:"description,omitempty"`
    Level          int       `json:"level" gorm:"not null;default:1"`
    CreatedAt      time.Time `json:"created_at"`

    // Relación recursiva para construir árbol
    Children []Topic `json:"children,omitempty" gorm:"foreignKey:ParentID"`
}
```

### Provider interface (`src/core/providers/topic_provider.go`)

```go
package providers

import "alizia/src/core/entities"

type TopicProvider interface {
    Create(topic *entities.Topic) error
    GetByID(id int) (*entities.Topic, error)
    ListTree(organizationID int) ([]entities.Topic, error)
    ListByLevel(organizationID int, level int) ([]entities.Topic, error)
    Update(topic *entities.Topic) error
    Delete(id int) error
    GetChildren(parentID int) ([]entities.Topic, error)
}
```

## Notas

- `ParentID` es `*int` (puntero) para permitir `NULL` en la base de datos — los topics raíz no tienen padre
- `Children` usa `gorm:"foreignKey:ParentID"` pero NO se usa con Preload recursivo de GORM (es ineficiente) — el árbol se construye en memoria (ver T-3.3.3)
- El tag `json:"children,omitempty"` evita enviar arrays vacíos en las hojas del árbol
- `ListTree` devuelve la lista plana; el armado del árbol es responsabilidad del repositorio (T-3.3.3)
