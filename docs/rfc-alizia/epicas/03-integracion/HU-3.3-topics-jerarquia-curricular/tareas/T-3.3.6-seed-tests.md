func TestCreateRootTopic(t *testing.T) {

### Test: Query de árbol devuelve estructura anidada

```go
func TestListTreeReturnsNestedStructure(t *testing.T) {
    tree, err := topicRepo.ListTree(1)
    assert.NoError(t, err)
    assert.Len(t, tree, 2) // 2 raíces

    // Primer nodo raíz tiene hijos
    assert.Equal(t, "Pensamiento Lógico-Matemático", tree[0].Name)
    assert.Len(t, tree[0].Children, 2) // 2 áreas de conocimiento

    // Los hijos tienen sub-hijos
    assert.Len(t, tree[0].Children[0].Children, 2) // 2 categorías
}
```

### Test: Mover topic re-computa niveles del subárbol

```go
func TestMoveTopicRecomputesLevels(t *testing.T) {
    // Mover "Aritmética Básica" (nivel 2) a raíz (nivel 1)
    err := moveUC.Execute(3, nil, 1)
    assert.NoError(t, err)

    // Verificar que "Aritmética" ahora es nivel 1
    topic, _ := topicProvider.GetByID(3)
    assert.Equal(t, 1, topic.Level)

    // Verificar que sus hijos bajaron un nivel
    children, _ := topicProvider.GetChildren(3)
    for _, child := range children {
        assert.Equal(t, 2, child.Level) // eran nivel 3, ahora nivel 2
    }
}
```

## Notas

- Los IDs del seed son referenciales — en la implementación real dependen del SERIAL auto-incremento
- Los tests deben ejecutarse contra una base de datos de test limpia (truncar tabla topics antes de cada test)
- El test de `ListTree` valida que `buildTree` (T-3.3.3) funciona correctamente de extremo a extremo
- Se recomienda agregar un test adicional para verificar que `children` es `[]` (no null) en las hojas del árbol
- El test de movimiento verifica la re-computación recursiva de niveles del subárbol completo
