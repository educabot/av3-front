# T-3.4.2: Entities Go

**Historia:** HU-3.4 — Cursos, alumnos y asignaciones
**Tipo:** Backend
**Estado:** ⬜ Pendiente

---

## Descripción

Crear las entities Go para courses, students y course_subjects. `CourseSubject` es la entidad de join central que conecta el mundo administrativo (cursos, disciplinas) con el mundo docente (planificaciones de clase).

## Archivos

### `internal/admin/entities/course.go`

```go
type Course struct {
    ID             uint            `json:"id" gorm:"primaryKey"`
    OrganizationID uint            `json:"organization_id"`
    Name           string          `json:"name"`
    CreatedAt      time.Time       `json:"created_at"`
    Students       []Student       `json:"students,omitempty" gorm:"foreignKey:CourseID"`
    CourseSubjects []CourseSubject  `json:"course_subjects,omitempty" gorm:"foreignKey:CourseID"`
}
```

### `internal/admin/entities/student.go`

```go
type Student struct {
    ID        uint      `json:"id" gorm:"primaryKey"`
    CourseID  uint      `json:"course_id"`
    Name      string    `json:"name"`
    CreatedAt time.Time `json:"created_at"`
}
```

### `internal/admin/entities/course_subject.go`

```go
type CourseSubject struct {
    ID         uint      `json:"id" gorm:"primaryKey"`
    CourseID   uint      `json:"course_id"`
    SubjectID  uint      `json:"subject_id"`
    TeacherID  uint      `json:"teacher_id"`
    StartDate  *time.Time `json:"start_date"`
    EndDate    *time.Time `json:"end_date"`
    SchoolYear int       `json:"school_year"`
    CreatedAt  time.Time `json:"created_at"`
    Subject    *Subject  `json:"subject,omitempty" gorm:"foreignKey:SubjectID"`
    Teacher    *User     `json:"teacher,omitempty" gorm:"foreignKey:TeacherID"`
}
```

## Notas

- `Students` y `CourseSubjects` en Course se cargan con Preload solo cuando se pide el detalle
- `Subject` y `Teacher` en CourseSubject se preloadean para evitar N+1 en listados
- `StartDate` y `EndDate` son pointers (nullable)
- `CourseSubject` es el punto de conexion entre la estructura administrativa y las planificaciones docentes; los lesson plans referencian a esta entidad
- No usar `gorm.Model` para mantener consistencia con el resto del proyecto
