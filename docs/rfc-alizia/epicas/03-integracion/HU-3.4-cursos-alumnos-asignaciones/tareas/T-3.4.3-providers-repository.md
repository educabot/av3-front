## Descripción

Definir las interfaces de provider para courses, students y course_subjects, e implementar los repositorios GORM correspondientes. Todas las queries filtran por `organization_id` para garantizar aislamiento multi-tenant.

### Implementacion GORM (repositorios)

#### `internal/admin/repositories/course_repository.go`

```go
func (r *courseRepository) FindByID(ctx context.Context, orgID uint, id uint) (*entities.Course, error) {
    var course entities.Course
    err := r.db.WithContext(ctx).
        Where("id = ? AND organization_id = ?", id, orgID).
        Preload("Students").
        Preload("CourseSubjects").
        Preload("CourseSubjects.Subject").
        Preload("CourseSubjects.Teacher").
        First(&course).Error
    if err != nil {
        return nil, err
    }
    return &course, nil
}

func (r *courseRepository) FindAll(ctx context.Context, orgID uint) ([]entities.Course, error) {
    var courses []entities.Course
    err := r.db.WithContext(ctx).
        Where("organization_id = ?", orgID).
        Find(&courses).Error
    return courses, err
}
```

#### `internal/admin/repositories/student_repository.go`

```go
func (r *studentRepository) FindByCourseID(ctx context.Context, courseID uint) ([]entities.Student, error) {

```

#### `internal/admin/repositories/course_subject_repository.go`

```go
func (r *courseSubjectRepository) FindByCourseID(ctx context.Context, courseID uint) ([]entities.CourseSubject, error) {
    var cs []entities.CourseSubject
    err := r.db.WithContext(ctx).
        Where("course_id = ?", courseID).
        Preload("Subject").
        Preload("Teacher").
        Find(&cs).Error
    return cs, err
}
```

## Notas

- `FindAll` y `FindByID` de courses siempre filtran por `orgID` para aislamiento multi-tenant
- `FindByID` de courses hace Preload de Students, CourseSubjects, Subject y Teacher para devolver el detalle completo en un solo query
- `FindByCourseID` de students no necesita filtro de org porque el curso ya pertenece a una org (la validacion se hace en la capa de handler)
- Los mocks se generan a partir de estas interfaces para testing
