# T-3.4.4: Endpoints y Handlers

**Historia:** HU-3.4 — Cursos, alumnos y asignaciones
**Tipo:** Backend
**Estado:** ⬜ Pendiente

---

## Descripcion

Implementar los endpoints REST para gestionar cursos, alumnos y asignaciones curso-disciplina-docente. Registrar las rutas en `mapping.go`.

## Endpoints

| Metodo | Ruta | Rol requerido | Descripcion |
|--------|------|---------------|-------------|
| POST | `/api/v1/courses` | admin | Crear un curso |
| GET | `/api/v1/courses` | all | Listar cursos de la organizacion |
| GET | `/api/v1/courses/:id` | all | Detalle de curso con students + course_subjects + schedule |
| POST | `/api/v1/courses/:id/students` | admin | Agregar alumno a un curso |
| POST | `/api/v1/course-subjects` | admin | Crear asignacion curso-disciplina-docente |

## Implementacion

### Handler: `internal/admin/handlers/course_handler.go`

```go
type CourseHandler struct {
    courseRepo        providers.CourseRepository
    studentRepo       providers.StudentRepository
    courseSubjectRepo  providers.CourseSubjectRepository
}

// POST /api/v1/courses
type CreateCourseRequest struct {
    Name string `json:"name" binding:"required"`
}

// POST /api/v1/courses/:id/students
type CreateStudentRequest struct {
    Name string `json:"name" binding:"required"`
}

// POST /api/v1/course-subjects
type CreateCourseSubjectRequest struct {
    CourseID   uint   `json:"course_id" binding:"required"`
    SubjectID  uint   `json:"subject_id" binding:"required"`
    TeacherID  uint   `json:"teacher_id" binding:"required"`
    SchoolYear int    `json:"school_year" binding:"required"`
    StartDate  *string `json:"start_date"`  // formato "2006-01-02"
    EndDate    *string `json:"end_date"`    // formato "2006-01-02"
}
```

### Registro en `mapping.go`

```go
// Courses
courses := v1.Group("/courses")
{
    courses.POST("", adminMiddleware, courseHandler.Create)
    courses.GET("", courseHandler.List)
    courses.GET("/:id", courseHandler.GetByID)
    courses.POST("/:id/students", adminMiddleware, studentHandler.Create)
}

// Course-Subjects
v1.POST("/course-subjects", adminMiddleware, courseSubjectHandler.Create)
```

## Notas

- `organization_id` se extrae del contexto JWT (middleware de auth), no del body
- Los endpoints POST son solo para admin; los GET son accesibles por todos los roles autenticados
- El handler de `Create` para course_subjects debe validar que el `course_id`, `subject_id` y `teacher_id` existan y pertenezcan a la misma organizacion
- Las fechas `start_date` y `end_date` se reciben como strings y se parsean en el handler
