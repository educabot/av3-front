# T-3.5.1: Migración: time_slots y time_slot_subjects

**Historia:** HU-3.5 — Grilla horaria y clases compartidas
**Tipo:** DB
**Estado:** ⬜ Pendiente

---

## Descripción

Crear las tablas `time_slots` y `time_slot_subjects` que modelan la grilla horaria semanal de cada curso. Un time_slot representa un bloque horario en un día de la semana. Cada slot puede tener 1 subject (clase normal) o 2 subjects (clase compartida), vinculados a través de `time_slot_subjects`.

## Implementación

### Migración UP

```sql
-- Tabla de bloques horarios por curso
CREATE TABLE time_slots (
    id SERIAL PRIMARY KEY,
    course_id INTEGER NOT NULL REFERENCES courses(id) ON DELETE CASCADE,
    day_of_week SMALLINT NOT NULL CHECK (day_of_week BETWEEN 0 AND 6),
    start_time TIME NOT NULL,
    end_time TIME NOT NULL,
    created_at TIMESTAMP NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_time_slots_course_id ON time_slots(course_id);
CREATE INDEX idx_time_slots_day_of_week ON time_slots(course_id, day_of_week);

-- Tabla pivote: qué disciplinas se dictan en cada slot
CREATE TABLE time_slot_subjects (
    id SERIAL PRIMARY KEY,
    time_slot_id INTEGER NOT NULL REFERENCES time_slots(id) ON DELETE CASCADE,
    course_subject_id INTEGER NOT NULL REFERENCES course_subjects(id) ON DELETE CASCADE,
    UNIQUE(time_slot_id, course_subject_id)
);

CREATE INDEX idx_time_slot_subjects_slot ON time_slot_subjects(time_slot_id);
CREATE INDEX idx_time_slot_subjects_cs ON time_slot_subjects(course_subject_id);
```

### Migración DOWN

```sql
DROP TABLE IF EXISTS time_slot_subjects;
DROP TABLE IF EXISTS time_slots;
```

### Archivo

- `db/migrations/XXX_create_time_slots.sql`

## Notas

- `day_of_week` usa SMALLINT con CHECK constraint (0=Domingo, 1=Lunes ... 6=Sábado) para mantenerlo alineado con convenciones estándar.
- La constraint UNIQUE en `time_slot_subjects` impide asignar la misma disciplina dos veces al mismo slot.
- CASCADE en FKs asegura que al borrar un curso o course_subject se limpien los registros asociados.
- Depende de que las tablas `courses` y `course_subjects` ya existan (HU-3.4).
