# T-3.5.2: Trigger: validate_time_slot_subject

**Historia:** HU-3.5 — Grilla horaria y clases compartidas
**Tipo:** DB
**Estado:** ⬜ Pendiente

---

## Descripción

Crear un trigger que se ejecute BEFORE INSERT OR UPDATE en `time_slot_subjects` para validar que el `course_subject` pertenezca al mismo curso que el `time_slot`. Esto previene inconsistencias a nivel de base de datos, independientemente de lo que valide la aplicación.

## Implementación

### Migración UP

```sql
-- Función de validación
CREATE OR REPLACE FUNCTION validate_time_slot_subject() RETURNS TRIGGER AS $$
BEGIN
    IF NOT EXISTS (
        SELECT 1 FROM course_subjects cs
        JOIN time_slots ts ON ts.course_id = cs.course_id
        WHERE cs.id = NEW.course_subject_id AND ts.id = NEW.time_slot_id
    ) THEN
        RAISE EXCEPTION 'course_subject does not belong to the same course as the time_slot';
    END IF;
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

-- Trigger que ejecuta la validación antes de insertar o actualizar
CREATE TRIGGER trg_validate_time_slot_subject
    BEFORE INSERT OR UPDATE ON time_slot_subjects
    FOR EACH ROW
    EXECUTE FUNCTION validate_time_slot_subject();
```

### Migración DOWN

```sql
DROP TRIGGER IF EXISTS trg_validate_time_slot_subject ON time_slot_subjects;
DROP FUNCTION IF EXISTS validate_time_slot_subject();
```

### Archivo

- `db/migrations/XXX_trigger_validate_time_slot_subject.sql`

## Notas

- El trigger hace un JOIN entre `course_subjects` y `time_slots` verificando que compartan el mismo `course_id`. Si no coinciden, lanza una excepción que aborta la operación.
- Es una validación de integridad referencial cruzada que no se puede expresar con FKs simples.
- El error que lanza PostgreSQL será capturado por GORM y debe traducirse a un HTTP 422 en el handler.
- Ejecutar esta migración despues de T-3.5.1 ya que depende de que ambas tablas existan.
