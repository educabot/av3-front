# T-3.6.1: Migracion: enum class_moment y tabla activities

**Historia:** HU-3.6 — Actividades didacticas
**Tipo:** DB
**Estado:** ⬜ Pendiente

---

## Descripcion

Crear el enum `class_moment` con los tres momentos de clase (apertura, desarrollo, cierre) y la tabla `activities` que almacena las actividades didacticas disponibles por organizacion. Cada actividad pertenece a un unico momento de clase.

## Implementacion

### Migracion UP

```sql
-- Enum para los momentos de clase
CREATE TYPE class_moment AS ENUM ('apertura', 'desarrollo', 'cierre');

-- Tabla de actividades didacticas
CREATE TABLE activities (
    id SERIAL PRIMARY KEY,
    organization_id INTEGER NOT NULL REFERENCES organizations(id) ON DELETE CASCADE,
    moment class_moment NOT NULL,
    name VARCHAR(255) NOT NULL,
    description TEXT,
    duration_minutes INTEGER,
    created_at TIMESTAMP NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_activities_organization ON activities(organization_id);
CREATE INDEX idx_activities_moment ON activities(organization_id, moment);
```

### Migracion DOWN

```sql
DROP TABLE IF EXISTS activities;
DROP TYPE IF EXISTS class_moment;
```

### Archivo

- `db/migrations/XXX_create_activities.sql`

## Notas

- El enum `class_moment` es fijo y no configurable por organizacion. Los tres momentos (apertura, desarrollo, cierre) representan la estructura pedagogica estandar de una clase.
- `duration_minutes` es opcional (puede ser NULL) para actividades sin duracion fija.
- El indice compuesto `(organization_id, moment)` optimiza la query mas comun: listar actividades de una org filtradas por momento.
- La FK a `organizations` con CASCADE asegura que al eliminar una organizacion se borren sus actividades.
- Depende de que la tabla `organizations` exista (HU-3.1).
