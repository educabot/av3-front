# T-3.2.3: Provider interfaces

**Historia:** HU-3.2 — Áreas y disciplinas
**Tipo:** Backend
**Estado:** ⬜ Pendiente

---

## Descripción

Definir las interfaces de repositorio para áreas y disciplinas. Estas interfaces viven en `providers/` y son implementadas por los repositories GORM. Todas las operaciones de listado reciben `organizationID` para garantizar aislamiento multi-tenant.

## Archivos

### `src/core/providers/admin.go`

```go
package providers

import (
    "context"
    "src/core/entities"
)

type AreaProvider interface {
    // Create crea un área nueva dentro de una organización
    Create(ctx context.Context, area *entities.Area) error

    // List retorna todas las áreas de una organización con Subjects y Coordinators precargados
    List(ctx context.Context, organizationID uint) ([]entities.Area, error)

    // Update actualiza nombre y/o descripción de un área
    Update(ctx context.Context, area *entities.Area) error

    // AssignCoordinator asigna un usuario como coordinador del área
    // Retorna error 409 si el coordinador ya está asignado
    AssignCoordinator(ctx context.Context, areaID uint, userID uint) (*entities.AreaCoordinator, error)
}

type SubjectProvider interface {
    // Create crea una disciplina nueva asociada a un área
    Create(ctx context.Context, subject *entities.Subject) error

    // List retorna todas las disciplinas de una organización
    List(ctx context.Context, organizationID uint) ([]entities.Subject, error)
}
```

## Notas

- Las interfaces son mínimas — solo lo que se necesita para los criterios de aceptación de HU-3.2
- Se agregan métodos (FindByID, Delete, etc.) a medida que los usecases lo requieran
- `organizationID` es parámetro explícito en `List` para que el filtrado sea evidente en la firma
- Los mocks se generan a partir de estas interfaces para testing unitario
- `AssignCoordinator` retorna el registro creado para confirmar la operación al cliente
