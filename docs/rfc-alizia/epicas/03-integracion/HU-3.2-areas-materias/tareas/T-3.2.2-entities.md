## Descripción

Crear las entities Go que representan áreas, disciplinas y la relación de coordinadores. Incluyen struct tags de GORM para mapeo automático y preload de relaciones.

## Archivos

### `src/core/entities/area.go`

```go
package entities

import "time"

type Area struct {
    ID             uint      `json:"id" gorm:"primaryKey"`
    OrganizationID uint      `json:"organization_id"`
    Name           string    `json:"name"`
    Description    *string   `json:"description"`
    CreatedAt      time.Time `json:"created_at"`
    Subjects       []Subject `json:"subjects" gorm:"foreignKey:AreaID"`
    Coordinators   []User    `json:"coordinators" gorm:"many2many:area_coordinators;foreignKey:ID;joinForeignKey:AreaID;references:ID;joinReferences:UserID"`
}

type AreaCoordinator struct {
    ID     uint `json:"id" gorm:"primaryKey"`
    AreaID uint `json:"area_id"`
    UserID uint `json:"user_id"`
}
```

### `src/core/entities/subject.go`

```go
package entities

import "time"

type Subject struct {
    ID             uint      `json:"id" gorm:"primaryKey"`
    OrganizationID uint      `json:"organization_id"`
    AreaID         uint      `json:"area_id"`
    Name           string    `json:"name"`
    Description    *string   `json:"description"`
    CreatedAt      time.Time `json:"created_at"`
}
```

## Notas

- `Description` es pointer (`*string`) para representar el nullable del SQL
- `Subjects` en Area usa `foreignKey:AreaID` para que GORM haga el Preload correctamente
- `Coordinators` usa `many2many:area_coordinators` — GORM resuelve la tabla intermedia automáticamente
- `AreaCoordinator` se define como entity separada para operaciones directas sobre la tabla intermedia (crear, verificar duplicados)
- No usar `gorm.Model` (tiene campos extra que no necesitamos)
