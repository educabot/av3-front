# T-3.2.5: Endpoints y handlers

**Historia:** HU-3.2 — Áreas y disciplinas
**Tipo:** Backend
**Estado:** ⬜ Pendiente

---

## Descripción

Implementar los endpoints HTTP para CRUD de áreas y disciplinas, y la asignación de coordinadores. Registrar las rutas en `mapping.go` con los interceptores de autorización por rol.

## Endpoints

### Crear área

```
POST /api/v1/areas
Roles: coordinator, admin
```

**Request:**
```json
{
    "name": "Ciencias Exactas",
    "description": "Área de matemáticas, física y química"
}
```

**Response 201:**
```json
{
    "id": 1,
    "organization_id": 1,
    "name": "Ciencias Exactas",
    "description": "Área de matemáticas, física y química",
    "created_at": "2026-03-26T10:00:00Z"
}
```

### Listar áreas

```
GET /api/v1/areas
Roles: todos (autenticados)
```

**Response 200:**
```json
[
    {
        "id": 1,
        "organization_id": 1,
        "name": "Ciencias Exactas",
        "description": "Área de matemáticas, física y química",
        "created_at": "2026-03-26T10:00:00Z",
        "subjects": [
            { "id": 1, "name": "Matemáticas", "area_id": 1 },
            { "id": 2, "name": "Física", "area_id": 1 }
        ],
        "coordinators": [
            { "id": 2, "name": "María Coordinadora", "email": "coordinador@neuquen.edu.ar" }
        ]
    }
]
```

### Actualizar área

```
PUT /api/v1/areas/:id
Roles: coordinator, admin
```

**Request:**
```json
{
    "name": "Ciencias Exactas y Naturales",
    "description": "Actualizado"
}
```

**Response 200:** Área actualizada

### Asignar coordinador a área

```
POST /api/v1/areas/:id/coordinators
Roles: admin
```

**Request:**
```json
{
    "user_id": 3
}
```

**Response 201:**
```json
{
    "id": 1,
    "area_id": 1,
    "user_id": 3
}
```

**Errores:**
- 404 — Área o usuario no existe
- 409 — Coordinador ya asignado a esta área (`already_assigned`)
- 403 — No es admin

## Implementación

### Registro de rutas en `mapping.go`

```go
coordOrAdmin := middleware.RequireRole("coordinator", "admin")
adminOnly := middleware.RequireRole("admin")

// Areas
api.POST("/areas", webgin.Adapt(h.Admin.CreateArea.Handle(), coordOrAdmin))
api.GET("/areas", webgin.Adapt(h.Admin.ListAreas.Handle()))
api.PUT("/areas/:id", webgin.Adapt(h.Admin.UpdateArea.Handle(), coordOrAdmin))
api.POST("/areas/:id/coordinators", webgin.Adapt(h.Admin.AssignCoordinator.Handle(), adminOnly))

// Subjects
api.POST("/subjects", webgin.Adapt(h.Admin.CreateSubject.Handle(), coordOrAdmin))
api.GET("/subjects", webgin.Adapt(h.Admin.ListSubjects.Handle()))
```

## Notas

- `organization_id` se extrae del context (JWT claim), no se recibe en el body
- GET endpoints no requieren rol específico, solo estar autenticado
- POST/PUT de áreas y disciplinas permiten coordinator y admin
- POST de coordinadores solo admin (para evitar que un coordinador se auto-asigne)
- Validar que `area_id` en POST /subjects pertenezca a la misma organización del usuario
- Validar que el `user_id` en POST /areas/:id/coordinators exista y pertenezca a la misma organización
