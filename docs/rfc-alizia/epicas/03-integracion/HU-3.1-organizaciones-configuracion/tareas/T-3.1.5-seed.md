# T-3.1.5: Seed de organización

**Historia:** HU-3.1 — Organizaciones y configuración
**Tipo:** DB
**Estado:** ⬜ Pendiente

---

## Descripción

Crear el seed con al menos 1 organización de ejemplo con config JSONB completa, incluyendo todas las keys de configuración que el sistema utiliza: niveles de topics, secciones del documento de coordinación, clases compartidas y actividades de desarrollo.

## Implementación

### SQL

```sql
INSERT INTO organizations (name, slug, config) VALUES (
    'Provincia de Neuquén',
    'neuquen',
    '{
        "topic_max_levels": 3,
        "topic_level_names": [
            "Núcleos problemáticos",
            "Áreas de conocimiento",
            "Categorías"
        ],
        "topic_selection_level": 3,
        "shared_classes_enabled": true,
        "desarrollo_max_activities": 3,
        "coord_doc_sections": [
            {
                "key": "problem_edge",
                "label": "Eje problemático",
```

### Archivo

- `db/seeds/seed.sql` (se agrega al seed existente o se reemplaza si ya tiene la org de T-1.2.5)

### Descripción de las keys del config

| Key | Tipo | Descripción |
|-----|------|-------------|
| `topic_max_levels` | integer | Cantidad máxima de niveles jerárquicos en el árbol de topics |
| `topic_level_names` | string[] | Nombres de cada nivel (de mayor a menor granularidad) |
| `topic_selection_level` | integer | En qué nivel del árbol el usuario selecciona topics para el documento |
| `shared_classes_enabled` | boolean | Si la organización permite clases compartidas entre disciplinas |
| `desarrollo_max_activities` | integer | Máximo de actividades de desarrollo por clase en el plan |
| `coord_doc_sections` | object[] | Secciones configurables del documento de coordinación |
| `coord_doc_sections[].key` | string | Identificador único de la sección |
| `coord_doc_sections[].label` | string | Nombre visible de la sección |
| `coord_doc_sections[].type` | string | Tipo de input: `text`, `select_text` |
| `coord_doc_sections[].options` | string[] | Opciones disponibles (solo para tipo `select_text`) |
| `coord_doc_sections[].ai_prompt` | string | Prompt base para generación con IA |
| `coord_doc_sections[].required` | boolean | Si la sección es obligatoria |

## Notas

- Este seed puede coincidir con el de T-1.2.5 (Épica 1). Si ya existe, verificar que el config esté completo con todas las keys listadas.
- El seed se ejecuta con `psql -f db/seeds/seed.sql` o como parte de `./scripts/reset_db.sh`.
- Los `ai_prompt` son textos base que el backend usa para construir el prompt completo enviado a Azure OpenAI.
- El tipo `select_text` indica que el usuario primero elige una opción (ej: "proyecto") y luego Alizia genera texto basado en esa selección.
