# T-3.1.2: Entity y Provider de Organization

**Historia:** HU-3.1 — Organizaciones y configuración
**Tipo:** Backend
**Estado:** ⬜ Pendiente

---

## Descripción

Crear la entity `Organization` en Go con struct tags de GORM y la interface `OrganizationProvider` con los métodos necesarios para obtener y actualizar organizaciones.

## Implementación

### `src/core/entities/organization.go`

```go
package entities

import (
    "time"

    "gorm.io/datatypes"
)

type Organization struct {
    ID        uint           `json:"id" gorm:"primaryKey"`
    Name      string         `json:"name" gorm:"type:varchar(255);not null"`
    Slug      string         `json:"slug" gorm:"type:varchar(100);uniqueIndex;not null"`
    Config    datatypes.JSON `json:"config" gorm:"type:jsonb;not null;default:'{}'"`
    CreatedAt time.Time      `json:"created_at" gorm:"autoCreateTime"`
}

func (Organization) TableName() string {
    return "organizations"
}
```

### `src/core/providers/admin.go`

```go
package providers

import (
    "context"

    "github.com/educabot/demo-alizia/src/core/entities"
)

type OrganizationProvider interface {
    // GetByID obtiene una organización por su ID
    GetByID(ctx context.Context, id uint) (*entities.Organization, error)

    // Update actualiza campos de la organización (merge parcial de config JSONB)
    Update(ctx context.Context, id uint, updates map[string]interface{}) (*entities.Organization, error)
}
```

## Notas

- Se usa `datatypes.JSON` de `gorm.io/datatypes` para el campo JSONB. Esto permite leer/escribir JSON arbitrario.
- La entity podría extenderse en el futuro si se necesita un struct tipado para `Config`, pero por ahora `datatypes.JSON` es suficiente para la POC.
- `TableName()` se define explícitamente para evitar que GORM pluralice el nombre.
- El provider define solo `GetByID` y `Update` ya que la creación de organizaciones es una operación poco frecuente (seed o admin manual).
