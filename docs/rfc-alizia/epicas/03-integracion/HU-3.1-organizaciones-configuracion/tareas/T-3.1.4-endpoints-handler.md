# T-3.1.4: Endpoints y Handler de Organization

**Historia:** HU-3.1 — Organizaciones y configuración
**Tipo:** Backend
**Estado:** ⬜ Pendiente

---

## Descripción

Crear los endpoints REST para obtener y actualizar organizaciones. Ambos requieren autenticación (JWT) y el org_id se extrae del token para garantizar multi-tenancy. El handler se ubica en `src/entrypoints/rest/admin/` y se registra en el mapping central.

## Implementación

### `src/entrypoints/rest/admin/organization_handler.go`

```go
package admin

import (
    "net/http"
    "strconv"

    "github.com/gin-gonic/gin"

    "github.com/educabot/demo-alizia/src/core/providers"
    "github.com/educabot/demo-alizia/src/entrypoints/rest/middleware"
)

type OrganizationHandler struct {
    provider providers.OrganizationProvider
}

func NewOrganizationHandler(p providers.OrganizationProvider) *OrganizationHandler {
    return &OrganizationHandler{provider: p}
}

// GET /api/v1/organizations/:id
func (h *OrganizationHandler) GetByID(c *gin.Context) {
    id, err := strconv.ParseUint(c.Param("id"), 10, 64)
    if err != nil {
        c.JSON(http.StatusBadRequest, gin.H{"error": "invalid id"})
        return
    }

    // Validar que el usuario solo acceda a su propia org
    claims := middleware.GetClaims(c)
    if claims.ID != c.Param("id") {
        c.JSON(http.StatusForbidden, gin.H{"error": "access denied"})
        return
    }

    org, err := h.provider.GetByID(c.Request.Context(), uint(id))
    if err != nil {
        c.JSON(http.StatusNotFound, gin.H{"error": "organization not found"})
        return
    }

    c.JSON(http.StatusOK, org)
}

// PATCH /api/v1/organizations/:id
func (h *OrganizationHandler) Update(c *gin.Context) {
    id, err := strconv.ParseUint(c.Param("id"), 10, 64)
    if err != nil {
        c.JSON(http.StatusBadRequest, gin.H{"error": "invalid id"})
        return
    }

    claims := middleware.GetClaims(c)
    if claims.ID != c.Param("id") {
        c.JSON(http.StatusForbidden, gin.H{"error": "access denied"})
        return
    }

    var body map[string]interface{}
    if err := c.ShouldBindJSON(&body); err != nil {
        c.JSON(http.StatusBadRequest, gin.H{"error": "invalid body"})
        return
    }

    org, err := h.provider.Update(c.Request.Context(), uint(id), body)
    if err != nil {
        c.JSON(http.StatusInternalServerError, gin.H{"error": "update failed"})
        return
    }

    c.JSON(http.StatusOK, org)
}
```

### Registro en `src/app/web/mapping.go`

```go
// En la función de registro de rutas:
orgHandler := admin.NewOrganizationHandler(orgProvider)

v1 := router.Group("/api/v1")
v1.Use(authMiddleware) // JWT requerido
{
    v1.GET("/organizations/:id", orgHandler.GetByID)
    v1.PATCH("/organizations/:id", orgHandler.Update)
}
```

## Notas

- El PATCH hace merge parcial: solo las keys enviadas en `config` se actualizan, el resto se conservan intactas.
- Multi-tenancy: el handler valida que el `org_id` del JWT coincida con el `:id` del path. Un usuario no puede acceder a otra organización.
- Para la POC se usa `map[string]interface{}` como body del PATCH. En producción conviene definir un DTO con validación.
- El endpoint PATCH podría restringirse solo a roles `admin` con el middleware `RequireRole("admin")`.
