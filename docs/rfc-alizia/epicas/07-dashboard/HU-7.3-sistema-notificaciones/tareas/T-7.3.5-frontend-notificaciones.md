## Descripción

Implementar el componente de notificaciones en el header de la aplicación: ícono de campana con badge de conteo, panel desplegable con lista de notificaciones, y acciones de lectura.

## Implementación

### Componente de campana (header)

```javascript
// Se monta en el header global de la app
const NotificationBell = {
    unreadCount: 0,
    isOpen: false,
    notifications: [],

    async loadUnreadCount() {
        const data = await fetch('/notifications/unread-count').then(r => r.json());
        this.unreadCount = data.unread_count;
        this.renderBadge();
    },

    async togglePanel() {
        this.isOpen = !this.isOpen;
        if (this.isOpen) {
            await this.loadNotifications();
        }
        this.render();
    },

    async loadNotifications() {
        const data = await fetch('/notifications?limit=20').then(r => r.json());
        this.notifications = data.notifications;
    },

    async markAsRead(id) {
        await fetch(`/notifications/${id}/read`, { method: 'PATCH' });
        // Actualizar UI
    },

    async markAllAsRead() {
        await fetch('/notifications/read-all', { method: 'POST' });
        this.unreadCount = 0;
        this.notifications.forEach(n => n.read_at = new Date().toISOString());
        this.render();
    }
};
```

### Navegación al hacer click

```javascript
function goToEntity(notification) {
    // Marcar como leída
    markAsRead(notification.id);

    // Navegar según entity_type
    switch (notification.entity_type) {
        case 'coordination_document':
            navigateTo(`/documents/${notification.entity_id}`);
            break;
        case 'lesson_plan':
            navigateTo(`/lesson-plans/${notification.entity_id}`);
            break;
        case 'course_subject':
            navigateTo(`/course-subjects/${notification.entity_id}`);
            break;
    }
}
```

## Notas

- El conteo de no leídas se carga al montar el header (cada page load) — no websocket en MVP
- El panel se cierra al hacer click fuera o al navegar a una notificación
- El tiempo relativo ("hace 2 horas") se calcula en frontend
- Si `unread_count` > 99, mostrar "99+"
- El panel tiene scroll si hay muchas notificaciones, con lazy loading al llegar al final
