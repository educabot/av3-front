# T-7.3.3: Job programado de plazos próximos

**Historia:** HU-7.3 — Sistema de notificaciones
**Tipo:** Backend
**Estado:** ⬜ Pendiente
**Fase:** Post-MVP

---

## Descripción

Implementar un job periódico que revisa clases sin planificar cuya fecha está dentro de los próximos N días y genera notificaciones de alerta para los docentes correspondientes. Se ejecuta diariamente.

## Implementación

### Job

```go
// src/core/usecases/notifications/deadline_job.go
package notifications

// DeadlineNotificationJob revisa clases próximas sin planificar.
type DeadlineNotificationJob struct {
    notifService *NotificationService
    csRepo       CourseSubjectRepository
    planRepo     LessonPlanRepository
    scheduleCalc ScheduleCalculator
}

// Run se ejecuta una vez al día (ej: 8:00 AM).
func (j *DeadlineNotificationJob) Run() error {
    // 1. Obtener todos los course_subjects activos
    // 2. Para cada uno, calcular próximas clases en los próximos N días
    // 3. Verificar cuáles no tienen lesson_plan
    // 4. Agrupar por docente
    // 5. Generar una notificación por docente con resumen

    courseSubjects, err := j.csRepo.GetAllActive()
    if err != nil {
        return err
    }

    daysAhead := 7 // configurable por org
    alertsByTeacher := make(map[int][]UnplannedClass)

    for _, cs := range courseSubjects {
        upcoming := j.scheduleCalc.GetClassesInRange(cs, time.Now(), time.Now().AddDate(0, 0, daysAhead))
        for _, class := range upcoming {
            plan, _ := j.planRepo.FindByClassAndCourseSubject(class.CoordDocClassID, cs.ID)
            if plan == nil {
                alertsByTeacher[cs.TeacherID] = append(alertsByTeacher[cs.TeacherID], class)
            }
        }
    }

    for teacherID, classes := range alertsByTeacher {
        j.notifService.NotifyDeadlineApproaching(teacherID, classes)
    }

    return nil
}
```

### Notificación de plazo

```go
func (s *NotificationService) NotifyDeadlineApproaching(teacherID int, classes []UnplannedClass) error {
    // Evitar duplicar: no notificar si ya se notificó hoy
    today := time.Now().Truncate(24 * time.Hour)
    recent, _ := s.repo.FindRecentForUser(teacherID, NotifDeadlineApproaching, today)
    if recent != nil {
        return nil
    }

    message := fmt.Sprintf("Tenés %d clases sin planificar en los próximos días. La primera es en %d días.",
        len(classes), daysUntil(classes[0].Date))

    return s.repo.Create(&Notification{
        UserID:     teacherID,
        Type:       NotifDeadlineApproaching,
        Title:      "Clases sin planificar",
        Message:    message,
        EntityType: "course_subject",
        EntityID:   classes[0].CourseSubjectID,
    })
}
```

### Registro del job

```go
// En main.go o en el setup de la app
func setupScheduledJobs(deps Dependencies) {
    deadlineJob := notifications.NewDeadlineNotificationJob(deps)

    // Ejecutar diariamente a las 8:00 AM
    scheduler.Every(1).Day().At("08:00").Do(deadlineJob.Run)
}
```

## Configuración por org

| Parámetro | Default | Ubicación |
|-----------|---------|-----------|
| `deadline_alert_days` | 7 | `organizations.config.notification_settings` |
| `deadline_reminder_frequency` | daily | `organizations.config.notification_settings` |

## Notas

- El job se ejecuta una vez al día — no es real-time pero suficiente para alertas de planificación
- Se evita duplicar notificaciones del mismo tipo el mismo día para el mismo docente
- El umbral de días es configurable por org (algunas provincias pueden querer 14 días de anticipación)
- Usar un scheduler simple (go-cron o similar) — no se necesita infraestructura de colas para MVP
- Si el server se reinicia, el job se re-registra al arrancar
- Considerar log de ejecución del job para debugging
