# T-7.3.6: Tests del sistema de notificaciones

**Historia:** HU-7.3 — Sistema de notificaciones
**Tipo:** Testing
**Estado:** ⬜ Pendiente
**Fase:** Post-MVP

---

## Descripción

Tests unitarios y de integración para el sistema completo de notificaciones: modelo de datos, generación por eventos, job de plazos y endpoints.

## Casos de test

### Modelo de datos

| Test | Input | Expected |
|------|-------|----------|
| Crear notificación | Datos válidos | Registro con read_at null |
| Marcar como leída | PATCH con ID válido | read_at con timestamp |
| Marcar todas como leídas | POST read-all | Todas las no leídas actualizadas |
| Retención 90 días | Notificación de hace 91 días | Eliminada por job de limpieza |

### Generación por eventos

| Test | Input | Expected |
|------|-------|----------|
| Publicar documento | Doc publicado en área con 3 docentes | 3 notificaciones creadas (una por docente) |
| Publicar con debounce | Doc publicado dos veces en 24hs | Solo notificaciones de la primera vez |
| Revertir a borrador | Doc vuelto a draft | Notificaciones doc_drafted para docentes |
| Planificación completa | Docente completa todas las clases | Notificación para coordinador del área |
| Clase compartida modificada | Docente modifica planificación docente compartida | Notificación para el otro docente |
| Archivar documento | Doc archivado | Notificaciones doc_archived para docentes |

### Job de plazos

| Test | Input | Expected |
|------|-------|----------|
| Clases sin planificar en 5 días | 3 clases pendientes, umbral 7 días | Notificación generada |
| Clases sin planificar en 10 días | 3 clases pendientes, umbral 7 días | No genera notificación (fuera de umbral) |
| Todas planificadas | Docente con todo planificado | No genera notificación |
| Duplicado mismo día | Job ejecutado dos veces hoy | Solo una notificación |
| Umbral configurable | Org con deadline_alert_days: 14 | Alerta a 14 días, no a 7 |

### Endpoints

| Test | Input | Expected |
|------|-------|----------|
| GET notificaciones | User con 15 notificaciones | 200, lista paginada |
| GET unread-count | User con 8 no leídas | 200, `{ unread_count: 8 }` |
| GET con filtro unread_only | unread_only=true | Solo notificaciones sin read_at |
| PATCH read con ID ajeno | Notificación de otro user | 404 Not Found |
| POST read-all | User con 5 no leídas | 200, `{ updated_count: 5 }` |
| Paginación | limit=5, offset=5 | Segundo page de resultados |

### Seguridad

| Test | Input | Expected |
|------|-------|----------|
| Solo mis notificaciones | GET como user A | No ve notificaciones de user B |
| No puedo leer ajena | PATCH read notif de otro user | 404 |
| No puedo listar sin auth | GET sin token | 401 |

## Notas

- Tests del job de plazos requieren mock del clock para controlar "hoy"
- Tests de generación por eventos verifican que se llama al servicio de notificaciones desde los usecases
- Verificar que el índice parcial `WHERE read_at IS NULL` se usa en la query de unread-count
- Tests de debounce requieren control de timestamps
