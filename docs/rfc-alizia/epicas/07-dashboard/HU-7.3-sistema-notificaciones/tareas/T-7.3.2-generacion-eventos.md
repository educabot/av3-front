# T-7.3.2: Generación de notificaciones por eventos

**Historia:** HU-7.3 — Sistema de notificaciones
**Tipo:** Backend
**Estado:** ⬜ Pendiente
**Fase:** Post-MVP

---

## Descripción

Implementar el servicio que genera notificaciones automáticamente cuando ocurren eventos relevantes en el sistema. Incluye tanto eventos síncronos (disparados por acciones del usuario) como eventos programados (job periódico para plazos).

## Implementación

### Servicio de notificaciones

```go
// src/core/usecases/notifications/notification_service.go
package notifications

type NotificationService struct {
    repo     NotificationRepository
    userRepo UserRepository
}

// NotifyDocumentPublished genera notificaciones para todos los docentes del área
func (s *NotificationService) NotifyDocumentPublished(doc CoordinationDocument) error {
    teachers, err := s.userRepo.GetTeachersByAreaID(doc.AreaID)
    if err != nil {
        return err
    }

    for _, teacher := range teachers {
        s.repo.Create(&Notification{
            UserID:         teacher.ID,
            OrganizationID: doc.OrganizationID,
            Type:           NotifDocPublished,
            Title:          "Nuevo documento publicado",
            Message:        fmt.Sprintf("Se publicó el documento \"%s\" para %s", doc.Name, doc.AreaName),
            EntityType:     "coordination_document",
            EntityID:       doc.ID,
        })
    }
    return nil
}

// NotifyPlanningComplete notifica al coordinador cuando un docente termina de planificar
func (s *NotificationService) NotifyPlanningComplete(cs CourseSubject, teacherName string) error {
    coordinator, err := s.userRepo.GetCoordinatorByAreaID(cs.AreaID)
    if err != nil {
        return err
    }

    return s.repo.Create(&Notification{
        UserID:         coordinator.ID,
        OrganizationID: cs.OrganizationID,
        Type:           NotifPlanningComplete,
        Title:          "Planificación completada",
        Message:        fmt.Sprintf("%s completó la planificación de %s (%s)", teacherName, cs.SubjectName, cs.CourseName),
        EntityType:     "course_subject",
        EntityID:       cs.ID,
    })
}

// NotifySharedClassModified notifica al docente de la otra disciplina
func (s *NotificationService) NotifySharedClassModified(plan TeacherLessonPlan, otherTeacherID int) error {
    return s.repo.Create(&Notification{
        UserID:         otherTeacherID,
        OrganizationID: plan.OrganizationID,
        Type:           NotifSharedClassModified,
        Title:          "Clase compartida modificada",
        Message:        fmt.Sprintf("Se modificó la planificación de Clase %d (%s) que compartís", plan.ClassNumber, plan.SubjectName),
        EntityType:     "lesson_plan",
        EntityID:       plan.ID,
    })
}
```

### Integración con usecases existentes

```go
// En el usecase de publicar documento (Épica 4)
func (uc *PublishDocumentUsecase) Execute(docID int) error {
    doc, err := uc.docRepo.Publish(docID)
```

### Anti-debounce para publicación

```go
// Evitar notificar si el documento fue publicado en las últimas 24hs
func (s *NotificationService) NotifyDocumentPublished(doc CoordinationDocument) error {
    recent, _ := s.repo.FindRecent(NotifDocPublished, "coordination_document", doc.ID, 24*time.Hour)
    if recent != nil {
        return nil // ya se notificó recientemente
    }
    // ... generar notificaciones
}
```

## Eventos soportados

| Evento | Trigger | Destinatarios |
|--------|---------|---------------|
| `doc_published` | Coordinador publica doc | Docentes del área |
| `doc_drafted` | Coordinador revierte a borrador | Docentes del área |
| `doc_archived` | Coordinador archiva doc | Docentes del área |
| `shared_class_modified` | Docente modifica planificación docente compartida | Docente de la otra disciplina |
| `planning_complete` | Docente completa todas las clases | Coordinador del área |
| `deadline_approaching` | Job programado | Docentes con clases sin planificar |

## Notas

- Las notificaciones se generan de forma síncrona dentro del usecase — si la generación falla, no bloquea la acción principal (log error, continuar)
- El debounce de 24hs para publicación evita spam si el coordinador publica/despublica varias veces
- `NotifySharedClassModified` requiere resolver quién es el otro docente de la clase compartida
- La notificación de `deadline_approaching` se implementa en T-7.3.3 (job programado)
