### Detección de "documento nuevo"

```sql
-- Documentos que el docente no ha visto (requiere tabla user_document_views)
SELECT cd.* FROM coordination_documents cd
JOIN subjects s ON s.area_id = cd.area_id
JOIN course_subjects cs ON cs.subject_id = s.id
WHERE cs.teacher_id = $1
  AND cd.status = 'published'
  AND NOT EXISTS (
      SELECT 1 FROM user_document_views udv
      WHERE udv.user_id = $1 AND udv.document_id = cd.id
  );
```

## Notas

- Las próximas clases se calculan a partir de la grilla horaria y la fecha actual — no es un campo almacenado
- Límite de 5 clases próximas por default (configurable)
- `is_new` en documentos requiere la tabla `user_document_views` (ver T-7.2.2)
- Las clases compartidas se extraen del mismo cálculo de upcoming_classes filtrando `is_shared: true`
- El endpoint filtra por `teacher_id` del user autenticado — un docente solo ve sus datos
