# T-7.2.4: Tests del dashboard docente

**Historia:** HU-7.2 — Dashboard docente
**Tipo:** Testing
**Estado:** ⬜ Pendiente
**Fase:** Post-MVP

---

## Descripción

Tests de integración para el endpoint del dashboard docente y tests unitarios para el cálculo de próximas clases y tracking de vistas.

## Casos de test

### Endpoint /dashboard/teacher

| Test | Input | Expected |
|------|-------|----------|
| Docente con disciplinas | User con 2 course_subjects | 200, datos de ambas disciplinas |
| Docente sin disciplinas | User sin asignaciones | 200, arrays vacíos |
| Solo ve sus disciplinas | Docente de Matemáticas | No ve datos de Física de otro docente |
| Coordinador intenta acceder | User con rol coordinador | 403 Forbidden |

### Próximas clases

| Test | Input | Expected |
|------|-------|----------|
| Clases esta semana | Schedule con clases lun-vie | Próximas clases ordenadas por fecha |
| Clase con plan | Clase con lesson_plan existente | status: "published", lesson_plan_id set |
| Clase sin plan | Clase sin lesson_plan | status: "pending", lesson_plan_id null |
| Clase compartida | Slot con shared_with | is_shared: true, shared_with poblado |
| Límite de resultados | 20 clases futuras, limit 5 | Solo 5 resultados |

### Tracking de vistas

| Test | Input | Expected |
|------|-------|----------|
| Primera vista | Docente nunca abrió doc | is_new: true |
| Vista registrada | Docente abrió doc después de publicación | is_new: false |
| Re-publicación | Doc re-publicado después del último view | is_new: true |
| UPSERT sin duplicados | Abrir mismo doc dos veces | Un solo registro, last_viewed_at actualizado |

### Planning status

| Test | Input | Expected |
|------|-------|----------|
| Sin planificaciones | 0 lesson_plans de 10 clases | published: 0, total: 10 |
| Planificación parcial | 6 de 10 publicadas | published: 6, total: 10 |
| Planificación completa | 10 de 10 publicadas | published: 10, total: 10 |
| Múltiples disciplinas | 2 disciplinas con distinto progreso | Conteo independiente por disciplina |

## Notas

- El cálculo de próximas clases depende de la fecha actual — usar clock mock en tests
- Verificar que las clases compartidas se detectan correctamente desde el schedule JSONB
- Verificar que `user_document_views` se crea correctamente con el UPSERT
- Tests de integración necesitan datos seed con schedule, docs publicados y lesson plans
