# T-7.2.2: Tracking de vistas de documentos

**Historia:** HU-7.2 — Dashboard docente
**Tipo:** Backend / DB
**Estado:** ⬜ Pendiente
**Fase:** Post-MVP

---

## Descripción

Crear la tabla `user_document_views` para trackear cuándo un usuario vio por última vez un documento de coordinación. Esto permite mostrar el badge "nuevo" en el dashboard del docente cuando hay un documento publicado que aún no abrió.

## Implementación

### Migración

```sql
CREATE TABLE user_document_views (
    id SERIAL PRIMARY KEY,
    user_id INTEGER NOT NULL REFERENCES users(id),
    document_id INTEGER NOT NULL REFERENCES coordination_documents(id),
    last_viewed_at TIMESTAMP NOT NULL DEFAULT NOW(),

    CONSTRAINT unique_user_document UNIQUE (user_id, document_id)
);

CREATE INDEX idx_user_doc_views_user ON user_document_views(user_id);
```

### Entity Go

```go
// src/core/entities/user_document_view.go
package entities

import "time"

type UserDocumentView struct {
    ID           int       `json:"id" gorm:"primaryKey"`
    UserID       int       `json:"user_id"`
    DocumentID   int       `json:"document_id"`
    LastViewedAt time.Time `json:"last_viewed_at"`
}
```

### Registro de vista

El registro se crea/actualiza automáticamente cuando el docente abre un documento:

```go
// Se ejecuta al hacer GET /coordination-documents/{id}
func RecordDocumentView(userID, documentID int) error {
    // UPSERT: crear si no existe, actualizar last_viewed_at si existe
    return db.Exec(`
        INSERT INTO user_document_views (user_id, document_id, last_viewed_at)
        VALUES ($1, $2, NOW())
        ON CONFLICT (user_id, document_id)
        DO UPDATE SET last_viewed_at = NOW()
    `, userID, documentID).Error
}
```

### Consulta de documentos nuevos

```go
// Retorna true si el documento fue publicado después del último view del usuario
func IsDocumentNew(userID, documentID int, publishedAt time.Time) bool {
    var view UserDocumentView
    err := db.Where("user_id = ? AND document_id = ?", userID, documentID).First(&view).Error
    if err != nil {
        return true // nunca lo vio → es nuevo
    }
    return publishedAt.After(view.LastViewedAt)
}
```

## Notas

- El UPSERT garantiza que no haya duplicados (un solo registro por usuario+documento)
- Se registra la vista al abrir el detalle del documento, no al verlo en el listado
- Si un documento se re-publica (vuelve a draft y luego a published), el badge aparece de nuevo
- No se guarda historial de vistas — solo la última vez (`last_viewed_at`)
- Esta tabla también puede usarse en el futuro para analytics de engagement del coordinador
