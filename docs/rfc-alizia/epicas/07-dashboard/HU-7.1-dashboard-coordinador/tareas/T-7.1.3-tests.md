# T-7.1.3: Tests del dashboard coordinador

**Historia:** HU-7.1 — Dashboard coordinador
**Tipo:** Testing
**Estado:** ⬜ Pendiente
**Fase:** Post-MVP

---

## Descripción

Tests de integración para el endpoint del dashboard de coordinador y tests unitarios para la lógica de alertas de atención.

## Casos de test

### Endpoint /dashboard/coordinator

| Test | Input | Expected |
|------|-------|----------|
| Coordinador con documentos | User con área que tiene docs | 200, documentos listados con estado |
| Coordinador sin documentos | User con área vacía | 200, arrays vacíos |
| Progreso de planificación | Doc publicado con 3/5 docentes planificados | Barras de progreso correctas |
| Solo ve su área | Coordinador de Ciencias | No ve documentos de Humanidades |
| Docente intenta acceder | User con rol docente | 403 Forbidden |

### Lógica de atención

| Test | Input | Expected |
|------|-------|----------|
| Draft stale (7 días) | Doc en draft hace 8 días | Aparece alerta `draft_stale` |
| Draft reciente | Doc en draft hace 2 días | No aparece alerta |
| Docente sin planificar | Clase en 10 días sin plan | Aparece alerta `teacher_no_planning` |
| Docente con plan completo | Todas las clases planificadas | No aparece alerta |
| Umbral configurable por org | Config con 14 días para draft | Alerta solo después de 14 días |

### Query de progreso

| Test | Input | Expected |
|------|-------|----------|
| Documento con 0 planificaciones | Doc publicado, ningún plan | 0/N en todas las disciplinas |
| Documento con planificación parcial | 3 de 5 docentes planificaron | Conteo correcto por disciplina |
| Documento con planificación completa | Todos planificaron | N/N, 100% en progreso |
| Múltiples documentos | 2 docs publicados | Progreso independiente por doc |

## Notas

- Tests de integración usan DB real con datos de seed
- Verificar que las queries de agregación son correctas con datos edge case (docente sin disciplina asignada, curso sin docentes)
- Verificar performance con dataset grande (simular 50+ docentes)
