## Descripción

Implementar los endpoints que alimentan los widgets del dashboard del coordinador. Cada endpoint retorna datos agregados optimizados para consumo directo del frontend, sin necesidad de múltiples requests.

## Implementación

### Endpoint principal

```
GET /dashboard/coordinator
```

### Lógica de "Requiere atención"

```go
// src/core/usecases/dashboard/coordinator_attention.go
package dashboard

type AttentionItem struct {
    Type       string `json:"type"`
    Message    string `json:"message"`
    EntityType string `json:"entity_type"`
    EntityID   int    `json:"entity_id"`
    Severity   string `json:"severity"` // "warning", "urgent"
}

// Reglas de atención:
// 1. draft_stale: documento en draft más de N días (configurable por org, default 7)
// 2. teacher_no_planning: docente sin planificar con clases dentro de los próximos N días (default 14)
// 3. unpublished_ready: documento con todos los datos completos pero sin publicar
```

### Query de progreso

```sql
-- Progreso de planificación por documento publicado
SELECT
    cd.id AS document_id,
    cd.name AS document_name,
    s.name AS subject_name,
    u.name AS teacher_name,
    COUNT(tlp.id) FILTER (WHERE tlp.status = 'published') AS published,
    COUNT(cdc.id) AS total
FROM coordination_documents cd
JOIN coord_doc_classes cdc ON cdc.coordination_document_id = cd.id
JOIN subjects s ON s.id = cdc.subject_id
JOIN course_subjects cs ON cs.subject_id = s.id
JOIN users u ON u.id = cs.teacher_id
LEFT JOIN teacher_lesson_plans tlp ON tlp.coord_doc_class_id = cdc.id
    AND tlp.course_subject_id = cs.id
WHERE cd.area_id = $1 AND cd.status = 'published'
GROUP BY cd.id, cd.name, s.name, u.name;
```

## Notas

- Un solo endpoint `/dashboard/coordinator` para evitar múltiples roundtrips del frontend
- El coordinador solo ve datos de su área (filtrado por `area_id`)
- Los umbrales de "requiere atención" (7 días, 14 días) deben ser configurables por org en `organizations.config`
- Si el volumen de datos crece, considerar vistas materializadas para las queries de agregación
- El endpoint no requiere real-time — se carga al entrar y se refresca manualmente
