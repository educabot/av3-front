# T-7.1.2: Frontend del dashboard coordinador

**Historia:** HU-7.1 — Dashboard coordinador
**Tipo:** Frontend
**Estado:** ⬜ Pendiente
**Fase:** Post-MVP

---

## Descripción

Implementar la vista de dashboard para el rol coordinador con los 4 widgets: documentos, progreso de planificación, cursos del área y alertas de atención. La vista se carga al hacer login y se refresca con botón manual.

## Implementación

### Layout

Grid de 2x2 con los widgets:

```
┌──────────────────────┬──────────────────────┐
│ Mis documentos       │ Progreso planificación│
│ (lista + badges)     │ (barras de progreso)  │
├──────────────────────┼──────────────────────┤
│ Cursos del área      │ Requiere atención     │
│ (lista con counts)   │ (alertas con links)   │
└──────────────────────┴──────────────────────┘
```

### Carga de datos

```javascript
async function loadCoordinatorDashboard() {
    state.dashboardLoading = true;
    const data = await fetch('/dashboard/coordinator').then(r => r.json());
    state.dashboard = data;
    state.dashboardLoading = true;
    renderDashboard();
}
```

## Notas

- Responsive: en mobile los widgets se apilan en una sola columna
- El widget de progreso tiene un estado expandido (click) que muestra detalle por disciplina
- El botón de refresh recarga todo el dashboard desde el endpoint
- Si no hay documentos ni cursos, mostrar un empty state con guía de primeros pasos
- Los badges de estado usan los mismos colores que el resto de la plataforma (consistencia)
