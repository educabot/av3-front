# T-1.3.1: Implementar RequireRole middleware

**Historia:** HU-1.3 — Middleware de autorización
**Tipo:** Backend
**Estado:** ✅ Completada

---

## Descripción

Crear un middleware reutilizable que valide que el usuario autenticado tenga al menos uno de los roles requeridos por el endpoint.

## Implementación

```go
// internal/middleware/require_role.go
package middleware

import (
    "context"
    "team-ai-toolkit/tokens"
    "team-ai-toolkit/web"
)

// RequireRole retorna un interceptor que verifica roles.
// Acepta uno o más roles — el usuario debe tener AL MENOS uno.
func RequireRole(roles ...string) web.Interceptor {
    allowed := make(map[string]bool, len(roles))
    for _, r := range roles {
        allowed[r] = true
    }

    return func(ctx context.Context, req web.Request) (context.Context, *web.Response) {
        claims := tokens.GetClaims(ctx)
        if claims == nil {
            return ctx, web.ErrorResponse(403, "forbidden", "no claims in context")
        }

        for _, userRole := range claims.Roles {
            if allowed[userRole] {
                return ctx, nil // permitido, continúa
            }
        }

        return ctx, web.ErrorResponse(403, "forbidden", "insufficient role")
    }
}
```

## Comportamiento

1. Extrae claims del context (puestos por el JWT middleware en HU-1.1)
2. Si no hay claims → 403
3. Itera los roles del usuario y compara contra los roles permitidos
4. Si al menos un rol coincide → continúa al handler
5. Si ninguno coincide → 403

## Notas

- Usa `map[string]bool` para lookup O(1) — más eficiente que iterar un slice
- El middleware es un `web.Interceptor` estándar de team-ai-toolkit
- No hardcodear roles en el middleware — se pasan como parámetro en la configuración de rutas
- `claims.Roles` viene como `[]string` del JWT (ej: `["teacher", "coordinator"]`)
