# T-1.3.2: Interceptor chain: Auth → Tenant → Role

**Historia:** HU-1.3 — Middleware de autorización
**Tipo:** Backend
**Estado:** ✅ Completada

---

## Descripción

Configurar la cadena de interceptors en las rutas para que la autorización por rol se aplique después de autenticación y tenant.

## Implementación

```go
// cmd/routes.go
func ConfigureMappings(engine *gin.Engine, h *entrypoints.WebHandlerContainer, conf *config.Config) {
    api := engine.Group("/api/v1")

    // Middleware global (todos los endpoints protegidos)
    api.Use(webgin.AdaptMiddleware(h.AuthMiddleware))
    api.Use(webgin.AdaptMiddleware(h.TenantMiddleware))

    // Rutas públicas (solo auth + tenant, sin restricción de rol)
    api.GET("/areas", webgin.Adapt(h.Admin.ListAreas.Handle()))
    api.GET("/subjects", webgin.Adapt(h.Admin.ListSubjects.Handle()))
    api.GET("/courses", webgin.Adapt(h.Admin.ListCourses.Handle()))
    api.GET("/topics", webgin.Adapt(h.Admin.ListTopics.Handle()))

    // Rutas de coordinador
    coordOnly := middleware.RequireRole("coordinator", "admin")
    api.POST("/areas", webgin.Adapt(h.Admin.CreateArea.Handle(), coordOnly))
    api.PUT("/areas/:id", webgin.Adapt(h.Admin.UpdateArea.Handle(), coordOnly))
    api.POST("/subjects", webgin.Adapt(h.Admin.CreateSubject.Handle(), coordOnly))
    api.POST("/coordination-documents", webgin.Adapt(h.Coord.CreateDocument.Handle(), coordOnly))
    api.PATCH("/coordination-documents/:id", webgin.Adapt(h.Coord.UpdateDocument.Handle(), coordOnly))
    api.DELETE("/coordination-documents/:id", webgin.Adapt(h.Coord.DeleteDocument.Handle(), coordOnly))

    // Rutas de admin
    adminOnly := middleware.RequireRole("admin")
    api.POST("/courses", webgin.Adapt(h.Admin.CreateCourse.Handle(), adminOnly))
    api.POST("/areas/:id/coordinators", webgin.Adapt(h.Admin.AssignCoordinator.Handle(), adminOnly))
}
```

## Orden de ejecución

```
Request
  → Auth middleware (JWT validation, extracts claims)
  → Tenant middleware (sets org_id in context)
  → RequireRole interceptor (checks roles from claims)
  → Handler
```

## Notas

- Los interceptors de `RequireRole` se pasan como argumento a `webgin.Adapt()`, no como middleware de Gin — esto sigue el patrón de team-ai-toolkit
- Los endpoints GET de listado generalmente son accesibles por todos los roles autenticados (el filtrado es por org_id, no por rol)
- La tabla de permisos por endpoint está en [endpoints.md](../../../tecnico/endpoints.md)
