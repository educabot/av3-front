# T-1.1.1: Configurar JWT authentication

**Historia:** HU-1.1 — Autenticación JWT
**Tipo:** Infra
**Estado:** ✅ Completada

---

## Descripción

Configurar la autenticación JWT para Alizia via team-ai-toolkit/tokens.

## Pasos

1. Configurar JWT secret para validación de tokens:
   - Signing Algorithm: HS256 (symmetric secret)
2. Configurar claims en el JWT:
   - `id` — user ID (string)
   - `name` — user name
   - `email` — user email
   - `avatar` — user avatar URL
   - `roles` — array of roles (teacher, coordinator, admin)
3. Obtener JWT secret para staging

## Variables de entorno resultantes

```
JWT_SECRET=<jwt-secret-staging>
```

## Notas

- Claims struct has: ID, Name, Email, Avatar, Roles (no OrgID)
- Los roles vienen del JWT como claim
- El frontend obtiene el token y lo envia como Bearer token
- team-ai-toolkit/tokens valida via `tokens.New(cfg.JWTSecret)`
