# T-1.1.2: Integrar JWT middleware (HS256)

**Historia:** HU-1.1 — Autenticación JWT
**Tipo:** Backend
**Estado:** ✅ Completada

---

## Descripción

Integrar el middleware de autenticación JWT de `team-ai-toolkit/tokens` para validar tokens via JWT secret.

## Implementación

```go
// cmd/main.go o cmd/routes.go
tokenValidator := tokens.New(cfg.JWTSecret)
authMiddleware := tokens.NewAuthInterceptor(tokenValidator)

// Aplicar a todas las rutas protegidas
api := engine.Group("/api/v1")
api.Use(web.AdaptMiddleware(authMiddleware))
```

## Comportamiento

1. Extrae Bearer token del header `Authorization`
2. Valida firma via JWT secret (HS256)
3. Parsea claims: `id` (string), `name`, `email`, `avatar`, `roles`
4. Inyecta claims en el contexto via `tokens.SetClaims()`

## Errores

- Sin header Authorization → 401 `{"error": "missing_token"}`
- Token malformado o expirado → 401 `{"error": "invalid_token"}`
- Audience no coincide → 401 `{"error": "invalid_token"}`

## Notas

- team-ai-toolkit/tokens valida JWT via symmetric secret (`tokens.New(secret)`)
- No requiere HTTP calls externos para validación
- Claims struct: ID (string), Name, Email, Avatar, Roles
