# T-1.2.3: Provider interfaces

**Historia:** HU-1.2 — Modelo de usuarios y roles
**Tipo:** Backend
**Estado:** ✅ Completada

---

## Descripción

Definir las interfaces de repositorio para organizations y users. Estas interfaces viven en `providers/` y son implementadas en `repositories/`.

## Archivos

### `internal/admin/providers/organization_repository.go`

```go
type OrganizationRepository interface {
    FindByID(ctx context.Context, id uint) (*entities.Organization, error)
    FindBySlug(ctx context.Context, slug string) (*entities.Organization, error)
}
```

### `internal/admin/providers/user_repository.go`

```go
type UserRepository interface {
    FindByID(ctx context.Context, id uint) (*entities.User, error)
    FindByEmail(ctx context.Context, orgID uint, email string) (*entities.User, error)
    FindByOrgID(ctx context.Context, orgID uint) ([]entities.User, error)
    Create(ctx context.Context, user *entities.User) error
    AssignRole(ctx context.Context, userID uint, role string) error
    RemoveRole(ctx context.Context, userID uint, role string) error
    HasRole(ctx context.Context, userID uint, role string) (bool, error)
}
```

## Notas

- Las interfaces son mínimas — solo lo que se necesita ahora
- Se agregan métodos a medida que los usecases los requieran
- Los mocks se generan a partir de estas interfaces en `src/mocks/providers/`
