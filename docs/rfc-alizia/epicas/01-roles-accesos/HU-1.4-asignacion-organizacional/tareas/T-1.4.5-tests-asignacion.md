**Tipo:** Testing
**Estado:** ✅ Completada

## Descripción

Tests de integración para los endpoints de asignación y para el filtrado por rol/asignación en queries.

## Tests — Endpoints de asignación

```go
func TestAssignCoordinator_Success(t *testing.T) {
    // Admin asigna coordinador a área → 201
}

func TestAssignCoordinator_AlreadyAssigned(t *testing.T) {
    // Asignar coordinador duplicado → 409
}

func TestAssignCoordinator_NotAdmin(t *testing.T) {
    // Teacher intenta asignar → 403
}

func TestAssignCoordinator_UserNotInOrg(t *testing.T) {
    // Asignar usuario de otra org → 404
}

func TestRemoveCoordinator_Success(t *testing.T) {
    // Admin remueve coordinador → 204
}
```

## Tests — Filtrado por asignación

```go
func TestCoordinator_SeesOnlyTheirAreas(t *testing.T) {
    // Coordinador asignado a "Ciencias" NO ve documentos de "Humanidades"
}

func TestTeacher_SeesOnlyTheirCourses(t *testing.T) {
    // Docente asignado a Matemáticas en 3a NO ve cursos donde no está asignado
}

func TestAdmin_SeesEverythingInOrg(t *testing.T) {
    // Admin ve todas las áreas, cursos y documentos de su organización
}

func TestMultiTenancy_NoLeakBetweenOrgs(t *testing.T) {
    // Usuario de org A no ve asignaciones de org B
}
```

## Tests — Multi-role

```go
func TestUserWithMultipleRoles_SeesCombinedData(t *testing.T) {
    // Usuario que es teacher + coordinator ve datos de ambos roles
}
```

## Notas

- Tests de integración: usar DB real (PostgreSQL de test), no mocks
- Seed de test con al menos 2 organizaciones, cada una con áreas, coordinadores y docentes distintos
- Verificar aislamiento: que ninguna query filtre por org_id incorrectamente
- Los tests de multi-role son importantes porque un usuario puede ser teacher en una disciplina y coordinator de un área
