# T-1.4.2: Entities y providers para area_coordinators

**Historia:** HU-1.4 — Asignación organizacional
**Tipo:** Backend
**Estado:** ✅ Completada

## Descripción

Definir la entity Go y la interfaz de provider para `area_coordinators`.

> **Nota:** La entity y provider de `course_subjects` se definen en [T-3.4.2](../../../03-integracion/HU-3.4-cursos-alumnos-asignaciones/tareas/T-3.4.2-entities.md) y [T-3.4.3](../../../03-integracion/HU-3.4-cursos-alumnos-asignaciones/tareas/T-3.4.3-providers-repository.md) (Épica 3).

## Entities

```go
// internal/admin/entities/area_coordinator.go
type AreaCoordinator struct {
    ID        uint      `gorm:"primaryKey"`
    AreaID    uint      `gorm:"not null"`
    UserID    uint      `gorm:"not null"`
    CreatedAt time.Time

    // Relations
    Area *Area `gorm:"foreignKey:AreaID"`
    User *User `gorm:"foreignKey:UserID"`
}
```

## Providers

```go
// internal/admin/providers/area_coordinator_repository.go
type AreaCoordinatorRepository interface {
    Assign(ctx context.Context, areaID, userID uint) (*entities.AreaCoordinator, error)
    Remove(ctx context.Context, areaID, userID uint) error
    FindByAreaID(ctx context.Context, areaID uint) ([]entities.AreaCoordinator, error)
    FindByUserID(ctx context.Context, userID uint) ([]entities.AreaCoordinator, error)
    IsCoordinator(ctx context.Context, areaID, userID uint) (bool, error)
}
```

## Notas

- `IsCoordinator` es un helper para validar rápido si un usuario coordina un área (usado en authz de documentos)
