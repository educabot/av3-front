# T-1.4.4: Filtrado por asignación en queries

**Historia:** HU-1.4 — Asignación organizacional
**Tipo:** Backend
**Estado:** ⏳ Parcial — estructura lista, filtrado real se aplica cuando existan handlers de listado (Épica 3-4)

---

## Descripción

Modificar los repositories existentes para que las queries filtren datos según las asignaciones del usuario. Un coordinador solo ve datos de sus áreas; un docente solo ve sus cursos/disciplinas asignados.

## Reglas de filtrado

| Rol | Recurso | Filtro |
|-----|---------|--------|
| coordinator | areas | Solo áreas donde es coordinador (via `area_coordinators`) |
| coordinator | coordination_documents | Solo documentos de sus áreas |
| coordinator | subjects | Solo disciplinas de sus áreas |
| teacher | course_subjects | Solo donde es `teacher_id` (via `course_subjects`, ver HU-3.4) |
| teacher | courses | Solo cursos donde tiene al menos una disciplina asignada (via `course_subjects`, ver HU-3.4) |
| admin | todo | Filtro solo por `organization_id` (ve todo en su org) |

## Implementación — Coordinador

```go
// Repository de documentos: filtrar por áreas del coordinador
func (r *coordDocRepo) FindByUser(ctx context.Context, userID, orgID uint) ([]entities.CoordinationDocument, error) {
    var docs []entities.CoordinationDocument
    err := r.db.WithContext(ctx).
        Joins("JOIN areas ON areas.id = coordination_documents.area_id").
        Joins("JOIN area_coordinators ON area_coordinators.area_id = areas.id").
        Where("area_coordinators.user_id = ? AND coordination_documents.organization_id = ?", userID, orgID).
        Find(&docs).Error
    return docs, err
}
```

## Implementación — Docente

```go
// Repository de course_subjects: filtrar por teacher_id
func (r *courseSubjectRepo) FindByTeacher(ctx context.Context, teacherID, orgID uint) ([]entities.CourseSubject, error) {
    var cs []entities.CourseSubject
    err := r.db.WithContext(ctx).
        Where("teacher_id = ? AND organization_id = ?", teacherID, orgID).
        Preload("Course").Preload("Subject").
        Find(&cs).Error
    return cs, err
}
```

## Notas

- El filtrado se aplica en la capa de repository, no en middleware — el middleware solo valida el rol, el repository decide qué datos mostrar
- Para admin: las queries solo filtran por `organization_id` (acceso completo dentro de la org)
- Patrón: el handler lee `userID` y `roles` del context, decide qué método del repository llamar
- Los endpoints GET de listado deben verificar: si es admin → `FindByOrgID`, si es coordinator → `FindByUser`, si es teacher → `FindByTeacher`
