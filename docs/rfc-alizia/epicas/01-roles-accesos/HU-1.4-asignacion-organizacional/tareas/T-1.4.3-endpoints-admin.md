**Estado:** ✅ Completada

## Descripción

Implementar los endpoints admin para gestionar asignaciones de coordinadores a áreas.

> **Nota:** Los endpoints de `course_subjects` (POST, GET) se definen en [T-3.4.4](../../../03-integracion/HU-3.4-cursos-alumnos-asignaciones/tareas/T-3.4.4-endpoints-handlers.md) (Épica 3).

## Endpoints

### Asignar coordinador a área

```
POST /api/v1/areas/:id/coordinators
Roles: admin
```

**Request:**
```json
{
    "user_id": 3
}
```

**Response 201:**
```json
{
    "id": 1,
    "area_id": 2,
    "user_id": 3,
    "created_at": "2026-01-15T10:00:00Z"
}
```

**Errores:**
- 404 — Área o usuario no existe
- 409 — Ya está asignado (`already_assigned`)
- 403 — No es admin

### Remover coordinador de área

```
DELETE /api/v1/areas/:id/coordinators/:user_id
Roles: admin
```

**Response:** 204 No Content

## Implementación

```go
// Registrar rutas en cmd/routes.go
adminOnly := middleware.RequireRole("admin")

api.POST("/areas/:id/coordinators", webgin.Adapt(h.Admin.AssignCoordinator.Handle(), adminOnly))
api.DELETE("/areas/:id/coordinators/:user_id", webgin.Adapt(h.Admin.RemoveCoordinator.Handle(), adminOnly))
```

## Notas

- Validar que el `user_id` a asignar pertenezca a la misma organización
- Validar que el usuario tenga rol de coordinador
