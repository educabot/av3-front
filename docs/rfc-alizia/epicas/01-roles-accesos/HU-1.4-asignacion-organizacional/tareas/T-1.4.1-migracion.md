# T-1.4.1: Migración: area_coordinators

**Historia:** HU-1.4 — Asignación organizacional
**Tipo:** Backend / DB
**Estado:** ✅ Completada

---

## Descripción

Crear la migración con la tabla de asignación de coordinadores a áreas.

> **Nota:** La tabla `course_subjects` se migra en [T-3.4.1](../../../03-integracion/HU-3.4-cursos-alumnos-asignaciones/tareas/T-3.4.1-migracion.md) (Épica 3).

## SQL (up)

```sql
-- Area coordinators (M2M: users ↔ areas)
CREATE TABLE area_coordinators (
    id SERIAL PRIMARY KEY,
    area_id INTEGER NOT NULL REFERENCES areas(id) ON DELETE CASCADE,
    user_id INTEGER NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    created_at TIMESTAMP NOT NULL DEFAULT NOW(),
    UNIQUE(area_id, user_id)
);

-- Indexes
CREATE INDEX idx_area_coordinators_area_id ON area_coordinators(area_id);
CREATE INDEX idx_area_coordinators_user_id ON area_coordinators(user_id);
```

## SQL (down)

```sql
DROP TABLE IF EXISTS area_coordinators;
```

## Notas

- `area_coordinators` es una tabla join pura (M2M) — un coordinador puede manejar varias áreas y un área puede tener varios coordinadores
- Esta migración depende de que `areas` ya existan (Épica 3)
