# T-0.1.2: Configurar go.mod con dependencias

**Historia:** HU-0.1 — Setup del proyecto
**Tipo:** Config
**Estado:** ⬜ Pendiente

---

## Descripción

Inicializar `go.mod` con el módulo `github.com/educabot/alizia-api` y agregar todas las dependencias directas del proyecto.

## Dependencias Go

```
module alizia-api

go 1.26

require (
    github.com/educabot/team-ai-toolkit  v1.x.x    // Infra compartida (web, boot, tokens, dbconn, errors, pagination, transactions, applog, config)
    gorm.io/gorm                         v1.25.x   // ORM (estándar de la empresa)
    gorm.io/driver/postgres              v1.5.x    // PostgreSQL driver para GORM
    github.com/golang-migrate/migrate    v4.x.x    // DB migrations (up/down, embebidas en binario)
    github.com/stretchr/testify          v1.9.x    // Testing assertions
    github.com/Azure/azure-sdk-for-go/sdk/ai/azopenai  // Azure OpenAI SDK
)
```

**6 dependencias directas.** web/, boot/, tokens/, errors/, pagination/ vienen transitivamente de team-ai-toolkit.

## Pasos

1. `go mod init github.com/educabot/alizia-api`
2. `go get github.com/educabot/team-ai-toolkit`
3. `go get gorm.io/gorm gorm.io/driver/postgres`
4. `go get github.com/golang-migrate/migrate/v4`
5. `go get github.com/Azure/azure-sdk-for-go/sdk/ai/azopenai`
6. `go get github.com/stretchr/testify`
7. Verificar que compila: `go build ./...`

## Notas

- team-ai-toolkit debe estar taggeado (v0.1.0+) para que go mod lo resuelva
- Si el repo es privado, configurar `GOPRIVATE=github.com/educabot/*`
- golang-migrate se usa para migraciones SQL (up/down), no GORM AutoMigrate
- Azure OpenAI SDK se usa para generación de contenido y chat con Alizia
