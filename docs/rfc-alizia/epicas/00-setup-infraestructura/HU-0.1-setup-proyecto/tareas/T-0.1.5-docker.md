## Descripción

Crear Dockerfile multi-stage para producción, docker-compose para desarrollo local, Makefile con targets comunes, Air para hot reload, y pre-commit hooks.

## Dockerfile

```dockerfile
# Build stage
FROM golang:1.22-alpine AS builder
WORKDIR /app
COPY go.mod go.sum ./
RUN go mod download
COPY . .
RUN CGO_ENABLED=0 go build -o /alizia-api ./cmd

# Run stage
FROM alpine:3.19
RUN apk --no-cache add ca-certificates
WORKDIR /app
COPY --from=builder /alizia-api .
EXPOSE 8080
CMD ["./alizia-api"]
```

## docker-compose.yml

```yaml
services:
  db:
    image: postgres:16
    environment:
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: postgres
      POSTGRES_DB: alizia
    ports:
      - "5432:5432"
    volumes:
      - pgdata:/var/lib/postgresql/data

volumes:
  pgdata:
```

## Makefile

```makefile
.PHONY: build test test-cover vet lint docker migrate run

build:
	CGO_ENABLED=0 go build -o alizia-api ./cmd

test:
	go test -race ./...

test-cover:
	go test -race -coverprofile=coverage.out ./...
	go tool cover -func=coverage.out

vet:
	go vet ./...

lint:
	golangci-lint run

docker:
	docker compose up -d

migrate:
	migrate -path db/migrations -database "$$DATABASE_URL" up

run:
	air
```

## .air.toml

```toml
root = "."
tmp_dir = "tmp"

[build]
cmd = "go build -o ./tmp/alizia-api ./cmd"
bin = "./tmp/alizia-api"
include_ext = ["go", "tpl", "tmpl", "html", "txt"]
exclude_dir = ["tmp", "vendor", "frontend", "docs"]
delay = 1000

[log]
time = false

[misc]
clean_on_exit = true
```

## .pre-commit-config.yaml

```yaml
repos:
  - repo: local
    hooks:
      - id: go-vet
        name: go vet
        entry: go vet ./...
        language: system
        types: [go]
        pass_filenames: false
      - id: golangci-lint
        name: golangci-lint
        entry: golangci-lint run
        language: system
        types: [go]
        pass_filenames: false
      - id: go-test
        name: go test
        entry: go test ./...
        language: system
        types: [go]
        pass_filenames: false
```

## Notas

- El servicio Go se corre local con `air` para hot reload (no en Docker)
- docker-compose solo levanta PostgreSQL para desarrollo
- El Dockerfile se usa para Railway y para testing local del build
- Air detecta cambios en archivos .go y recompila automáticamente
- Pre-commit hooks corren vet, lint y tests antes de cada commit
