# T-0.1.7: Deploy inicial a staging

**Historia:** HU-0.1 — Setup del proyecto
**Tipo:** Infra
**Estado:** ⬜ Pendiente

---

## Descripción

Hacer el primer deploy a Railway staging y verificar que `/health` responde.

## Pasos

1. Push a `main` con el proyecto base (estructura + /health)
2. Verificar que Railway detecta el push y hace build
3. Verificar que el build pasa (Dockerfile o Go buildpack)
4. Verificar que `/health` responde 200 en la URL de staging
5. Verificar que los logs muestran el startup de slog

## Criterios de aceptación

- [ ] Build exitoso en Railway
- [ ] `/health` responde 200 `{"status": "ok"}`
- [ ] Logs visibles en Railway dashboard
- [ ] Auto-deploy funciona en pushes posteriores

## Dependencias

- T-0.1.1 a T-0.1.6 completadas
- T-0.1.4 (Railway provisionado)
