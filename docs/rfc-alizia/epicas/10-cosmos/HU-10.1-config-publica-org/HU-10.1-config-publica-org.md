# HU-10.1: Configuracion publica por organizacion

> Como frontend, necesito obtener la configuracion publica de la organizacion (feature flags, identidad visual, nomenclatura) para adaptar la UI al cliente.

**Fase:** Transversal (se implementa junto con HU-3.1)
**Prioridad:** Alta
**Estimacion:** —

---

## Criterios de aceptacion

- [ ] Endpoint `GET /api/v1/organizations/:id/public-config` retorna feature flags, identidad visual y nomenclatura
- [ ] Feature flags (`modules.*`) determinan que secciones de la UI se muestran
- [ ] Middleware `RequireModule(moduleName)` bloquea endpoints de modulos deshabilitados para la org
- [ ] Identidad visual (`visual_identity.*`) sirve logo_url, primary_color, secondary_color, platform_name
- [ ] Nomenclatura (`nomenclature.*`) sirve los terminos configurables (coordination_document, lesson_plan, area)
- [ ] `DefaultConfig()` incluye valores por defecto para los 3 bloques (modules, visual_identity, nomenclature)
- [ ] Endpoint publico (no requiere auth) para que el frontend cargue branding antes del login
