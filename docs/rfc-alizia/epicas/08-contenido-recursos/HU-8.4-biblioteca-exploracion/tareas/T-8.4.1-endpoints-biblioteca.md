# T-8.4.1: Endpoints de biblioteca

**Historia:** HU-8.4 — Biblioteca y exploración
**Tipo:** Backend
**Estado:** ⬜ Pendiente

---

## Descripción

Implementar los endpoints para listar, filtrar y buscar recursos de la organización.

## Implementación

### Endpoints

| Método | Ruta | Descripción |
|--------|------|-------------|
| GET | `/resources` | Listar recursos con filtros |
| GET | `/resources/{id}` | Detalle de un recurso |

### Listar recursos

```
GET /resources?type_id=1&subject_id=2&search=martín&limit=20&offset=0
```

Respuesta:
```json
{
    "resources": [
        {
            "id": 42,
            "title": "Guía de lectura - Martín Fierro",
            "resource_type": {
                "id": 1,
                "slug": "lecture_guide",
                "name": "Guía de lectura"
            },
            "subject": "Lengua y Literatura",
            "course": "3a",
            "creator": "Doc. García",
            "status": "generated",
            "created_at": "2026-03-27T14:00:00Z"
        }
    ],
    "total": 45
}
```

Parámetros de filtro:

| Param | Tipo | Descripción |
|-------|------|-------------|
| `type_id` | int | Filtrar por tipo de recurso |
| `subject_id` | int | Filtrar por disciplina (soft, via course_subject) |
| `search` | string | Búsqueda parcial en título (ILIKE) |
| `created_by` | int | Filtrar por creador |
| `status` | string | Filtrar por estado |
| `limit` | int | Paginación (default 20) |
| `offset` | int | Paginación (default 0) |

### Detalle de recurso

```
GET /resources/{id}
```

Retorna el recurso completo con `content` y `output_schema` del tipo para que el frontend sepa cómo renderizar.

```json
{
    "id": 42,
    "title": "Guía de lectura - Martín Fierro",
    "resource_type": {
        "id": 1,
        "name": "Guía de lectura",
        "output_schema": { ... }
    },
    "content": { ... },
    "font_reference": "Martín Fierro, Cap. 1-3",
    "status": "generated",
    "creator": { "id": 3, "name": "Doc. García" },
    "course_subject": { "subject": "Lengua", "course": "3a" },
    "created_at": "2026-03-27T14:00:00Z"
}
```

### Query de búsqueda

```sql
SELECT r.*, rt.name AS type_name, u.name AS creator_name
FROM resources r
JOIN resource_types rt ON rt.id = r.resource_type_id
JOIN users u ON u.id = r.created_by
LEFT JOIN course_subjects cs ON cs.id = r.course_subject_id
WHERE r.organization_id = $1
  AND ($2::int IS NULL OR r.resource_type_id = $2)
  AND ($3::int IS NULL OR cs.subject_id = $3)
  AND ($4::text IS NULL OR r.title ILIKE '%' || $4 || '%')
ORDER BY r.created_at DESC
LIMIT $5 OFFSET $6;
```

## Notas

- Todos los recursos son accesibles a nivel organización — no hay filtro de permisos por docente
- El filtro por disciplina es via JOIN con course_subjects, no es FK directa
- La búsqueda por título usa ILIKE (case-insensitive, parcial)
- El detalle incluye el `output_schema` del tipo para que el frontend sepa cómo renderizar el content
- `organization_id` se resuelve del token — un docente solo ve recursos de su org
