## Descripción

Tests para los endpoints de biblioteca y la lógica de filtrado.

## Casos de test

### Listado y filtros

| Test | Input | Expected |
|------|-------|----------|
| Sin filtros | GET /resources | Todos los recursos de la org |
| Filtro por tipo | type_id=1 | Solo guías de lectura |
| Filtro por disciplina | subject_id=2 | Recursos vinculados a esa disciplina |
| Filtro por creador | created_by=3 | Solo recursos del docente |
| Búsqueda por título | search=martín | Recursos con "martín" en título (case-insensitive) |
| Combinación de filtros | type_id=1&subject_id=2&search=guía | Intersección de filtros |
| Sin resultados | search=xyzinexistente | Array vacío, total 0 |
| Paginación | limit=5&offset=5 | Segunda página |

### Detalle

| Test | Input | Expected |
|------|-------|----------|
| Recurso existente | GET /resources/42 | 200, content + output_schema |
| Recurso de otra org | GET recurso de org B como user de org A | 404 |
| Recurso inexistente | GET /resources/99999 | 404 |

### Seguridad

| Test | Input | Expected |
|------|-------|----------|
| Solo mi org | User de org A lista recursos | No ve recursos de org B |
| Sin auth | GET sin token | 401 |
