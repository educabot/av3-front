# T-8.1.1: Migración: tablas de recursos

**Historia:** HU-8.1 — Modelo de datos de recursos
**Tipo:** DB
**Estado:** ⬜ Pendiente

---

## Descripción

Crear las tablas `fonts`, `resource_types`, `organization_resource_types` y `resources` con sus índices, enums y constraints.

## Implementación

```sql
CREATE TYPE resource_status AS ENUM ('draft', 'generated', 'published');

-- Fuentes educativas (PDFs, videos, documentos oficiales)
CREATE TABLE fonts (
    id SERIAL PRIMARY KEY,
    organization_id INTEGER NOT NULL REFERENCES organizations(id),
    name VARCHAR(255) NOT NULL,
    description TEXT,
    file_url TEXT NOT NULL,
    file_type VARCHAR(50),
    thumbnail_url TEXT,
    area_id INTEGER REFERENCES areas(id),
    is_validated BOOLEAN NOT NULL DEFAULT false,
    created_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_fonts_org ON fonts(organization_id);
CREATE INDEX idx_fonts_area ON fonts(area_id);

-- Tipos de recurso (globales y privados)
CREATE TABLE resource_types (
    id SERIAL PRIMARY KEY,
    slug VARCHAR(50) NOT NULL UNIQUE,
    name VARCHAR(100) NOT NULL,
    description TEXT,
    prompt TEXT NOT NULL,                   -- prompt por defecto para generación
    output_schema JSONB NOT NULL,           -- JSON Schema del output esperado
    requires_font BOOLEAN NOT NULL DEFAULT false,
    is_active BOOLEAN NOT NULL DEFAULT true,
    organization_id INTEGER REFERENCES organizations(id),  -- NULL = público
    created_at TIMESTAMP DEFAULT NOW()
);

-- Activación y customización por organización
CREATE TABLE organization_resource_types (
    id SERIAL PRIMARY KEY,
    organization_id INTEGER NOT NULL REFERENCES organizations(id),
    resource_type_id INTEGER NOT NULL REFERENCES resource_types(id),
    enabled BOOLEAN NOT NULL DEFAULT true,
    custom_prompt TEXT,                     -- sobreescribe resource_types.prompt
    custom_output_schema JSONB,            -- sobreescribe resource_types.output_schema
    created_at TIMESTAMP DEFAULT NOW(),

    CONSTRAINT unique_org_resource_type UNIQUE (organization_id, resource_type_id)
);

-- Recursos generados
CREATE TABLE resources (
    id SERIAL PRIMARY KEY,
    organization_id INTEGER NOT NULL REFERENCES organizations(id),
    resource_type_id INTEGER NOT NULL REFERENCES resource_types(id),
    course_subject_id INTEGER REFERENCES course_subjects(id),  -- contexto, nullable
    title VARCHAR(200) NOT NULL,
    content JSONB NOT NULL DEFAULT '{}',    -- contenido generado según output_schema
    font_id INTEGER REFERENCES fonts(id),   -- fuente seleccionada al crear (si aplica)
    status resource_status NOT NULL DEFAULT 'draft',
    created_by INTEGER NOT NULL REFERENCES users(id),
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_resources_org ON resources(organization_id);
CREATE INDEX idx_resources_type ON resources(resource_type_id);
CREATE INDEX idx_resources_course_subject ON resources(course_subject_id);
CREATE INDEX idx_resources_created_by ON resources(created_by);
CREATE INDEX idx_resources_font ON resources(font_id);
CREATE INDEX idx_resource_types_org ON resource_types(organization_id);
```

## Notas

- `resource_types.organization_id` NULL = tipo público (visible para todas las orgs)
- `organization_resource_types` controla qué tipos están habilitados por org y permite customizar prompt/schema
- `resources.content` almacena el output generado según el schema del tipo — el frontend lo renderiza dinámicamente
- `resources.course_subject_id` es nullable porque un recurso puede crearse sin contexto de disciplina específica
- `font_id` es FK nullable a `fonts` — se usa cuando el tipo de recurso requiere fuente (`requires_font = true`)
- `fonts` almacena fuentes educativas oficiales (PDFs, videos, documentos del ministerio). `is_validated` indica aprobacion por coordinadores
- `resource_types.is_active` permite desactivar un tipo globalmente (diferente de `organization_resource_types.enabled` que es por org)
