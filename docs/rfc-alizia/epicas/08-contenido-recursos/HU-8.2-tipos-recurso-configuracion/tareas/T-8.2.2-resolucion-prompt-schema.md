# T-8.2.2: Resolución de prompt y schema

**Historia:** HU-8.2 — Tipos de recurso y configuración por organización
**Tipo:** Backend
**Estado:** ⬜ Pendiente

---

## Descripción

Implementar la lógica de resolución que determina qué prompt y output_schema usar para generar un recurso, priorizando la configuración custom de la organización sobre los defaults del tipo.

## Implementación

```go
// src/core/usecases/resources/prompt_resolver.go
package resources

// ResolvedConfig contiene el prompt y schema efectivos para una generación.
type ResolvedConfig struct {
    Prompt       string
    OutputSchema JSON
}

// ResolvePromptAndSchema resuelve la configuración efectiva para un tipo de recurso en una org.
// Prioridad: custom_prompt/custom_output_schema de la org > defaults del resource_type.
func ResolvePromptAndSchema(
    resourceType entities.ResourceType,
    orgConfig *entities.OrganizationResourceType,
) ResolvedConfig {
    config := ResolvedConfig{
        Prompt:       resourceType.Prompt,
        OutputSchema: resourceType.OutputSchema,
    }

    if orgConfig != nil {
        if orgConfig.CustomPrompt != nil && *orgConfig.CustomPrompt != "" {
            config.Prompt = *orgConfig.CustomPrompt
        }
        if orgConfig.CustomOutputSchema != nil {
            config.OutputSchema = *orgConfig.CustomOutputSchema
        }
    }

    return config
}
```

### Uso en el flujo de generación

```go
// En el usecase de generación (HU-8.3)
func (uc *GenerateResourceUsecase) Execute(req GenerateRequest) (*entities.Resource, error) {
    resourceType, _ := uc.typeRepo.GetByID(req.ResourceTypeID)
    orgConfig, _ := uc.orgTypeRepo.GetByOrgAndType(req.OrgID, req.ResourceTypeID)

    resolved := ResolvePromptAndSchema(*resourceType, orgConfig)

    // Usar resolved.Prompt y resolved.OutputSchema para la generación
    content, err := uc.aiProvider.GenerateJSON(ctx, resolved.Prompt, userPrompt, resolved.OutputSchema)
    // ...
}
```

## Notas

- La resolución es un fallback simple: custom ?? default. No hay merge parcial de schemas
- Si la org customizó el output_schema, el frontend debe renderizar según el nuevo schema
- La resolución se ejecuta en cada generación, no se cachea — la config puede cambiar en cualquier momento
- Si un custom_prompt está vacío (""), se ignora y se usa el default
