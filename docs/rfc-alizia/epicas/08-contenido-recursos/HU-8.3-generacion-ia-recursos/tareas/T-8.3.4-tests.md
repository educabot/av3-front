# T-8.3.4: Tests

**Historia:** HU-8.3 — Generación de recursos con IA
**Tipo:** Testing
**Estado:** ⬜ Pendiente

---

## Descripción

Tests del flujo de generación de recursos: usecase, endpoint y context builder.

## Casos de test

### Usecase

| Test | Input | Expected |
|------|-------|----------|
| Generación exitosa | Tipo válido + datos completos | Recurso creado con status generated |
| Tipo requiere fuente, sin fuente | requires_font=true, font_reference nil | Error validación |
| Tipo no encontrado | resource_type_id inexistente | Error 404 |
| Fallo de IA | AI Provider retorna error | Error, recurso NO creado |
| Creación libre con instrucciones | type=free_creation + instructions | Instrucciones en el prompt |

### Endpoint

| Test | Input | Expected |
|------|-------|----------|
| POST generate exitoso | Datos válidos | 201, recurso con content |
| Tipo deshabilitado para org | Tipo con enabled=false | 403 |
| Sin título | title vacío | 400 |
| Con course_subject | course_subject_id válido | Recurso vinculado |
| Sin course_subject | course_subject_id null | Recurso sin vínculo (OK) |

### Context builder

| Test | Input | Expected |
|------|-------|----------|
| Contexto básico | Solo tipo + título | String con tipo y título |
| Con fuente | font_reference set | Incluye fuente |
| Contexto enriquecido | course_subject + topics | Incluye disciplina, curso, alumnos, topics |
| Sin course_subject | cs nil | Solo contexto básico |
| Sin topics | topics vacío | No incluye sección de temas |

## Notas

- Tests del usecase requieren mock del AI Provider
- Verificar que el content generado sea JSON válido que cumple el output_schema
- Tests del context builder son puramente unitarios
