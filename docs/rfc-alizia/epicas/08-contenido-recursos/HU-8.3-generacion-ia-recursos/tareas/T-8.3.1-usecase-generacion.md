# T-8.3.1: Usecase de generación de recursos

**Historia:** HU-8.3 — Generación de recursos con IA
**Tipo:** Backend
**Estado:** ⬜ Pendiente

---

## Descripción

Implementar el usecase que orquesta la generación de un recurso: valida el tipo, resuelve prompt/schema, construye contexto, llama al AI Provider con GenerateJSON, y almacena el resultado.

## Implementación

```go
// src/core/usecases/resources/generate_resource.go
package resources

type GenerateResourceRequest struct {
    OrganizationID  int    `json:"organization_id"`
    ResourceTypeID  int    `json:"resource_type_id"`
    CourseSubjectID *int   `json:"course_subject_id,omitempty"`
    Title           string `json:"title"`
    FontReference   *string `json:"font_reference,omitempty"`
    UserInstructions *string `json:"user_instructions,omitempty"` // para creación libre
    CreatedBy       int    `json:"created_by"`
}

type GenerateResourceUsecase struct {
    typeRepo    providers.ResourceTypeProvider
    orgTypeRepo providers.OrganizationResourceTypeProvider
    resourceRepo providers.ResourceProvider
    csRepo      providers.CourseSubjectProvider
    aiProvider  providers.AIProvider
}

func (uc *GenerateResourceUsecase) Execute(ctx context.Context, req GenerateResourceRequest) (*entities.Resource, error) {
    // 1. Obtener y validar tipo de recurso
    resourceType, err := uc.typeRepo.GetByID(req.ResourceTypeID)
    if err != nil {
        return nil, fmt.Errorf("resource type not found: %w", err)
    }

    // 2. Validar que requiere fuente si aplica
    if resourceType.RequiresFont && (req.FontReference == nil || *req.FontReference == "") {
        return nil, fmt.Errorf("this resource type requires a font reference")
    }

    // 3. Resolver prompt y schema (custom de org o default)
    orgConfig, _ := uc.orgTypeRepo.GetByOrgAndType(req.OrganizationID, req.ResourceTypeID)
    resolved := ResolvePromptAndSchema(*resourceType, orgConfig)

    // 4. Construir contexto
    contextStr := BuildResourceContext(req, resourceType)

    // 5. Construir user prompt
    userPrompt := contextStr
    if req.UserInstructions != nil {
        userPrompt += "\n\nInstrucciones del docente:\n" + *req.UserInstructions
    }

    // 6. Generar con IA
    content, err := uc.aiProvider.GenerateJSON(ctx, resolved.Prompt, userPrompt, resolved.OutputSchema)
    if err != nil {
        return nil, fmt.Errorf("AI generation failed: %w", err)
    }

    // 7. Crear recurso en DB
    resource := &entities.Resource{
        OrganizationID:  req.OrganizationID,
        ResourceTypeID:  req.ResourceTypeID,
        CourseSubjectID: req.CourseSubjectID,
        Title:           req.Title,
        Content:         content,
        FontReference:   req.FontReference,
        Status:          entities.ResourceGenerated,
        CreatedBy:       req.CreatedBy,
    }

    if err := uc.resourceRepo.Create(resource); err != nil {
        return nil, err
    }

    return resource, nil
}
```

## Notas

- El recurso se crea directamente en estado `generated` (no pasa por `draft` en el flujo normal)
- Si la generación falla, no se crea el recurso — el docente puede reintentar
- `UserInstructions` solo se usa para "Creación libre" — se agrega al user prompt
- El AI Provider usa `GenerateJSON` con el output_schema resuelto para garantizar structured output
- El content se almacena como JSONB — su estructura varía según el tipo de recurso
