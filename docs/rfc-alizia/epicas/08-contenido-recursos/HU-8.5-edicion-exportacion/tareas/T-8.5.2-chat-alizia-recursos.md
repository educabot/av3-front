# T-8.5.2: Chat con Alizia para recursos

**Historia:** HU-8.5 — Edición y exportación
**Tipo:** Backend
**Estado:** ⬜ Pendiente

---

## Descripción

Implementar el chat con Alizia dentro de la vista de edición de recursos. Alizia puede modificar secciones del content via function calling, siguiendo el mismo patrón del chat de documentos de coordinación (HU-4.6).

## Implementación

### Endpoint

```
POST /resources/{id}/chat
```

```json
{
    "message": "Cambiá la introducción para que sea más motivadora y agregá una actividad de cierre grupal",
    "history": [...]
}
```

### Tools disponibles

```go
// Tools para edición de recursos via function calling
var ResourceChatTools = []providers.ToolDefinition{
    {
        Name:        "update_resource_section",
        Description: "Updates a specific section of the resource content",
        Parameters: map[string]interface{}{
            "type": "object",
            "properties": map[string]interface{}{
                "section_key": map[string]string{
                    "type":        "string",
                    "description": "Key of the section to update (e.g., 'introduction', 'pre_reading')",
                },
                "new_value": map[string]string{
                    "type":        "string",
                    "description": "New content for the section (string or JSON string for arrays/objects)",
                },
            },
            "required": []string{"section_key", "new_value"},
        },
    },
    {
        Name:        "append_to_section",
        Description: "Appends an item to an array section of the resource",
        Parameters: map[string]interface{}{
            "type": "object",
            "properties": map[string]interface{}{
                "section_key": map[string]string{
                    "type":        "string",
                    "description": "Key of the array section",
                },
                "item": map[string]string{
                    "type":        "string",
                    "description": "Item to append (string or JSON string for objects)",
                },
            },
            "required": []string{"section_key", "item"},
        },
    },
    {
        Name:        "remove_from_section",
        Description: "Removes an item from an array section by index",
        Parameters: map[string]interface{}{
            "type": "object",
            "properties": map[string]interface{}{
                "section_key": map[string]string{
                    "type":        "string",
                    "description": "Key of the array section",
                },
                "index": map[string]string{
                    "type":        "integer",
                    "description": "Index of the item to remove (0-based)",
                },
            },
            "required": []string{"section_key", "index"},
        },
    },
}
```

### System prompt

```
Sos Alizia, asistente pedagógica. Estás ayudando a un docente a editar un recurso didáctico.

Recurso actual:
- Tipo: {resource_type_name}
- Título: {title}
- Contenido actual: {content_json}

Podés usar las herramientas para modificar secciones del recurso.
Si el docente pide cambios, usá las herramientas para aplicarlos.
Si pide algo que no se puede hacer con las herramientas, explicá las limitaciones.
```

## Notas

- Sigue el mismo patrón de ChatWithTools que el chat de documentos (HU-4.6)
- Las tools operan sobre las keys del content JSONB — son genéricas para cualquier tipo de recurso
- `update_resource_section` reemplaza; `append_to_section` agrega a arrays; `remove_from_section` elimina por índice
- El content actual se incluye en el system prompt para que Alizia conozca el estado del recurso
- El historial de chat se envía con cada request (mismo patrón del frontend)
- Máximo 5 iteraciones de tool calling por seguridad (igual que HU-6.3)
