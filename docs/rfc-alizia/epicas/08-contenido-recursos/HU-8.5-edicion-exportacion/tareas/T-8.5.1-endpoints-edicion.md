# T-8.5.1: Endpoints de edición

**Historia:** HU-8.5 — Edición y exportación
**Tipo:** Backend
**Estado:** ⬜ Pendiente

---

## Descripción

Implementar los endpoints para editar el contenido de un recurso, cambiar su estado y eliminarlo.

## Implementación

### Endpoints

| Método | Ruta | Descripción |
|--------|------|-------------|
| PATCH | `/resources/{id}` | Actualizar contenido, título o estado |
| DELETE | `/resources/{id}` | Eliminar recurso |
| POST | `/resources/{id}/publish` | Marcar como publicado |

### Actualizar recurso

```
PATCH /resources/{id}
```

```json
{
    "title": "Guía de lectura - Martín Fierro (actualizada)",
    "content": {
        "introduction": "Texto editado...",
        "pre_reading": ["Nueva pregunta"],
        "during_reading": [
            { "question": "Pregunta editada", "page_reference": "Cap. 1" }
        ],
        "post_reading": ["Actividad editada"],
        "synthesis": "Síntesis editada"
    }
}
```

El PATCH acepta actualizaciones parciales:
- Solo `title` → actualiza título
- Solo `content` → actualiza contenido completo
- Ambos → actualiza ambos

### Publicar recurso

```
POST /resources/{id}/publish
```

Cambia el status de `generated` (o `draft`) a `published`.

Respuesta:
```json
{
    "id": 42,
    "status": "published",
    "updated_at": "2026-03-27T16:00:00Z"
}
```

### Eliminar recurso

```
DELETE /resources/{id}
```

Solo el creador del recurso puede eliminarlo.

### Validaciones

| Validación | Condición | Error |
|-----------|-----------|-------|
| Recurso existe | ID válido | 404 |
| Pertenece a mi org | resource.organization_id == user.org_id | 404 |
| Solo creador puede editar/eliminar | resource.created_by == user.id | 403 |
| Content válido según schema | Validar content contra output_schema | 400 |

## Notas

- Solo el creador puede editar y eliminar — otros docentes pueden ver pero no modificar
- La validación de content contra output_schema es deseable pero no bloqueante en MVP (el frontend ya lo estructura)
- El estado published no es "bloqueante" — el docente puede seguir editando después de publicar
- Publicar es una acción semántica: indica que el recurso está listo para usar en clase
