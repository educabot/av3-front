# T-6.5.5: Tests de customizacion por organizacion

**Historia:** HU-6.5 — Customizacion por organizacion
**Tipo:** Testing
**Estado:** ⬜ Pendiente
**Fase:** Post-MVP

---

## Descripcion

Tests unitarios y de integracion para validar que la configuracion de IA por organizacion se aplica correctamente en cada capa: settings, feature flags, vocabulario, tono y prompts custom.

## Casos de test

### AISettings

| Test | Input | Expected |
|------|-------|----------|
| Default settings cuando config vacio | org sin `ai_settings` | Retorna `DefaultAISettings()` |
| Parse correcto de ai_settings | JSON valido en config | Struct poblado correctamente |
| IsFeatureEnabled con feature existente | `["generation", "chat"]`, buscar `"chat"` | `true` |
| IsFeatureEnabled con feature inexistente | `["generation"]`, buscar `"chat"` | `false` |

### VocabularyProcessor

| Test | Input | Expected |
|------|-------|----------|
| Sin overrides | texto cualquiera, overrides vacio | Texto sin cambios |
| Reemplazo simple | `"contenidos"` → `"saberes"` | Texto con reemplazo aplicado |
| Reemplazo capitalizado | `"Contenidos"` → `"Saberes"` | Preserva capitalizacion |
| Multiples reemplazos | 3 overrides sobre texto con las 3 palabras | Todos reemplazados |
| Palabra no encontrada | override para palabra que no esta en el texto | Texto sin cambios |

### Feature flags middleware

| Test | Input | Expected |
|------|-------|----------|
| Feature habilitada | org con `"chat"` enabled, request a chat | 200 (pasa al handler) |
| Feature deshabilitada | org sin `"chat"` enabled, request a chat | 403 Forbidden |
| Org sin config | org sin ai_settings, request a generation | 200 (defaults incluyen generation) |

### Tone injection

| Test | Input | Expected |
|------|-------|----------|
| Tono default | sin tone en config | System prompt incluye instruccion neutral_formal |
| Tono rioplatense | `tone: "rioplatense_informal"` | System prompt incluye instruccion con voseo |
| Tono invalido | `tone: "inexistente"` | Fallback a neutral_formal |

### Prompt overrides

| Test | Input | Expected |
|------|-------|----------|
| Sin override | seccion `"strategy"` sin override | Usa prompt default |
| Con override | seccion `"strategy"` con override definido | Usa prompt custom |
| Override no afecta otra seccion | override para `"strategy"`, generar `"class_plan"` | class_plan usa prompt default |

## Notas

- Los tests del middleware requieren un org mock con configuracion inyectable
- Los tests del VocabularyProcessor son puramente unitarios (no dependen de DB ni provider)
- Los tests de tone y prompt overrides validan el output del SystemPromptBuilder, no la respuesta del LLM
