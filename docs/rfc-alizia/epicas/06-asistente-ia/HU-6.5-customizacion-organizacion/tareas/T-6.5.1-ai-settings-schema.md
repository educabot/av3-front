# T-6.5.1: Schema de ai_settings en configuracion de organizacion

**Historia:** HU-6.5 — Customizacion por organizacion
**Tipo:** Backend
**Estado:** ⬜ Pendiente
**Fase:** Post-MVP

---

## Descripcion

Definir y migrar el campo `ai_settings` dentro de `organizations.config` (JSONB). Este campo centraliza toda la configuracion de IA por organizacion: tono, limites, features habilitadas, fuentes curriculares, vocabulario y prompts custom.

## Implementacion

### Migracion

```sql
-- Agregar ai_settings al config existente de cada organizacion
UPDATE organizations
SET config = jsonb_set(
    COALESCE(config, '{}'),
    '{ai_settings}',
    '{
        "tone": "neutral_formal",
        "max_generation_length": 2000,
        "max_chat_interactions": 20,
        "enabled_features": ["generation", "chat"],
        "curriculum_sources": [],
        "vocabulary_overrides": {},
        "prompt_overrides": {}
    }'
)
WHERE config -> 'ai_settings' IS NULL;
```

### Estructura Go

```go
// src/core/entities/ai_settings.go
package entities

// AISettings define la configuracion de IA por organizacion.
type AISettings struct {
    Tone                  string            `json:"tone"`
    MaxGenerationLength   int               `json:"max_generation_length"`
    MaxChatInteractions   int               `json:"max_chat_interactions"`
    EnabledFeatures       []string          `json:"enabled_features"`
    CurriculumSources     []string          `json:"curriculum_sources"`
    VocabularyOverrides   map[string]string `json:"vocabulary_overrides"`
    PromptOverrides       map[string]string `json:"prompt_overrides"`
}

// DefaultAISettings retorna la configuracion por defecto para orgs sin customizacion.
func DefaultAISettings() AISettings {
    return AISettings{
        Tone:                "neutral_formal",
        MaxGenerationLength: 2000,
        MaxChatInteractions: 20,
        EnabledFeatures:     []string{"generation", "chat"},
    }
}
```

### Valores de tone

| Valor | Descripcion |
|-------|-------------|
| `neutral_formal` | Espanol neutro, tono profesional (default) |
| `neutral_informal` | Espanol neutro, tono cercano |
| `rioplatense_formal` | Voseo, tono profesional |
| `rioplatense_informal` | Voseo, tono cercano |

## Notas

- Si `ai_settings` no existe en el config de la org, se usa `DefaultAISettings()`
- Los valores de `tone` se inyectan en el SystemPromptBuilder (HU-6.2) como instruccion de estilo
- `enabled_features` controla que endpoints de IA estan disponibles — se valida en middleware (T-6.5.3)
- `curriculum_sources` referencia identificadores de documentos curriculares que se resuelven en el context builder (HU-6.4)
