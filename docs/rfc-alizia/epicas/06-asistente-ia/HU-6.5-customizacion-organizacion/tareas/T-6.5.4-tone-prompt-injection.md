# T-6.5.4: Inyeccion de tono y prompts custom en SystemPromptBuilder

**Historia:** HU-6.5 — Customizacion por organizacion
**Tipo:** Backend
**Estado:** ⬜ Pendiente
**Fase:** Post-MVP

---

## Descripcion

Extender el SystemPromptBuilder (HU-6.2) para que inyecte el tono configurado por organizacion y soporte prompts custom que sobreescriban los prompts por defecto. Esto permite que cada provincia defina el estilo comunicacional de Alizia y ajuste las instrucciones de generacion sin cambios de codigo.

## Implementacion

### Instrucciones de tono

```go
// src/core/usecases/ai/tone_instructions.go
package ai

// ToneInstructions mapea cada valor de tone a instrucciones para el system prompt.
var ToneInstructions = map[string]string{
    "neutral_formal": "Comunicate en español neutro con tono profesional y respetuoso. Usa 'usted' para dirigirte al usuario.",
    "neutral_informal": "Comunicate en español neutro con tono cercano y amigable. Usa 'tú' para dirigirte al usuario.",
    "rioplatense_formal": "Comunicate en español rioplatense con tono profesional. Usa 'vos' pero mantené un registro formal.",
    "rioplatense_informal": "Comunicate en español rioplatense con tono cercano y amigable. Usa 'vos' y un registro coloquial pero respetuoso.",
}

// GetToneInstruction retorna la instruccion de tono para el system prompt.
// Si el tone no existe, retorna el default (neutral_formal).
func GetToneInstruction(tone string) string {
    if instruction, ok := ToneInstructions[tone]; ok {
        return instruction
    }
    return ToneInstructions["neutral_formal"]
}
```

### Extension del SystemPromptBuilder

```go
// Modificar el builder existente (HU-6.2) para aceptar ai_settings

func (b *SystemPromptBuilder) WithAISettings(settings entities.AISettings) *SystemPromptBuilder {
    // Inyectar tono
    b.toneInstruction = GetToneInstruction(settings.Tone)

    // Inyectar fuentes curriculares como contexto
    if len(settings.CurriculumSources) > 0 {
        b.curriculumContext = formatCurriculumSources(settings.CurriculumSources)
    }

    // Sobreescribir prompt si hay override para esta seccion
    if override, ok := settings.PromptOverrides[b.section]; ok {
        b.customPrompt = override
    }

    return b
}

// Build() prioriza: customPrompt > defaultPrompt, e inyecta tone siempre
func (b *SystemPromptBuilder) Build() string {
    basePrompt := b.defaultPrompt
    if b.customPrompt != "" {
        basePrompt = b.customPrompt
    }

    return fmt.Sprintf("%s\n\n## Estilo de comunicacion\n%s\n\n%s",
        basePrompt,
        b.toneInstruction,
        b.curriculumContext,
    )
}
```

## Comportamiento

1. El usecase obtiene los `AISettings` de la organizacion del usuario
2. Pasa los settings al SystemPromptBuilder via `WithAISettings()`
3. El builder inyecta el tono como seccion del system prompt
4. Si hay un `prompt_override` para la seccion actual (ej: `"strategy"`, `"class_plan"`), reemplaza el prompt default
5. Las fuentes curriculares se agregan como contexto adicional al final del prompt

## Notas

- El tono se inyecta en TODOS los system prompts — es transversal a todas las generaciones
- Los prompt overrides son por seccion: `"strategy"`, `"class_plan"`, `"chat"`. Si no hay override, se usa el prompt default
- Validar que los prompts custom no rompan el formato esperado — considerar un modo "preview" en una iteracion futura
- Las fuentes curriculares se resuelven a texto descriptivo, no se inyecta el documento completo
