# T-6.5.2: Post-procesador de vocabulario

**Historia:** HU-6.5 — Customizacion por organizacion
**Tipo:** Backend
**Estado:** ⬜ Pendiente
**Fase:** Post-MVP

## Descripcion

Implementar un post-procesador que aplique las sobreescrituras de vocabulario definidas por organizacion sobre el texto generado por el LLM. Es un find & replace simple que permite adaptar la terminologia a cada jurisdiccion sin regenerar contenido.

## Implementacion

```go
// src/core/usecases/ai/vocabulary_processor.go
package ai

import "strings"

// VocabularyProcessor aplica reemplazos de terminologia sobre texto generado.
type VocabularyProcessor struct {
    overrides map[string]string
}

// NewVocabularyProcessor crea un procesador a partir del mapa de overrides de la org.
// Si overrides es nil o vacio, Apply() retorna el texto sin cambios.
func NewVocabularyProcessor(overrides map[string]string) *VocabularyProcessor {
    return &VocabularyProcessor{overrides: overrides}
}

// Apply ejecuta los reemplazos sobre el texto.
// Los reemplazos son case-insensitive y preservan el casing original cuando es posible.
func (p *VocabularyProcessor) Apply(text string) string {
    if len(p.overrides) == 0 {
        return text
    }
    result := text
    for original, replacement := range p.overrides {
        result = strings.ReplaceAll(result, original, replacement)
        // Tambien reemplazar version capitalizada
        if len(original) > 0 {
            capitalized := strings.ToUpper(original[:1]) + original[1:]
            capitalizedReplacement := strings.ToUpper(replacement[:1]) + replacement[1:]
            result = strings.ReplaceAll(result, capitalized, capitalizedReplacement)
        }
    }
    return result
}
```

## Integracion

El procesador se ejecuta como paso final despues de cada generacion de texto:

```go
// En el usecase de generacion (ejemplo: estrategia metodologica)
rawText, err := aiProvider.GenerateText(ctx, systemPrompt, userPrompt)
if err != nil {
    return "", err
}

processor := ai.NewVocabularyProcessor(org.AISettings.VocabularyOverrides)
return processor.Apply(rawText), nil
```

## Ejemplos de overrides por provincia

```json
{
    "contenidos": "saberes",
    "planificación": "secuencia didáctica",
    "NAP": "contenidos prioritarios",
    "evaluación": "instancia de valoración",
    "objetivos": "propósitos formativos"
}
```

## Notas

- Es post-procesamiento, no regeneracion — el LLM genera con terminologia standard y luego se adapta
- Los reemplazos se aplican tanto a GenerateText como a GenerateJSON (sobre los campos string del output)
- Para GenerateJSON, el procesador se aplica sobre los valores string del JSON, no sobre las keys
- Si una org no tiene overrides, el procesador es un no-op
- El orden de los reemplazos puede importar si hay solapamiento — documentar esta limitacion
