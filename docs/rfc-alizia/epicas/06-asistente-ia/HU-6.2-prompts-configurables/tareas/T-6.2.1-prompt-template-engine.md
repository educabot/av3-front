# T-6.2.1: Prompt template engine

**Historia:** HU-6.2 — Prompts configurables
**Tipo:** Backend
**Estado:** ⬜ Pendiente

---

## Descripcion

Implementar un motor de templates de prompts que cargue templates desde la configuracion de la organizacion, reemplace placeholders `{variable}` con valores en runtime, y provea la estructura de configuracion para los prompts de IA.

## Implementacion

```go
// src/core/usecases/ai/prompt_engine.go
package ai

import (
    "encoding/json"
    "fmt"
    "regexp"
    "strings"
)

var placeholderRe = regexp.MustCompile(`\{(\w+)\}`)

// RenderPrompt reemplaza todos los placeholders {variable} en el template
// con valores del map. Retorna error si algun placeholder no tiene valor.
func RenderPrompt(template string, values map[string]string) (string, error) {
    var missing []string

    result := placeholderRe.ReplaceAllStringFunc(template, func(match string) string {
        key := match[1 : len(match)-1] // strip { }
        if val, ok := values[key]; ok {
            return val
        }
        missing = append(missing, key)
        return match
    })

    if len(missing) > 0 {
        return "", fmt.Errorf("missing placeholder values: %s", strings.Join(missing, ", "))
    }

    return result, nil
}

// PromptConfig representa un template de prompt desde la config de la org.
type PromptConfig struct {
    Key          string          `json:"key"`
    AIPrompt     string          `json:"ai_prompt"`
    OutputSchema json.RawMessage `json:"output_schema,omitempty"` // JSON Schema
}
```

### Estructura en `organizations.config`

```jsonc
{
  "ai_prompts": {
    "coord_doc_sections": [
      {
        "key": "problem_edge",
        "ai_prompt": "Genera un eje problematico que integre las categorias: {categories}. Area: {area_name}. Periodo: {period}.",
        "output_schema": null  // texto libre, sin schema
      }
    ],
    "class_plan": {
      "ai_prompt": "Genera un plan de {class_count} clases para {subject_name}. Topics: {topics}. Estrategia: {strategy}.",
      "output_schema": { "type": "array", "items": { "type": "object", "properties": { "class_number": { "type": "integer" }, "title": { "type": "string" }, "category_ids": { "type": "array", "items": { "type": "integer" } } }, "required": ["class_number", "title", "category_ids"] } }
    },
    "lesson_proposal": {
      "ai_prompt": "Genera una propuesta detallada para la clase '{class_title}'. Objetivo: {objective}. Actividades: {activities}. Fuente: {source}.",
      "output_schema": null
    }
  }
}
```

## Comportamiento

1. Se carga la configuracion de prompts desde `organizations.config` al iniciar
2. Al generar contenido, se busca el `PromptConfig` por `key`
3. Se llama a `RenderPrompt` con el template y los valores de runtime
4. Si faltan placeholders → se retorna error inmediato (no se envia al LLM un prompt incompleto)
5. Si hay `output_schema` → se pasa al AI Provider para structured output (via `GenerateJSON`)

## Notas

- Los templates se almacenan en la config de la org, no hardcodeados en el codigo
- Cada organizacion puede personalizar los prompts segun sus necesidades pedagogicas
- El campo `output_schema` es usado por `GenerateJSON` (HU-6.1) para forzar structured output
- Los prompts de `coord_doc_sections` (existentes desde HU-4.3) ya usan el campo `ai_prompt` — este sistema lo formaliza
- La regex `\{(\w+)\}` solo matchea caracteres alfanumericos y underscore dentro de las llaves
- Templates soportan multi-linea con formato markdown (el contenido del string puede incluir newlines)
