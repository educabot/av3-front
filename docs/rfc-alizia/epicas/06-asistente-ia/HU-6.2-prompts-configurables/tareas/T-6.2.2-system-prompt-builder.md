# T-6.2.2: System prompt builder

**Historia:** HU-6.2 — Prompts configurables
**Tipo:** Backend
**Estado:** ⬜ Pendiente

---

## Descripcion

Implementar un builder de system prompts que construya prompts con una estructura consistente: definicion de rol, secciones de contexto, instruccion especifica y restricciones. Esto garantiza que todos los system prompts enviados al LLM sigan el mismo formato.

## Implementacion

```go
// src/core/usecases/ai/system_prompt.go
package ai

import (
    "fmt"
    "strings"
)

// SystemPromptBuilder construye system prompts con estructura consistente:
// 1. Definicion de rol (quien es Alizia)
// 2. Contexto (datos del documento/plan)
// 3. Instruccion especifica (que generar)
// 4. Restricciones (limites y reglas)
type SystemPromptBuilder struct {
    role        string
    context     []string
    instruction string
    constraints []string
}

func NewSystemPromptBuilder() *SystemPromptBuilder {
    return &SystemPromptBuilder{
        role: defaultRole,
    }
}

const defaultRole = `Sos Alizia, una asistente pedagogica especializada en planificacion educativa.
Respondes en espanol rioplatense. Sos clara, directa y fundamentas tus propuestas en criterios pedagogicos.
No inventas informacion que no tengas disponible.`

func (b *SystemPromptBuilder) WithRole(role string) *SystemPromptBuilder {
    b.role = role
    return b
}

func (b *SystemPromptBuilder) AddContext(label, value string) *SystemPromptBuilder {
    b.context = append(b.context, fmt.Sprintf("## %s\n%s", label, value))
    return b
}

func (b *SystemPromptBuilder) WithInstruction(instruction string) *SystemPromptBuilder {
    b.instruction = instruction
    return b
}

func (b *SystemPromptBuilder) AddConstraint(constraint string) *SystemPromptBuilder {
    b.constraints = append(b.constraints, constraint)
    return b
}

func (b *SystemPromptBuilder) Build() string {
    var parts []string

    parts = append(parts, b.role)

    if len(b.context) > 0 {
        parts = append(parts, "\n# Contexto\n"+strings.Join(b.context, "\n\n"))
    }

    if b.instruction != "" {
        parts = append(parts, "\n# Instruccion\n"+b.instruction)
    }

    if len(b.constraints) > 0 {
        parts = append(parts, "\n# Restricciones\n- "+strings.Join(b.constraints, "\n- "))
    }

    return strings.Join(parts, "\n")
}
```

## Comportamiento

1. Se crea un builder con `NewSystemPromptBuilder()` — incluye el rol default de Alizia
2. Opcionalmente se sobreescribe el rol con `WithRole()` (para organizaciones con tono custom)
3. Se agregan secciones de contexto con `AddContext(label, value)` — cada una con un heading `##`
4. Se define la instruccion especifica con `WithInstruction()`
5. Se agregan restricciones con `AddConstraint()` — se listan como bullet points
6. `Build()` concatena todo en un string con estructura clara para el LLM

### Ejemplo de uso

```go
prompt := NewSystemPromptBuilder().
    AddContext("Documento", "Titulo: Plan Anual Matematicas\nArea: Ciencias Exactas").
    AddContext("Disciplinas", "Matematicas (20 clases), Fisica (15 clases)").
    WithInstruction("Genera una estrategia metodologica que integre ambas disciplinas.").
    AddConstraint("Maximo 500 palabras").
    AddConstraint("Incluir al menos 2 referencias pedagogicas").
    Build()
```

Resultado:

```
Sos Alizia, una asistente pedagogica especializada en planificacion educativa.
Respondes en espanol rioplatense. Sos clara, directa y fundamentas tus propuestas en criterios pedagogicos.
No inventas informacion que no tengas disponible.

# Contexto
## Documento
Titulo: Plan Anual Matematicas
Area: Ciencias Exactas

## Disciplinas
Matematicas (20 clases), Fisica (15 clases)

# Instruccion
Genera una estrategia metodologica que integre ambas disciplinas.

# Restricciones
- Maximo 500 palabras
- Incluir al menos 2 referencias pedagogicas
```

## Notas

- El builder garantiza que TODOS los system prompts sigan la misma estructura
- El rol default es Alizia en espanol rioplatense — puede sobreescribirse por org (post-MVP, HU-6.5)
- Las secciones de contexto se etiquetan (ej: "Documento", "Clase", "Actividades") para claridad del LLM
- Las restricciones se formatean como bullet points para que el LLM las respete mejor
- Consumido por: HU-4.3 (generacion de secciones), HU-4.4 (plan de clases), HU-4.6 (chat), HU-5.4 (propuesta de clase)
- Patron fluent (method chaining) para API ergonomica
