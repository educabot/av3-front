# T-6.1.1: Definicion de interfaz del provider

**Historia:** HU-6.1 — AI Provider
**Tipo:** Backend
**Estado:** ⬜ Pendiente

---

## Descripcion

Definir la interfaz `AIProvider` y los tipos asociados en la capa core. Esta interfaz es el contrato que consumen los usecases de las Epicas 4, 5 y 8 para interactuar con el modelo de IA sin acoplarse a la implementacion concreta.

## Implementacion

```go
// src/core/providers/ai_provider.go
package providers

import "context"

// AIProvider abstrae la interaccion con un modelo de lenguaje.
// Todas las features de IA consumen esta interfaz.
type AIProvider interface {
    // GenerateText envia un prompt y retorna texto libre.
    GenerateText(ctx context.Context, systemPrompt, userPrompt string) (string, error)

    // GenerateJSON envia un prompt con un JSON schema y retorna structured output parseado.
    GenerateJSON(ctx context.Context, systemPrompt, userPrompt string, schema interface{}) ([]byte, error)

    // ChatWithTools envia mensajes con definiciones de tools y maneja el loop de tool calls.
    ChatWithTools(ctx context.Context, req ChatWithToolsRequest) (*ChatWithToolsResponse, error)
}

// ChatMessage representa un mensaje en una conversacion multi-turn.
type ChatMessage struct {
    Role       string     `json:"role"` // system, user, assistant, tool
    Content    string     `json:"content"`
    ToolCallID string     `json:"tool_call_id,omitempty"`
    ToolCalls  []ToolCall `json:"tool_calls,omitempty"`
}

// ToolCall representa una llamada a tool solicitada por el modelo.
type ToolCall struct {
    ID       string `json:"id"`
    Function struct {
        Name      string `json:"name"`
        Arguments string `json:"arguments"` // JSON string
    } `json:"function"`
}

// ToolDefinition define una tool disponible para el modelo.
type ToolDefinition struct {
    Name        string      `json:"name"`
    Description string      `json:"description"`
    Parameters  interface{} `json:"parameters"` // JSON Schema
}

// ChatWithToolsRequest contiene todo lo necesario para una conversacion con tools.
type ChatWithToolsRequest struct {
    Messages    []ChatMessage
    Tools       []ToolDefinition
    ToolHandler func(name string, args string) (string, error) // ejecuta una tool call
}

// ChatWithToolsResponse contiene el resultado final de la conversacion.
type ChatWithToolsResponse struct {
    Message    string   // mensaje final del asistente
    ToolsUsed  []string // nombres de tools que se ejecutaron
    TokensUsed int      // tokens totales consumidos
}
```

## Comportamiento

1. `GenerateText` — Envia system + user prompt, retorna la respuesta como string. Para generacion de estrategias metodologicas, titulos, etc.
2. `GenerateJSON` — Igual que `GenerateText` pero usa `response_format: json_schema` para garantizar structured output. El parametro `schema` define la estructura esperada.
3. `ChatWithTools` — Maneja el loop completo de function calling:
   - Envia los mensajes al modelo con las definiciones de tools
   - Si el modelo responde con `tool_calls`, ejecuta cada una via `ToolHandler`
   - Agrega los resultados como mensajes `tool` y reenvia
   - Repite hasta obtener un mensaje final sin tool calls (o alcanzar el limite de iteraciones)

## Notas

- La interfaz vive en `src/core/providers/` siguiendo Clean Architecture — la capa core no depende de infraestructura
- `ToolHandler` es un callback que el consumidor provee para ejecutar las tool calls. El provider no conoce la logica de negocio de cada tool
- El campo `schema` en `GenerateJSON` acepta `interface{}` para flexibilidad — cada usecase define su propio schema
- Los tipos `ChatMessage`, `ToolCall`, etc. son DTOs del provider, no entities del dominio
