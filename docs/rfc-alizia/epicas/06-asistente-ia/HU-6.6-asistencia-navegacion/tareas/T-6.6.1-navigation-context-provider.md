# T-6.6.1: Navigation context provider

**Historia:** HU-6.6 — Asistencia de navegacion
**Tipo:** Backend
**Estado:** ⬜ Pendiente
**Fase:** Post-MVP

---

## Descripcion

Implementar un endpoint y servicio que reciba metadata de la pantalla actual del usuario (ruta, entidad visible, estado) y construya un contexto para que Alizia pueda dar respuestas relevantes a la ubicacion del usuario en la plataforma.

## Implementacion

### Endpoint

```
POST /alizia/navigation-chat
```

```json
{
    "message": "¿Qué es un núcleo problemático?",
    "navigation_context": {
        "route": "/courses/3/documents/new/step-1",
        "screen": "wizard_step_1",
        "entity_type": "coordination_document",
        "entity_id": null,
        "visible_data": {
            "course_id": 3,
            "area_id": 1
        }
    },
    "history": []
}
```

### Context builder

```go
// src/core/usecases/ai/navigation_context_builder.go
package ai

// NavigationContext contiene la metadata de la pantalla actual del usuario.
type NavigationContext struct {
    Route       string            `json:"route"`
    Screen      string            `json:"screen"`
    EntityType  string            `json:"entity_type"`
    EntityID    *int              `json:"entity_id"`
    VisibleData map[string]any    `json:"visible_data"`
}

// ScreenHint mapea pantallas a instrucciones contextuales para el system prompt.
var ScreenHints = map[string]string{
    "wizard_step_1":     "El usuario esta seleccionando nucleos problematicos y categorias para un nuevo documento de coordinacion.",
    "wizard_step_2":     "El usuario esta definiendo fechas y cantidad de clases por disciplina.",
    "wizard_step_3":     "El usuario esta asignando categorias a cada disciplina.",
    "document_view":     "El usuario esta viendo/editando un documento de coordinacion existente.",
    "course_detail":     "El usuario esta viendo los detalles de un curso: alumnos, horarios y documentos.",
    "lesson_plan":       "El usuario esta trabajando en la planificacion de clases de una disciplina.",
    "home":              "El usuario esta en la pantalla principal viendo sus cursos asignados.",
}

// BuildNavigationPrompt construye la seccion de contexto de navegacion para el system prompt.
func BuildNavigationPrompt(ctx NavigationContext) string {
    hint, ok := ScreenHints[ctx.Screen]
    if !ok {
        hint = "El usuario esta navegando la plataforma."
    }

    return fmt.Sprintf(
        "## Contexto de navegacion\n%s\nRuta actual: %s\n",
        hint,
        ctx.Route,
    )
}
```

## Comportamiento

1. El frontend envia la metadata de la pantalla actual junto con el mensaje del usuario
2. El backend construye un system prompt que incluye el contexto de navegacion
3. Alizia responde con conocimiento de donde esta el usuario y que datos tiene visibles
4. El historial de navegacion es independiente del historial de chat de documentos

## Notas

- `ScreenHints` es un mapa estatico — se extiende agregando pantallas, no requiere configuracion por org
- `visible_data` permite enriquecer el contexto sin que el backend tenga que re-consultar los datos
- Para explicaciones de conceptos pedagogicos, el builder incluye las fuentes curriculares de la org (HU-6.5) si estan configuradas
- Este endpoint puede usar un modelo mas barato que el de generacion de documentos (decision de infra)
