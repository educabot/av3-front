# T-6.6.4: Tests de asistencia de navegacion

**Historia:** HU-6.6 — Asistencia de navegacion
**Tipo:** Testing
**Estado:** ⬜ Pendiente
**Fase:** Post-MVP

---

## Descripcion

Tests unitarios y de integracion para validar el sistema de asistencia de navegacion: context provider, triggers proactivos y widget global.

## Casos de test

### Navigation context builder

| Test | Input | Expected |
|------|-------|----------|
| Screen conocido | `screen: "wizard_step_1"` | Hint de wizard step 1 en el prompt |
| Screen desconocido | `screen: "unknown_page"` | Hint generico "navegando la plataforma" |
| Context con visible_data | `visible_data: { course_id: 3 }` | Datos incluidos en el contexto |
| Context sin entity | `entity_id: null` | Prompt valido sin referencia a entidad |

### Endpoint navigation-chat

| Test | Input | Expected |
|------|-------|----------|
| Mensaje con contexto valido | Mensaje + navigation_context completo | 200, respuesta contextual |
| Sin navigation_context | Mensaje solo | 200, respuesta generica (funciona sin contexto) |
| Con historial | Mensaje + history de 3 turnos | 200, respuesta que considera historial |
| Feature deshabilitada | Org sin `"recommendations"` | 403 Forbidden |

### Proactive triggers

| Test | Input | Expected |
|------|-------|----------|
| First visit trigger | Screen no visitada antes | Genera mensaje proactivo |
| First visit ya visitada | Screen con flag en localStorage | No genera trigger |
| Inactivity en wizard | 61s sin interaccion en wizard_step_2 | Genera mensaje proactivo |
| Inactivity en home | 61s sin interaccion en home | No genera trigger (no es wizard) |
| Inactivity con cooldown activo | Trigger hace <5min | No genera trigger |
| Empty screen | Pantalla plannable sin contenido | Genera mensaje proactivo |
| Published doc | Documento publicado recien abierto | Genera mensaje con resumen |

### Endpoint proactive-message

| Test | Input | Expected |
|------|-------|----------|
| Trigger valido | `trigger: "first_visit"` + context | 200, mensaje + actions |
| Trigger invalido | `trigger: "nonexistent"` | 400 Bad Request |
| Respuesta dismissable | Cualquier trigger | `dismissable: true` siempre |

## Notas

- Los tests de triggers se ejecutan en el frontend (unit tests de la logica de evaluacion)
- Los tests de endpoints son de integracion con el AI Provider mockeado
- Verificar que el historial de navegacion se mantiene separado del historial de documentos
- Los tests de cooldown requieren mock del reloj (time-based logic)
