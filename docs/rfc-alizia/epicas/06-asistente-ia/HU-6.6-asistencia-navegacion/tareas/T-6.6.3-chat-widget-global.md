# T-6.6.3: Widget de chat global

**Historia:** HU-6.6 — Asistencia de navegacion
**Tipo:** Frontend
**Estado:** ⬜ Pendiente
**Fase:** Post-MVP

---

## Descripcion

Implementar un widget de chat flotante accesible desde todas las pantallas de la plataforma. A diferencia del chat de documentos (que vive dentro de la vista de edicion), este widget es global, liviano y context-aware.

## Implementacion

### Componente

El widget se renderiza como un boton flotante (bottom-right) que se expande a un panel de chat al hacer clic.

```javascript
// Widget de chat global — se monta una sola vez en el layout principal

const NavigationChatWidget = {
    isOpen: false,
    isMinimized: false,
    history: [],

    // Se actualiza en cada cambio de ruta
    currentContext: {
        route: window.location.pathname,
        screen: detectCurrentScreen(),
        entity_type: null,
        entity_id: null,
        visible_data: {}
    },

    async sendMessage(message) {
        const response = await fetch('/alizia/navigation-chat', {
            method: 'POST',
            body: JSON.stringify({
                message,
                navigation_context: this.currentContext,
                history: this.history
            })
        });
        // ...
    },

    // Recibe mensajes proactivos del sistema de triggers
    showProactiveMessage(message, actions) {
        this.isOpen = true;
        this.addSystemMessage(message, actions);
    },

    // Preferencia de minimizado se persiste en localStorage
    toggleMinimize() {
        this.isMinimized = !this.isMinimized;
        localStorage.setItem('alizia_widget_minimized', this.isMinimized);
    }
};
```

### Estados del widget

| Estado | Visual | Interaccion |
|--------|--------|-------------|
| Cerrado | Boton flotante con icono de Alizia | Click abre el panel |
| Abierto | Panel de chat expandido | Input de texto + historial |
| Minimizado | Boton flotante con badge de mensajes | Click re-abre |
| Proactivo | Panel abierto con mensaje + acciones | Usuario elige accion o cierra |


## Comportamiento

1. El widget se monta en el layout principal y persiste entre navegaciones
2. El `currentContext` se actualiza automaticamente en cada cambio de ruta
3. El historial de navegacion es independiente del historial de chat de documentos
4. Si el usuario minimiza el widget, la preferencia se persiste en localStorage
5. Los mensajes proactivos abren el widget automaticamente (pero el usuario puede cerrar)

## Notas

- El widget NO debe interferir con el chat existente de documentos — son dos chats separados con contextos distintos
- Las respuestas de navegacion deben ser cortas y accionables (no explicaciones extensas)
- El widget se oculta si la feature `recommendations` esta deshabilitada para la org (HU-6.5)
- Considerar un modo "solo reactivo" donde el widget existe pero no muestra proactivos
- El historial de navegacion se limpia al cerrar sesion
