# T-5.6.1: Modelo de datos de bitácora

**Historia:** HU-5.6 — Bitácora de cotejo
**Tipo:** Backend / DB
**Estado:** ⬜ Pendiente
**Fase:** Post-MVP

---

## Descripción

Crear la tabla `lesson_log_entries` que almacena el registro post-clase del docente, vinculado al lesson plan correspondiente. Incluye el texto original (transcrito o escrito), los datos estructurados que extrae Alizia, y metadata del registro.

## Implementación

### Migración

```sql
CREATE TYPE log_entry_input_type AS ENUM ('text', 'audio');
CREATE TYPE objective_fulfillment AS ENUM ('yes', 'partial', 'no');

CREATE TABLE lesson_log_entries (
    id SERIAL PRIMARY KEY,
    organization_id INTEGER NOT NULL REFERENCES organizations(id),
    teacher_lesson_plan_id INTEGER NOT NULL REFERENCES teacher_lesson_plans(id),
    input_type log_entry_input_type NOT NULL DEFAULT 'text',

    -- Contenido original
    raw_text TEXT NOT NULL,                    -- texto escrito o transcripción del audio
    audio_url TEXT,                            -- URL del audio original (si aplica)

    -- Datos estructurados (extraídos por IA o completados manualmente)
    objective_fulfilled objective_fulfillment,  -- ¿se cumplió el objetivo?
    activities_completed BOOLEAN,               -- ¿se completaron todas las actividades?
    actual_duration_minutes INTEGER,            -- tiempo real de la clase
    adjustments_suggested TEXT,                 -- ajustes sugeridos por el docente
    student_observations JSONB DEFAULT '[]',    -- observaciones sobre alumnos específicos

    -- Conversación con Alizia (preguntas de seguimiento)
    chat_history JSONB DEFAULT '[]',

    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW(),

    CONSTRAINT unique_log_per_plan UNIQUE (teacher_lesson_plan_id)
);

CREATE INDEX idx_log_entries_org ON lesson_log_entries(organization_id);
CREATE INDEX idx_log_entries_plan ON lesson_log_entries(teacher_lesson_plan_id);
```

### Estructura de student_observations

```json
[
    {
        "student_id": 15,
        "observation": "Tuvo dificultad con las ecuaciones de primer grado",
        "follow_up": true
    }
]
```

### Entity Go

```go
// src/core/entities/lesson_log_entry.go
package entities

import "time"

type LogEntryInputType string
type ObjectiveFulfillment string

const (
    LogInputText  LogEntryInputType = "text"
    LogInputAudio LogEntryInputType = "audio"

    ObjectiveYes     ObjectiveFulfillment = "yes"
    ObjectivePartial ObjectiveFulfillment = "partial"
    ObjectiveNo      ObjectiveFulfillment = "no"
)

type LessonLogEntry struct {
    ID                    int                   `json:"id" gorm:"primaryKey"`
    OrganizationID        int                   `json:"organization_id"`
    TeacherLessonPlanID   int                   `json:"teacher_lesson_plan_id"`
    InputType             LogEntryInputType     `json:"input_type"`
    RawText               string                `json:"raw_text"`
    AudioURL              *string               `json:"audio_url,omitempty"`
    ObjectiveFulfilled    *ObjectiveFulfillment `json:"objective_fulfilled,omitempty"`
    ActivitiesCompleted   *bool                 `json:"activities_completed,omitempty"`
    ActualDurationMinutes *int                  `json:"actual_duration_minutes,omitempty"`
    AdjustmentsSuggested  *string               `json:"adjustments_suggested,omitempty"`
    StudentObservations   JSON                  `json:"student_observations" gorm:"type:jsonb;default:'[]'"`
    ChatHistory           JSON                  `json:"chat_history" gorm:"type:jsonb;default:'[]'"`
    CreatedAt             time.Time             `json:"created_at"`
    UpdatedAt             time.Time             `json:"updated_at"`

    // Relations
    TeacherLessonPlan TeacherLessonPlan `json:"-" gorm:"foreignKey:TeacherLessonPlanID"`
}
```

## Notas

- Una sola bitácora por lesson plan (constraint unique)
- Los datos estructurados son nullable porque se pueden completar progresivamente (primero texto libre, luego Alizia extrae)
- `student_observations` es JSONB para flexibilidad — cada observación referencia un student_id
- `chat_history` almacena la conversación de seguimiento de Alizia (preguntas y respuestas)
- El `audio_url` apunta a un storage externo (S3/Azure Blob) — la transcripción se guarda en `raw_text`
