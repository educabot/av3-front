# T-5.6.3: Procesamiento estructurado y seguimiento

**Historia:** HU-5.6 — Bitácora de cotejo
**Tipo:** Backend
**Estado:** ⬜ Pendiente
**Fase:** Post-MVP

---

## Descripción

Implementar el servicio que toma el texto libre de la bitácora (escrito o transcrito) y usa el AI Provider para extraer datos estructurados y generar preguntas de seguimiento. Alizia analiza el texto, extrae lo que puede, y pregunta por lo que falta.

## Implementación

### Flujo de procesamiento

```
1. Docente envía texto (o se completa transcripción)
2. IA analiza el texto y extrae datos estructurados
3. IA detecta información faltante
4. Genera preguntas de seguimiento
5. Docente responde → IA actualiza datos estructurados
6. Repite hasta completar o docente cierra
```

### Prompt de extracción

```
Sos Alizia. Analizá el siguiente relato de un docente sobre cómo fue su clase.

Clase planificada:
- Título: {title}
- Objetivo: {objective}
- Actividades: {activities}
- Duración planificada: {planned_duration} minutos

Relato del docente:
"{raw_text}"

Extraé la siguiente información del relato. Si algo no se menciona, poné null:

1. objective_fulfilled: "yes" | "partial" | "no"
2. activities_completed: true | false
3. actual_duration_minutes: número
4. adjustments_suggested: texto libre
5. student_observations: array de { student_name, observation, follow_up: bool }

Además, generá hasta 2 preguntas de seguimiento sobre información relevante que falte.
```

### Schema de respuesta IA

```go
type LogExtractionResult struct {
    ObjectiveFulfilled    *string              `json:"objective_fulfilled"`
    ActivitiesCompleted   *bool                `json:"activities_completed"`
    ActualDurationMinutes *int                 `json:"actual_duration_minutes"`
    AdjustmentsSuggested  *string              `json:"adjustments_suggested"`
    StudentObservations   []StudentObservation `json:"student_observations"`
    FollowUpQuestions     []string             `json:"follow_up_questions"`
}

type StudentObservation struct {
    StudentName string `json:"student_name"`
    Observation string `json:"observation"`
    FollowUp    bool   `json:"follow_up"`
}
```

### Endpoint de chat de seguimiento

```
POST /lesson-plans/{id}/log-entry/chat
```

```json
{
    "message": "No, no llegamos al cierre. Nos quedamos cortos de tiempo.",
    "history": [...]
}
```

Alizia procesa la respuesta, actualiza los datos estructurados, y puede hacer otra pregunta o cerrar.

## Comportamiento

1. El procesamiento se dispara automáticamente al crear/actualizar el `raw_text`
2. Los datos extraídos se guardan en los campos de `lesson_log_entries`
3. Las preguntas de seguimiento se presentan como chat
4. Cada respuesta del docente se procesa y puede actualizar los datos
5. Máximo 3 rondas de preguntas para no agotar al docente
6. El docente puede cerrar el chat en cualquier momento

## Notas

- Usa `GenerateJSON` del AI Provider (HU-6.1) para la extracción estructurada
- Las preguntas de seguimiento usan `ChatWithTools` si necesitan actualizar datos
- Los `student_observations` referencian por nombre — el frontend resuelve el student_id al vincular
- El límite de 3 rondas es configurable — si el docente contesta rápido puede ser útil, si no, molesto
- La información extraída se puede corregir manualmente por el docente
