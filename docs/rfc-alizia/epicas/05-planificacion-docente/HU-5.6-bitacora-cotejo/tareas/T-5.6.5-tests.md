# T-5.6.5: Tests de bitácora de cotejo

**Historia:** HU-5.6 — Bitácora de cotejo
**Tipo:** Testing
**Estado:** ⬜ Pendiente
**Fase:** Post-MVP

---

## Descripción

Tests unitarios y de integración para el flujo completo de bitácora: modelo de datos, transcripción, procesamiento IA, y endpoints.

## Casos de test

### Modelo de datos

| Test | Input | Expected |
|------|-------|----------|
| Crear bitácora texto | raw_text válido | Registro creado con input_type "text" |
| Crear bitácora audio | audio_url + raw_text | Registro con input_type "audio" y URL |
| Unique constraint | Dos bitácoras para el mismo plan | Error: constraint violation |
| Campos nullable | Solo raw_text, sin datos estructurados | OK, campos null permitidos |

### Transcripción

| Test | Input | Expected |
|------|-------|----------|
| Audio válido mp3 | Archivo mp3 < 25MB | Transcripción en español |
| Audio válido m4a | Archivo m4a (formato iPhone) | Transcripción correcta |
| Audio demasiado grande | Archivo > 25MB | Error 413 |
| Formato no soportado | Archivo .ogg | Error 400 |

### Procesamiento estructurado

| Test | Input | Expected |
|------|-------|----------|
| Texto con toda la info | "Cumplimos el objetivo, hicimos todo en 45min" | Extrae: yes, true, 45 |
| Texto parcial | "No llegamos al cierre" | Extrae: partial, false; genera preguntas |
| Texto ambiguo | "Fue una clase interesante" | Extrae poco; genera preguntas específicas |
| Seguimiento con respuesta | Pregunta + respuesta del docente | Actualiza datos estructurados |

### Endpoints

| Test | Input | Expected |
|------|-------|----------|
| POST log-entry como docente dueño | Docente del plan | 201 Created |
| POST log-entry como otro docente | Docente de otra disciplina | 403 Forbidden |
| GET log-entry como coordinador | Coordinador del área | 200 con datos |
| GET log-entries por área | Coordinador + rango de fechas | Lista filtrada |
| PATCH datos estructurados | Corrección manual del docente | 200, datos actualizados |
| POST chat seguimiento | Mensaje de respuesta | 200, datos actualizados + posible nueva pregunta |

## Notas

- Tests de transcripción requieren mock del servicio Whisper
- Tests de procesamiento requieren mock del AI Provider
- Verificar que `student_observations` se parsea correctamente como JSONB
- Verificar que el coordinador NO puede ver bitácoras de otra área
