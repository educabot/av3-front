# T-5.6.2: Transcripción de audio

**Historia:** HU-5.6 — Bitácora de cotejo
**Tipo:** Backend
**Estado:** ⬜ Pendiente
**Fase:** Post-MVP

---

## Descripción

Implementar el servicio de transcripción de audio usando Azure OpenAI (Whisper) para convertir la grabación del docente en texto. El audio se sube, se transcribe, y el texto resultante se guarda como `raw_text` de la bitácora.

## Implementación

### Endpoint de upload + transcripción

```
POST /lesson-plans/{id}/log-entry/audio
Content-Type: multipart/form-data
```

Body: archivo de audio (mp3, m4a, wav, webm)

Respuesta:
```json
{
    "id": 1,
    "raw_text": "Hoy hicimos la actividad de grupo pero no llegamos al cierre porque...",
    "input_type": "audio",
    "audio_url": "https://storage.blob.core.windows.net/logs/abc123.m4a"
}
```

### Flujo

```
1. Docente graba audio en el celular
2. Frontend sube el archivo al endpoint
3. Backend almacena el audio en Azure Blob Storage
4. Backend envía el audio a Azure OpenAI Whisper para transcripción
5. Transcripción se guarda como raw_text en lesson_log_entries
6. Se dispara el procesamiento de datos estructurados (T-5.6.3)
```

### Servicio de transcripción

```go
// src/core/providers/audio_transcriber.go
package providers

import "context"

// AudioTranscriber abstrae el servicio de transcripción.
type AudioTranscriber interface {
    Transcribe(ctx context.Context, audioData []byte, format string) (string, error)
}
```

```go
// src/infrastructure/azure/whisper_transcriber.go
package azure

type WhisperTranscriber struct {
    client   *openai.Client
    model    string // "whisper-1"
}

func (t *WhisperTranscriber) Transcribe(ctx context.Context, audioData []byte, format string) (string, error) {
    resp, err := t.client.CreateTranscription(ctx, openai.AudioRequest{
        Model:    t.model,
        Reader:   bytes.NewReader(audioData),
        Format:   openai.AudioResponseFormatText,
        Language: "es",
    })
    if err != nil {
        return "", err
    }
    return resp.Text, nil
}
```

## Notas

- Formatos soportados: mp3, m4a, wav, webm (los más comunes en grabación móvil)
- Límite de tamaño: 25MB (límite de Whisper API)
- Se transcribe siempre en español (`Language: "es"`)
- El audio original se conserva por si se necesita re-transcribir
- La transcripción es asíncrona desde la perspectiva del usuario — puede tardar unos segundos
- Considerar un estado intermedio "transcribing" si el audio es largo
