# T-5.7.2: Modelo de datos de sugerencias

**Historia:** HU-5.7 — Repropuesta
**Tipo:** Backend / DB
**Estado:** ⬜ Pendiente
**Fase:** Post-MVP

---

## Descripción

Crear la tabla `repropuesta_suggestions` para almacenar las sugerencias generadas por Alizia a partir del análisis de bitácoras. Cada sugerencia es un cambio propuesto a una clase futura que el docente puede aceptar, rechazar o modificar.

## Implementación

### Migración

```sql
CREATE TYPE suggestion_status AS ENUM ('pending', 'accepted', 'rejected', 'modified');
CREATE TYPE suggestion_type AS ENUM ('change_activity', 'reinforce_topic', 'adjust_time', 'add_resource');

CREATE TABLE repropuesta_suggestions (
    id SERIAL PRIMARY KEY,
    organization_id INTEGER NOT NULL REFERENCES organizations(id),
    source_log_entry_id INTEGER NOT NULL REFERENCES lesson_log_entries(id),
    target_lesson_plan_id INTEGER NOT NULL REFERENCES teacher_lesson_plans(id),
    type suggestion_type NOT NULL,
    description TEXT NOT NULL,
    current_value TEXT NOT NULL,
    suggested_value TEXT NOT NULL,
    reason TEXT NOT NULL,
    priority VARCHAR(10) NOT NULL DEFAULT 'medium',
    status suggestion_status NOT NULL DEFAULT 'pending',
    resolved_at TIMESTAMP,
    resolved_value TEXT,           -- valor final si fue modificado
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_suggestions_target ON repropuesta_suggestions(target_lesson_plan_id);
CREATE INDEX idx_suggestions_status ON repropuesta_suggestions(status);
CREATE INDEX idx_suggestions_source ON repropuesta_suggestions(source_log_entry_id);
```

### Entity Go

```go
// src/core/entities/repropuesta_suggestion.go
package entities

import "time"

type SuggestionStatus string
type SuggestionType string

const (
    SuggestionPending  SuggestionStatus = "pending"
    SuggestionAccepted SuggestionStatus = "accepted"
    SuggestionRejected SuggestionStatus = "rejected"
    SuggestionModified SuggestionStatus = "modified"

    SugChangeActivity  SuggestionType = "change_activity"
    SugReinforceTopic  SuggestionType = "reinforce_topic"
    SugAdjustTime      SuggestionType = "adjust_time"
    SugAddResource     SuggestionType = "add_resource"
)

type RepropuestaSuggestion struct {
    ID                  int              `json:"id" gorm:"primaryKey"`
    OrganizationID      int              `json:"organization_id"`
    SourceLogEntryID    int              `json:"source_log_entry_id"`
    TargetLessonPlanID  int              `json:"target_lesson_plan_id"`
    Type                SuggestionType   `json:"type"`
    Description         string           `json:"description"`
    CurrentValue        string           `json:"current_value"`
    SuggestedValue      string           `json:"suggested_value"`
    Reason              string           `json:"reason"`
    Priority            string           `json:"priority"`
    Status              SuggestionStatus `json:"status"`
    ResolvedAt          *time.Time       `json:"resolved_at,omitempty"`
    ResolvedValue       *string          `json:"resolved_value,omitempty"`
    CreatedAt           time.Time        `json:"created_at"`
    UpdatedAt           time.Time        `json:"updated_at"`

    // Relations
    SourceLogEntry    LessonLogEntry    `json:"-" gorm:"foreignKey:SourceLogEntryID"`
    TargetLessonPlan  TeacherLessonPlan `json:"-" gorm:"foreignKey:TargetLessonPlanID"`
}
```

## Notas

- `source_log_entry_id` vincula la sugerencia a la bitácora que la originó (trazabilidad)
- `target_lesson_plan_id` es la clase futura que se propone modificar
- `current_value` y `suggested_value` son texto libre — el frontend los muestra como diff
- `resolved_value` se usa solo cuando el status es `modified` (el docente aceptó con cambios)
- Una bitácora puede generar 0 o N sugerencias (una por cada clase afectada)
- Las sugerencias solo aplican a planes en estado `pending` o `in_progress`
