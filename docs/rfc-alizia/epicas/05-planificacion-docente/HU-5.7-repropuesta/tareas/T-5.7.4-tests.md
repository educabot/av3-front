# T-5.7.4: Tests de repropuesta

**Historia:** HU-5.7 — Repropuesta
**Tipo:** Testing
**Estado:** ⬜ Pendiente
**Fase:** Post-MVP

---

## Descripción

Tests unitarios y de integración para el flujo completo de repropuesta: análisis de impacto, modelo de sugerencias, endpoints y aplicación de cambios.

## Casos de test

### Análisis de impacto

| Test | Input | Expected |
|------|-------|----------|
| Bitácora con impacto claro | objective: "partial", adjustments no vacío | has_impact: true, suggestions > 0 |
| Bitácora sin impacto | objective: "yes", todo completado | has_impact: false, suggestions vacío |
| Sin clases futuras | Última clase del curso dictada | Análisis se omite |
| Solo clases pending/in_progress | Mix de estados en clases futuras | Solo genera sugerencias para pending/in_progress |

### Modelo de sugerencias

| Test | Input | Expected |
|------|-------|----------|
| Crear sugerencia | Datos válidos de análisis | Registro creado con status "pending" |
| FK a log entry | source_log_entry_id válido | Relación correcta |
| FK a lesson plan | target_lesson_plan_id válido | Relación correcta |
| Múltiples sugerencias por bitácora | Análisis con 3 sugerencias | 3 registros creados |

### Endpoints

| Test | Input | Expected |
|------|-------|----------|
| GET sugerencias pendientes | Docente con sugerencias | 200, lista filtrada |
| GET sugerencias como otro docente | Docente de otra disciplina | 403 Forbidden |
| PATCH aceptar | status: "accepted" | 200, status actualizado, resolved_at set |
| PATCH rechazar | status: "rejected" | 200, status actualizado |
| PATCH modificar sin valor | status: "modified", sin resolved_value | 400 Bad Request |
| PATCH modificar con valor | status: "modified" + resolved_value | 200, valor guardado |
| POST apply aceptada | Sugerencia aceptada | 200, lesson plan modificado |
| POST apply pendiente | Sugerencia pendiente | 400, solo accepted/modified |
| POST apply plan ya dictado | Target plan dictado | 400, sugerencia auto-rejected |

### Flujo completo

| Test | Escenario | Expected |
|------|-----------|----------|
| Bitácora → análisis → sugerencia → accept → apply | Flujo happy path completo | Lesson plan modificado correctamente |
| Bitácora → análisis → sin impacto | Clase exitosa | No se generan sugerencias |
| Sugerencia → reject | Docente no quiere el cambio | Status rejected, plan sin cambios |
| Sugerencia → modify → apply | Docente ajusta la propuesta | Plan actualizado con resolved_value |

## Notas

- Tests de análisis de impacto requieren mock del AI Provider
- Tests de endpoints son de integración con DB real (patrón existente)
- Verificar que sugerencias para planes ya dictados se auto-rechazan
- Verificar permisos: solo el docente dueño puede resolver, coordinador read-only
