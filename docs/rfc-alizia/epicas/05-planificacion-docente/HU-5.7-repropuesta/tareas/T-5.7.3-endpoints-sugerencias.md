# T-5.7.3: Endpoints de sugerencias

**Historia:** HU-5.7 — Repropuesta
**Tipo:** Backend
**Estado:** ⬜ Pendiente
**Fase:** Post-MVP

---

## Descripción

Implementar los endpoints REST para consultar y resolver sugerencias de repropuesta. El docente ve las sugerencias pendientes, puede aceptar/rechazar/modificar cada una, y los cambios aceptados se aplican al lesson plan.

## Implementación

### Endpoints

| Método | Ruta | Descripción | Rol |
|--------|------|-------------|-----|
| GET | `/lesson-plans/{id}/suggestions` | Sugerencias para un plan específico | Docente |
| GET | `/course-subjects/{id}/suggestions` | Todas las sugerencias pendientes de la disciplina | Docente |
| PATCH | `/suggestions/{id}` | Resolver una sugerencia (accept/reject/modify) | Docente |
| POST | `/suggestions/{id}/apply` | Aplicar sugerencia aceptada al lesson plan | Docente |

### Listar sugerencias de una disciplina

```
GET /course-subjects/{id}/suggestions?status=pending
```

Respuesta:
```json
[
    {
        "id": 1,
        "type": "reinforce_topic",
        "description": "Reforzar ecuaciones de primer grado en la clase 5",
        "current_value": "Actividad: Resolución individual de ejercicios",
        "suggested_value": "Actividad: Trabajo en pares con ejercicios graduados + puesta en común",
        "reason": "En la clase 3, los alumnos mostraron dificultad con este tema",
        "priority": "high",
        "status": "pending",
        "target_lesson_plan": {
            "id": 45,
            "class_number": 5,
            "title": "Ecuaciones avanzadas"
        },
        "source_class": {
            "class_number": 3,
            "title": "Introducción a ecuaciones"
        }
    }
]
```

### Resolver sugerencia

```
PATCH /suggestions/{id}
```

```json
{
    "status": "accepted"
}
```

O con modificación:
```json
{
    "status": "modified",
    "resolved_value": "Trabajo en pares con ejercicios graduados, sin puesta en común por tiempo"
}
```

### Aplicar sugerencia

```
POST /suggestions/{id}/apply
```

Aplica el `suggested_value` (o `resolved_value` si fue modificado) al lesson plan target. La lógica de aplicación depende del `type`:

| Type | Acción |
|------|--------|
| `change_activity` | Reemplaza la actividad en el plan |
| `reinforce_topic` | Agrega/modifica actividad relacionada al topic |
| `adjust_time` | Modifica duración de actividades |
| `add_resource` | Agrega referencia de recurso al plan |

## Notas

- Solo el docente dueño del lesson plan puede resolver sugerencias
- `apply` solo funciona con sugerencias en status `accepted` o `modified`
- Si el target lesson plan ya fue dictado (status cambió), la sugerencia se marca como `rejected` automáticamente
- El diff visual (current_value → suggested_value) se renderiza en el frontend
- Las sugerencias de prioridad "high" se muestran con indicador visual destacado
- El coordinador puede ver las sugerencias pero no resolverlas (read-only)
