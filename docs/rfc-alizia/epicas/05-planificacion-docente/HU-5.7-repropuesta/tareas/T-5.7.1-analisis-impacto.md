# T-5.7.1: Análisis de impacto de bitácora

**Historia:** HU-5.7 — Repropuesta
**Tipo:** Backend
**Estado:** ⬜ Pendiente
**Fase:** Post-MVP

---

## Descripción

Implementar el servicio que analiza una bitácora completada y determina si hay impacto en clases futuras. Usa el AI Provider para evaluar los datos estructurados contra las clases planificadas y detectar ajustes necesarios.

## Implementación

### Trigger

El análisis se dispara automáticamente cuando una bitácora alcanza un estado "completo" (datos estructurados extraídos). No se dispara si el docente solo escribió texto libre sin completar el procesamiento.

### Prompt de análisis

```
Sos Alizia. Analizá la bitácora de la clase dictada y evaluá si hay impacto en las próximas clases planificadas.

Clase dictada:
- Título: {title}
- Objetivo: {objective}
- Resultado: objetivo {fulfilled}, actividades {completed}
- Duración real: {actual} min (planificada: {planned} min)
- Ajustes sugeridos por el docente: {adjustments}
- Observaciones de alumnos: {student_observations}

Próximas clases planificadas:
{next_classes_context}

Evaluá:
1. ¿Hay topics que necesitan refuerzo en clases futuras?
2. ¿Hay ajustes de tiempo necesarios?
3. ¿Alguna actividad debería cambiarse?
4. ¿Hay seguimiento de alumnos que requiera adaptación?

Si no hay impacto relevante, respondé con suggestions vacío.
```

### Schema de respuesta

```go
type ImpactAnalysisResult struct {
    HasImpact   bool                `json:"has_impact"`
    Suggestions []RepropuestaSuggestion `json:"suggestions"`
}

type RepropuestaSuggestion struct {
    TargetLessonPlanID int    `json:"target_lesson_plan_id"`
    Type               string `json:"type"` // "change_activity", "reinforce_topic", "adjust_time", "add_resource"
    Description        string `json:"description"`
    CurrentValue       string `json:"current_value"`   // qué hay ahora
    SuggestedValue     string `json:"suggested_value"`  // qué propone Alizia
    Reason             string `json:"reason"`
    Priority           string `json:"priority"` // "high", "medium", "low"
}
```

## Comportamiento

1. Bitácora completada → se dispara análisis
2. Se obtienen las próximas 5 clases del mismo course_subject en estado pending/in_progress
3. Se envía al AI Provider con `GenerateJSON`
4. Si `has_impact: true`, se almacenan las sugerencias (T-5.7.2)
5. Si `has_impact: false`, no se genera nada

## Notas

- El análisis es automático pero las sugerencias son opt-in — el docente decide si acepta
- Solo se analizan clases del mismo course_subject (misma disciplina, mismo curso)
- Se limita a 5 clases futuras para no sobrecargar el contexto
- El análisis no modifica nada directamente — solo genera sugerencias pendientes
- Si no hay clases futuras planificadas, el análisis se omite
