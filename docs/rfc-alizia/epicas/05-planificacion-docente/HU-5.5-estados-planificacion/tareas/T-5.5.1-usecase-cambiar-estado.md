# T-5.5.1: Usecase — cambiar estado

**Historia:** HU-5.5 — Estados de planificación
**Tipo:** Backend
**Estado:** ⬜ Pendiente

---

## Descripción

Usecase para cambiar el estado de una planificación entre pending, in_progress y published, con validaciones.

## Implementación

```go
// src/core/usecases/change_lesson_plan_status.go

type ChangeLessonPlanStatusInput struct {
    Status entities.LessonPlanStatus `json:"status" validate:"required,oneof=pending in_progress published"`
}

func (uc *ChangeLessonPlanStatusUsecase) Execute(userID uint, planID uint, input ChangeLessonPlanStatusInput) (*entities.TeacherLessonPlan, error) {
    plan, _ := uc.lessonPlanProvider.GetByID(planID)

    // Auth
    cs, _ := uc.courseSubjectProvider.GetByID(plan.CourseSubjectID)
    if cs.TeacherID != userID {
        return nil, errors.NewForbidden("only assigned teacher")
    }

    // Validar transición válida
    if !isValidTransition(plan.Status, input.Status) {
        return nil, errors.NewValidation("invalid status transition")
    }

    // Validar transición a published
    if input.Status == entities.LessonPlanStatusPublished {
        if len(plan.Activities) == 0 {
            return nil, errors.NewValidation("plan must have activities")
        }
        if plan.Proposal == nil || *plan.Proposal == "" {
            return nil, errors.NewValidation("plan must have a generated proposal")
        }
    }

    plan.Status = input.Status
    uc.lessonPlanProvider.Update(plan)

    return plan, nil
}

func isValidTransition(from, to entities.LessonPlanStatus) bool {
    transitions := map[entities.LessonPlanStatus]entities.LessonPlanStatus{
        entities.LessonPlanStatusPending:    entities.LessonPlanStatusInProgress,
        entities.LessonPlanStatusInProgress: entities.LessonPlanStatusPublished,
    }
    return transitions[from] == to
}
```

## Notas

- La transición pending → in_progress no tiene restricciones (el docente comienza a editar)
- La transición in_progress → published requiere actividades + proposal
- El flujo es unidireccional: pending → in_progress → published
