# T-4.3.3: Endpoints PATCH y generate

**Historia:** HU-4.3 — Secciones dinámicas
**Tipo:** Backend
**Estado:** ⬜ Pendiente

---

## Descripción

Crear los endpoints para edición manual de secciones (`PATCH`) y generación con IA (`POST /generate`). Ambos requieren rol coordinador. El PATCH valida section keys contra la config de la org. El generate llama a los usecases de generación de secciones y plan de clases.

## Implementación

### PATCH /api/v1/coordination-documents/:id

Actualización parcial del documento. Soporta campos como `name`, `sections`, y `status`.

```go
type PatchDocumentRequest struct {
    Name     *string                            `json:"name,omitempty"`
    Sections map[string]map[string]interface{}   `json:"sections,omitempty"`
    Status   *string                            `json:"status,omitempty"`
}

func (h *CoordinationHandler) Patch(c *gin.Context) {
    orgID := c.GetUint("organization_id")
    docID, _ := strconv.ParseUint(c.Param("id"), 10, 32)

    var req PatchDocumentRequest
    if err := c.ShouldBindJSON(&req); err != nil {
        c.JSON(400, gin.H{"error": err.Error()})
        return
    }

    // Si se envian secciones, usar usecase de UpdateSections
    if req.Sections != nil {
        input := coordination.UpdateSectionsInput{
            DocumentID: uint(docID),
            Sections:   req.Sections,
        }
        if err := h.updateSectionsUsecase.Execute(c.Request.Context(), orgID, input); err != nil {
            handleError(c, err)
            return
        }
    }

    // Si se envian otros campos (name, status), actualizar directamente
    // ...

    c.JSON(200, gin.H{"ok": true})
}
```

**Request ejemplo (editar seccion):**

```json
PATCH /api/v1/coordination-documents/1
{
    "sections": {
        "problem_edge": {
            "value": "Nuevo contenido del eje problematico editado manualmente..."
        }
    }
}
```

**Request ejemplo (seleccionar opcion):**

```json
PATCH /api/v1/coordination-documents/1
{
    "sections": {
        "methodological_strategy": {
            "selected_option": "proyecto"
        }
    }
}
```

### POST /api/v1/coordination-documents/:id/generate

Genera contenido IA para todas las secciones y el plan de clases.

```go
func (h *CoordinationHandler) Generate(c *gin.Context) {
    orgID := c.GetUint("organization_id")
    docID, _ := strconv.ParseUint(c.Param("id"), 10, 32)

    // 1. Generar secciones (HU-4.3)
    err := h.generateSectionsUsecase.Execute(c.Request.Context(), orgID, uint(docID))
    if err != nil {
        handleError(c, err)
        return
    }

    // 2. Generar plan de clases (HU-4.4)
    err = h.generatePlanUsecase.Execute(c.Request.Context(), orgID, uint(docID))
    if err != nil {
        handleError(c, err)
        return
    }

    // 3. Retornar documento actualizado
    doc, err := h.coordProvider.GetByID(c.Request.Context(), orgID, uint(docID))
    if err != nil {
        handleError(c, err)
        return
    }

    c.JSON(200, doc)
}
```

### Registro en mapping.go

```go
coordGroup := v1.Group("/coordination-documents")
coordGroup.Use(middleware.RequireRole("coordinator"))
{
    coordGroup.GET("", coordinationHandler.List)
    coordGroup.POST("", coordinationHandler.Create)
    coordGroup.GET("/:id", coordinationHandler.GetByID)
    coordGroup.PATCH("/:id", coordinationHandler.Patch)
    coordGroup.DELETE("/:id", coordinationHandler.Delete)
    coordGroup.POST("/:id/generate", coordinationHandler.Generate)
}
```

## Notas

- El PATCH es polivalente: puede actualizar `name`, `sections`, o `status` en la misma request o por separado
- La validación de section keys se hace dentro del usecase `UpdateSections`, no en el handler
- El endpoint `/generate` combina dos usecases (secciones + plan) — si uno falla, el otro no se ejecuta
- La respuesta de `/generate` retorna el documento completo actualizado para que el frontend pueda renderizar inmediatamente
- Considerar que `/generate` puede tardar varios segundos por las llamadas a Azure OpenAI — documentar timeout esperado
- El PATCH para `status` se completará en HU-4.5 (publicación); por ahora solo se menciona como campo soportado
