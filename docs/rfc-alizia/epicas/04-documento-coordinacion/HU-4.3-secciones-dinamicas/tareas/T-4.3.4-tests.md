## Descripción

Testear la edición manual de secciones, la validación de keys y tipos, la restricción de estado, y la generación de contenido con IA. Cubrir los casos de error (key inválida, doc publicado, select_text sin opción).

## Notas

- Los tests de generación necesitan un mock de `AIProvider` para no depender de Azure OpenAI
- El mock debe capturar el prompt recibido para verificar que los placeholders se reemplazaron
- El test de merge parcial es critico: un bug aqui podría borrar secciones existentes al editar una sola
- El test de documento publicado valida la regla de negocio de que solo drafts son editables
- Considerar tests de concurrencia si dos usuarios editan secciones distintas del mismo documento simultaneamente (fuera de scope de la POC)
- Los tests de integración (con DB real) se combinan con tests unitarios (con mocks) para cobertura completa
