## Descripción

Testear las operaciones CRUD del repositorio, la profundidad de preloads, el cascade delete y el comportamiento del enum de status. Verificar que la creación de un documento crea correctamente todas las junction tables asociadas.

## Notas

- Usar una base de datos de test (ver configuración existente en `tests/`)
- Cada test debe limpiar sus datos para no depender de orden de ejecución
- El test de cascade delete es critico: si falla, quedan datos huerfanos al borrar documentos
- El test de multi-tenancy valida que un usuario de otra org no puede acceder a documentos ajenos
- Considerar usar `t.Cleanup()` para borrar datos creados en cada test
