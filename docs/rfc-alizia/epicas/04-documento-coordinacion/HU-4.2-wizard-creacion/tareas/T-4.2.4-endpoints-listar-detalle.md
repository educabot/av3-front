# T-4.2.4: Endpoints GET listar y detalle, DELETE

**Historia:** HU-4.2 — Wizard de creación
**Tipo:** Backend
**Estado:** ⬜ Pendiente

---

## Descripción

Crear los endpoints de lectura y eliminación de documentos de coordinación: listado con filtro por area, detalle completo con todos los preloads, y eliminación (solo en estado pending). Registrar todos en `mapping.go`.

## Implementación

### GET /api/v1/coordination-documents

Lista documentos de la organización. Retorna resumen (sin preloads profundos).

```go
func (h *CoordinationHandler) List(c *gin.Context) {
    orgID := c.GetUint("organization_id")

    // Filtro opcional por area
    var areaID *uint
    if areaIDStr := c.Query("area_id"); areaIDStr != "" {
        parsed, _ := strconv.ParseUint(areaIDStr, 10, 32)
        id := uint(parsed)
        areaID = &id
    }

    docs, err := h.coordProvider.List(c.Request.Context(), orgID, areaID)
    if err != nil {
        handleError(c, err)
        return
    }

    c.JSON(200, docs)
}
```

**Response ejemplo:**

```json
[
    {
        "id": 1,
        "name": "Itinerario Ciencias Exactas",
        "area": { "id": 1, "name": "Ciencias Exactas" },
        "start_date": "2026-03-15",
        "end_date": "2026-07-15",
        "status": "pending"
    }
]
```

### GET /api/v1/coordination-documents/:id

Detalle completo con todos los preloads (topics, subjects, classes, etc.).

```go
func (h *CoordinationHandler) GetByID(c *gin.Context) {
    orgID := c.GetUint("organization_id")
    docID, _ := strconv.ParseUint(c.Param("id"), 10, 32)

    doc, err := h.coordProvider.GetByID(c.Request.Context(), orgID, uint(docID))
    if err != nil {
        handleError(c, err)
        return
    }

    c.JSON(200, doc)
}
```

### DELETE /api/v1/coordination-documents/:id

Solo permite borrar documentos en estado `pending`.

```go
func (h *CoordinationHandler) Delete(c *gin.Context) {
    orgID := c.GetUint("organization_id")
    docID, _ := strconv.ParseUint(c.Param("id"), 10, 32)

    // Primero verificar que el doc existe y esta en pending
    doc, err := h.coordProvider.GetByID(c.Request.Context(), orgID, uint(docID))
    if err != nil {
        handleError(c, err)
        return
    }
    if doc.Status != "pending" {
        c.JSON(403, gin.H{"error": "solo se pueden borrar documentos en estado pending"})
        return
    }

    err = h.coordProvider.Delete(c.Request.Context(), orgID, uint(docID))
    if err != nil {
        handleError(c, err)
        return
    }

    c.JSON(204, nil)
}
```

### Registro en mapping.go

```go
coordGroup := v1.Group("/coordination-documents")
coordGroup.Use(middleware.RequireRole("coordinator"))
{
    coordGroup.GET("", coordinationHandler.List)
    coordGroup.POST("", coordinationHandler.Create)
    coordGroup.GET("/:id", coordinationHandler.GetByID)
    coordGroup.DELETE("/:id", coordinationHandler.Delete)
}
```

## Notas

- `List` no hace preloads profundos para mantener la respuesta liviana — solo incluye `Area`
- `GetByID` hace preloads completos (ver T-4.1.4) — es la vista de detalle del documento
- `Delete` verifica estado `pending` antes de borrar — documentos en otros estados no se pueden eliminar
- El cascade delete de Postgres limpia todas las junction tables automaticamente
- Considerar agregar paginación a `List` si la cantidad de documentos crece (fuera de scope de la POC)
- Todos los endpoints filtran por `organization_id` del contexto (multi-tenancy)
