## Notas

- Los IDs del seed dependen del orden de inserción — ajustar si los seeds de otras HU cambian los IDs
- El seed incluye un documento `published` para testear que no se puede borrar
- El `sections` del documento publicado tiene un ejemplo de contenido para validar que el JSONB funciona
- Considerar agregar el seed al script `reset_db.sh` para que se ejecute junto con los demas seeds
- Los tests de integración necesitan la base de datos corriendo — usar el mismo setup que los tests existentes
