# T-4.2.3: Endpoint POST crear documento

**Historia:** HU-4.2 — Wizard de creación
**Tipo:** Backend
**Estado:** ⬜ Pendiente

## Descripción

Crear el endpoint `POST /api/v1/coordination-documents` que recibe el JSON completo del wizard y llama al usecase `CreateDocument`. Requiere rol coordinador y valida que el usuario está asignado al area via `area_coordinators`.

## Implementación

Archivo: `src/entrypoints/http/handlers/coordination_handler.go`

```go
package handlers

type CreateDocumentRequest struct {
    Name      string          `json:"name" binding:"required"`
    AreaID    uint            `json:"area_id" binding:"required"`
    StartDate string          `json:"start_date" binding:"required"` // "2026-03-15"
    EndDate   string          `json:"end_date" binding:"required"`
    TopicIDs  []uint          `json:"topic_ids" binding:"required,min=1"`
    Subjects  []SubjectRequest `json:"subjects" binding:"required,min=1"`
}

type SubjectRequest struct {
    SubjectID  uint   `json:"subject_id" binding:"required"`
    ClassCount int    `json:"class_count" binding:"required,min=1"`
    TopicIDs   []uint `json:"topic_ids" binding:"required,min=1"`
}

func (h *CoordinationHandler) Create(c *gin.Context) {
    var req CreateDocumentRequest
    if err := c.ShouldBindJSON(&req); err != nil {
        c.JSON(400, gin.H{"error": err.Error()})
        return
    }

    // Extraer orgID y userID del contexto (middleware auth)
    orgID := c.GetUint("organization_id")
    userID := c.GetUint("user_id")

    input := coordination.CreateDocumentInput{
        Name:      req.Name,
        AreaID:    req.AreaID,
        StartDate: req.StartDate,
        EndDate:   req.EndDate,
        TopicIDs:  req.TopicIDs,
        Subjects:  mapSubjects(req.Subjects),
    }

    docID, err := h.createUsecase.Execute(c.Request.Context(), orgID, userID, input)
    if err != nil {
        // Mapear errores del usecase a HTTP status codes
        // ValidationError → 422
        // ForbiddenError → 403
        // NotFoundError → 404
        handleError(c, err)
        return
    }

    c.JSON(201, gin.H{"id": docID})
}
```

### Registro en mapping.go

```go
// src/entrypoints/http/mapping.go
coordGroup := v1.Group("/coordination-documents")
coordGroup.Use(middleware.RequireRole("coordinator"))
{
    coordGroup.POST("", coordinationHandler.Create)
}
```

## Notas

- `RequireRole("coordinator")` se aplica a todo el grupo — solo coordinadores pueden acceder
- La validación de que el coordinador pertenece al area se hace dentro del usecase, no en el middleware
- `binding:"required,min=1"` en los slices asegura que no se envien arrays vacios
- El handler solo mapea request/response — toda la logica de negocio está en el usecase
- Response minimalista (solo ID): el frontend puede hacer GET del documento si necesita los datos completos
