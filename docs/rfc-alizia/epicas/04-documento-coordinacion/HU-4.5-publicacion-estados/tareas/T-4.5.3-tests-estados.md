**Historia:** HU-4.5 — Publicacion y estados

## Descripcion

Escribir tests para todas las transiciones de estado de los documentos de coordinacion, cubriendo los caminos exitosos, las validaciones de publicacion, y las restricciones de eliminacion y edicion segun el estado del documento.

## Notas

- Los tests de eliminacion (caso 4) deben verificar que el CASCADE funciona correctamente: despues del delete, consultar `coord_doc_topics`, `coord_doc_subjects`, `coord_doc_classes`, etc. para confirmar que no quedan registros huerfanos.
- Para el test del teacher (caso 8), configurar un usuario con rol `teacher` que pertenezca a la misma organizacion que el documento.
- Mockear la capa de repositorio en los tests de usecase; usar base de datos de test para los tests de integracion del handler.
- Cubrir tambien el caso de transiciones invalidas (ej: `in_progress` -> `published`) para asegurar que son rechazadas.
