# T-4.5.1: Usecase PublishDocument

**Historia:** HU-4.5 — Publicacion y estados
**Tipo:** Backend
**Estado:** ⬜ Pendiente

---

## Descripcion

Implementar el usecase `PublishDocument` que transiciona un documento de coordinacion de estado `in_progress` a `published`. Antes de publicar, se ejecutan validaciones estrictas que aseguran que el documento esta completo: todos los topics estan distribuidos en al menos una disciplina, y todas las secciones requeridas tienen contenido. Si alguna validacion falla, se retorna un error 422 con el detalle especifico de lo que falta.

## Implementacion

### Ubicacion

`src/core/usecases/coordination/publish_document.go`

### Validaciones

#### 1. Documento en estado in_progress

```go
if doc.Status != "in_progress" {
    return ErrDocumentNotInProgress // 422: "document must be in in_progress status to publish"
}
```

#### 2. Todos los topics distribuidos

Verificar que cada topic en `coord_doc_topics` (topics generales del documento) existe en al menos un registro de `coord_doc_subject_topics` (asignados a alguna disciplina).

```sql
-- Obtener topics no distribuidos
SELECT t.id, t.name
FROM coord_doc_topics cdt
JOIN topics t ON t.id = cdt.topic_id
WHERE cdt.document_id = $1
  AND cdt.topic_id NOT IN (
    SELECT cdst.topic_id
    FROM coord_doc_subject_topics cdst
    JOIN coord_doc_subjects cds ON cds.id = cdst.doc_subject_id
    WHERE cds.document_id = $1
  );
```

Si hay topics sin distribuir, retornar error con la lista:

```json
{
  "error": "validation_failed",
  "message": "No se puede publicar: hay topics sin distribuir",
  "details": {
    "undistributed_topics": [
      {"id": 5, "name": "Suma y resta"},
      {"id": 8, "name": "Ecuaciones lineales"}
    ]
  }
}
```

#### 3. Secciones requeridas con contenido

Consultar `coord_doc_sections` (tabla de configuracion) donde `required = true` y verificar que el JSONB `sections` del documento tenga un valor no vacio para cada `section_key` requerido.

```go
requiredSections, _ := sectionConfigRepo.GetRequired(ctx)
emptySections := []string{}

for _, section := range requiredSections {
    value, exists := doc.Sections[section.Key]
    if !exists || strings.TrimSpace(value) == "" {
        emptySections = append(emptySections, section.Name)
    }
}

if len(emptySections) > 0 {
    return ValidationError{
        Message: "No se puede publicar: hay secciones requeridas vacias",
        Details: map[string]interface{}{
            "empty_required_sections": emptySections,
        },
    }
}
```

### Flujo completo

```go
func (uc *PublishDocumentUsecase) Execute(ctx context.Context, docID int) error {
    doc, err := uc.docRepo.GetWithPreloads(ctx, docID)
    if err != nil {
        return err
    }

    // Validacion 1: estado
    if doc.Status != "in_progress" {
        return NewValidationError("document must be in in_progress status to publish")
    }

    // Validacion 2: topics distribuidos
    undistributed, err := uc.topicRepo.GetUndistributed(ctx, docID)
    if err != nil {
        return err
    }
    if len(undistributed) > 0 {
        return NewValidationError("undistributed topics", "undistributed_topics", undistributed)
    }

    // Validacion 3: secciones requeridas
    emptySections, err := uc.validateRequiredSections(ctx, doc)
    if err != nil {
        return err
    }
    if len(emptySections) > 0 {
        return NewValidationError("empty required sections", "empty_required_sections", emptySections)
    }

    // Publicar
    return uc.docRepo.UpdateStatus(ctx, docID, "published")
}
```

### Endpoint

Se invoca desde `PATCH /api/v1/coordination-documents/:id` cuando el body incluye `{"status": "published"}`.

### Response de error (422)

```json
{
  "error": "validation_failed",
  "message": "No se puede publicar el documento",
  "details": {
    "undistributed_topics": [
      {"id": 5, "name": "Suma y resta"}
    ],
    "empty_required_sections": [
      "Eje problematico",
      "Estrategia metodologica"
    ]
  }
}
```

## Notas

- Las validaciones se ejecutan todas antes de retornar, para dar al usuario una lista completa de lo que falta en un solo request.
- Una vez publicado, el documento no se puede editar directamente — debe coordinarse con el equipo técnico si se requiere un flujo de despublicar.
- La query de topics no distribuidos debe ser eficiente; usar un LEFT JOIN o NOT IN segun el volumen esperado.
