## Descripcion

Escribir tests unitarios e integracion para la funcionalidad de generacion y edicion del plan de clases por disciplina, cubriendo los casos principales y de borde.

### Ubicacion

### Casos de test

#### 1. Generacion de plan crea el numero correcto de clases por disciplina

```go
func TestGenerateClassPlan_CreatesCorrectNumberOfClasses(t *testing.T) {
    // Setup: documento con 2 disciplinas, class_count = 5 y 3 respectivamente
```

#### 3. PATCH clase actualiza titulo y topics


## Notas

- Mockear el cliente de Azure OpenAI en los tests de generacion para evitar llamadas reales a la API.
- Para tests de integracion del endpoint PATCH, usar una base de datos de test con datos seed.
- Los tests de clases compartidas necesitan un setup de schedule en el curso para simular slots compartidos.
- Verificar que la transaccion hace rollback correcto si falla a mitad de la generacion.
