# T-4.4.4: Enriquecimiento de clases compartidas

**Historia:** HU-4.4 — Plan de clases por disciplina
**Tipo:** Backend
**Estado:** ⬜ Pendiente

---

## Descripcion

Cuando se retorna el detalle de un documento de coordinacion (`GET /coordination-documents/:id`), enriquecer cada `coord_doc_class` con informacion de si esa clase es compartida con otra disciplina. Este es un calculo de solo lectura que no se persiste en la base de datos, sino que se agrega en tiempo de respuesta usando la logica de clases compartidas definida en HU-3.5.

## Implementacion

### Logica de enriquecimiento

Para cada disciplina en el documento:

1. Obtener los `course_subjects` asociados a esa disciplina en los cursos relevantes.
2. Usar la funcion `calculateSharedClassNumbers()` de HU-3.5 para obtener que numeros de clase caen en slots compartidos.
3. Para cada `coord_doc_class` de esa disciplina, verificar si `class_number` esta en la lista de clases compartidas.
4. Si lo esta, marcar `is_shared = true` e incluir el nombre de la disciplina con la que se comparte (`shared_with_subject`).

### Estructura de respuesta enriquecida

```json
{
  "id": 1,
  "class_number": 3,
  "title": "Introduccion a la cinematica",
  "objective": "Comprender los conceptos basicos de movimiento",
  "topic_ids": [1, 2],
  "is_shared": true,
  "shared_with_subject": "Matematicas"
}
```

Para clases no compartidas:

```json
{
  "id": 2,
  "class_number": 4,
  "title": "Leyes de Newton",
  "objective": "Aplicar las tres leyes de Newton",
  "topic_ids": [3],
  "is_shared": false,
  "shared_with_subject": null
}
```

### Pseudocodigo

```go
func enrichClassesWithSharedInfo(doc *CoordinationDocument, courses []Course) {
    for _, docSubject := range doc.Subjects {
        // Obtener numeros de clase compartida desde HU-3.5
        sharedNumbers, sharedWith := calculateSharedClassNumbers(
            docSubject.SubjectID,
            courses,
        )

        sharedSet := toSet(sharedNumbers)

        for i := range docSubject.Classes {
            class := &docSubject.Classes[i]
            if sharedSet[class.ClassNumber] {
                class.IsShared = true
                class.SharedWithSubject = sharedWith
            } else {
                class.IsShared = false
                class.SharedWithSubject = ""
            }
        }
    }
}
```

### Donde aplicar el enriquecimiento

En el handler de `GET /api/v1/coordination-documents/:id`, despues de cargar el documento con sus preloads y antes de serializar la respuesta:

```go
func (h *Handler) GetCoordinationDocument(c echo.Context) error {
    doc, err := h.docRepo.GetWithPreloads(ctx, docID)
    if err != nil {
        return echo.NewHTTPError(404, "document not found")
    }

    courses, err := h.courseRepo.GetByArea(ctx, doc.AreaID)
    if err != nil {
        return echo.NewHTTPError(500, "failed to load courses")
    }

    enrichClassesWithSharedInfo(doc, courses)

    return c.JSON(200, doc)
}
```

## Notas

- Este enriquecimiento **no se almacena** en la base de datos. Se calcula en cada request. Esto es aceptable para una POC dado que el volumen de datos es bajo.
- Depende directamente de la logica implementada en HU-3.5 (`calculateSharedClassNumbers`). Reutilizar esa funcion tal cual.
- Si un curso no tiene schedule definido, simplemente no habra clases compartidas para esa disciplina en ese curso.
- El campo `shared_with_subject` contiene el **nombre** de la disciplina companhera, no su ID, para facilitar el consumo desde el frontend.
