# T-4.6.3: Endpoint de chat con documento

**Historia:** HU-4.6 — Chat con Alizia
**Tipo:** Backend
**Estado:** ⬜ Pendiente

---

## Descripcion

Implementar el endpoint `POST /api/v1/coordination-documents/:id/chat` que permite al coordinador enviar mensajes a Alizia para modificar un documento de coordinacion via lenguaje natural. El endpoint recibe el mensaje del usuario y el historial de conversacion, y retorna la respuesta del asistente junto con las acciones ejecutadas.

## Implementacion

### Endpoint

```
POST /api/v1/coordination-documents/:id/chat
```

### Request body

```json
{
  "message": "Cambia el titulo de la clase 3 de Fisica a 'Introduccion a la cinematica'",
  "history": [
    {"role": "user", "content": "Hola Alizia"},
    {"role": "assistant", "content": "Hola! En que puedo ayudarte con el documento?"}
  ]
}
```

| Campo | Tipo | Requerido | Descripcion |
|-------|------|-----------|-------------|
| `message` | string | Si | Mensaje actual del usuario |
| `history` | array | No | Historial previo de la conversacion |
| `history[].role` | string | Si | `"user"` o `"assistant"` |
| `history[].content` | string | Si | Contenido del mensaje |

### Response exitosa (200)

```json
{
  "message": "Listo! Cambie el titulo de la clase 3 de Fisica a 'Introduccion a la cinematica'.",
  "actions": [
    {
      "tool": "update_class_title",
      "result": "Titulo de clase 42 actualizado a 'Introduccion a la cinematica'"
    }
  ]
}
```

### Responses de error

| Codigo | Situacion |
|--------|-----------|
| `400` | Body invalido o `message` vacio |
| `403` | Documento no esta en estado `in_progress` o usuario sin permisos |
| `404` | Documento no encontrado |
| `500` | Error interno (fallo de Azure OpenAI, etc.) |

### Handler

```go
func (h *Handler) ChatWithDocument(c echo.Context) error {
    docID, err := strconv.Atoi(c.Param("id"))
    if err != nil {
        return echo.NewHTTPError(400, "invalid document id")
    }

    var req ChatRequest
    if err := c.Bind(&req); err != nil {
        return echo.NewHTTPError(400, "invalid request body")
    }

    if strings.TrimSpace(req.Message) == "" {
        return echo.NewHTTPError(400, "message is required")
    }

    // Validar que el documento pertenece a la org del usuario
    userID := getUserIDFromContext(c)
    if err := h.authz.ValidateDocumentAccess(c.Request().Context(), userID, docID); err != nil {
        return echo.NewHTTPError(403, "access denied")
    }

    result, err := h.chatUsecase.Execute(
        c.Request().Context(),
        docID,
        req.Message,
        req.History,
    )
    if err != nil {
        if errors.Is(err, ErrForbidden) {
            return echo.NewHTTPError(403, err.Error())
        }
        return echo.NewHTTPError(500, "error processing chat message")
    }

    return c.JSON(200, result)
}
```

### Tipos del request

```go
type ChatRequest struct {
    Message string        `json:"message" validate:"required"`
    History []ChatMessage `json:"history"`
}
```

### Registro en router

En `mapping.go`:

```go
docs := v1.Group("/coordination-documents")
// ... otros endpoints existentes ...
docs.POST("/:id/chat", handler.ChatWithDocument, RequireRole("coordinator"))
```

### Validaciones

1. **Autenticacion**: usuario debe estar autenticado.
2. **Rol**: solo coordinadores pueden usar el chat (`RequireRole("coordinator")`).
3. **Pertenencia**: el documento debe pertenecer a un area de la organizacion del usuario.
4. **Estado in_progress**: delegado al usecase (T-4.6.2), que retorna `ErrForbidden` si no es in_progress.

## Notas

- El historial se envia completo desde el frontend en cada request. El backend es stateless respecto al chat.
- El frontend debe mantener el `chatHistory` en su estado local y enviarlo con cada mensaje.
- No hay limite de longitud del historial en esta POC, pero considerar truncar si se vuelve muy largo para no exceder el context window del LLM.
- El campo `actions` en la respuesta puede estar vacio si el LLM solo respondio con texto sin ejecutar tools.
- Loguear los errores de Azure OpenAI internamente pero retornar un mensaje generico al usuario.
