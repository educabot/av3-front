# T-4.6.1: Definicion de tools para function calling

**Historia:** HU-4.6 — Chat con Alizia
**Tipo:** Backend
**Estado:** ⬜ Pendiente

---

## Descripcion

Definir las herramientas (tools) de function calling que se envian a Azure OpenAI para que el chat de Alizia pueda modificar el documento de coordinacion mediante lenguaje natural. Cada tool se define como un struct en Go que serializa al formato de tools de OpenAI.

## Implementacion

### Ubicacion

`src/core/usecases/coordination/chat_tools.go`

### Tools a definir

#### 1. update_section

Reemplaza el contenido completo de una seccion del documento.

```go
var UpdateSectionTool = openai.Tool{
    Type: "function",
    Function: &openai.FunctionDefinition{
        Name:        "update_section",
        Description: "Reemplaza el contenido completo de una seccion del documento de coordinacion. Usar cuando el usuario pide cambiar o reescribir una seccion entera.",
        Parameters: jsonschema.Definition{
            Type: "object",
            Properties: map[string]jsonschema.Definition{
                "section_key": {
                    Type:        "string",
                    Description: "Clave de la seccion a actualizar (ej: 'eje_problematico', 'estrategia_metodologica')",
                },
                "content": {
                    Type:        "string",
                    Description: "Nuevo contenido completo de la seccion",
                },
            },
            Required: []string{"section_key", "content"},
        },
    },
}
```

#### 2. append_to_section

Agrega texto al final de una seccion existente, preservando el contenido actual.

```go
var AppendToSectionTool = openai.Tool{
    Type: "function",
    Function: &openai.FunctionDefinition{
        Name:        "append_to_section",
        Description: "Agrega texto al final de una seccion existente sin eliminar el contenido actual. Usar cuando el usuario pide agregar o complementar informacion a una seccion.",
        Parameters: jsonschema.Definition{
            Type: "object",
            Properties: map[string]jsonschema.Definition{
                "section_key": {
                    Type:        "string",
                    Description: "Clave de la seccion donde agregar contenido",
                },
                "content": {
                    Type:        "string",
                    Description: "Texto a agregar al final de la seccion",
                },
            },
            Required: []string{"section_key", "content"},
        },
    },
}
```

#### 3. update_class_title

Cambia el titulo de una clase especifica.

```go
var UpdateClassTitleTool = openai.Tool{
    Type: "function",
    Function: &openai.FunctionDefinition{
        Name:        "update_class_title",
        Description: "Cambia el titulo de una clase especifica del plan de clases. Usar cuando el usuario pide renombrar o cambiar el titulo de una clase.",
        Parameters: jsonschema.Definition{
            Type: "object",
            Properties: map[string]jsonschema.Definition{
                "class_id": {
                    Type:        "integer",
                    Description: "ID de la clase a actualizar",
                },
                "title": {
                    Type:        "string",
                    Description: "Nuevo titulo de la clase",
                },
            },
            Required: []string{"class_id", "title"},
        },
    },
}
```

#### 4. update_class_topics

Actualiza los topics asignados a una clase especifica.

```go
var UpdateClassTopicsTool = openai.Tool{
    Type: "function",
    Function: &openai.FunctionDefinition{
        Name:        "update_class_topics",
        Description: "Actualiza los topics (temas/conceptos) asignados a una clase especifica. Reemplaza la lista actual de topics. Usar cuando el usuario pide cambiar, agregar o quitar topics de una clase.",
        Parameters: jsonschema.Definition{
            Type: "object",
            Properties: map[string]jsonschema.Definition{
                "class_id": {
                    Type:        "integer",
                    Description: "ID de la clase a actualizar",
                },
                "topic_ids": {
                    Type:        "array",
                    Items:       &jsonschema.Definition{Type: "integer"},
                    Description: "Lista de IDs de topics a asignar a la clase",
                },
            },
            Required: []string{"class_id", "topic_ids"},
        },
    },
}
```

### Lista completa de tools

```go
var ChatTools = []openai.Tool{
    UpdateSectionTool,
    AppendToSectionTool,
    UpdateClassTitleTool,
    UpdateClassTopicsTool,
}
```

### Serializacion

Los structs deben serializar correctamente al formato JSON que espera la API de Azure OpenAI:

```json
{
  "tools": [
    {
      "type": "function",
      "function": {
        "name": "update_section",
        "description": "...",
        "parameters": {
          "type": "object",
          "properties": { ... },
          "required": ["section_key", "content"]
        }
      }
    }
  ]
}
```

## Notas

- Usar el paquete `github.com/sashabaranov/go-openai` o el SDK oficial de Azure para los tipos de tools. Si no coinciden exactamente, definir structs propios que serialicen al formato correcto.
- Las descripciones de cada tool son importantes: el LLM las usa para decidir cuando invocar cada una. Ser claro sobre cuando usar `update_section` vs `append_to_section`.
- Los `class_id` que el LLM usa deben corresponder a IDs reales de `coord_doc_classes`. El system prompt (ver T-4.6.2) debe incluir los IDs disponibles.
- No incluir tools de eliminacion o creacion de clases en esta iteracion para mantener la simplicidad.
