## Descripcion

Implementar el usecase `ChatWithDocument` que permite al coordinador conversar con Alizia (asistente IA) para modificar un documento de coordinacion mediante lenguaje natural. El usecase carga el contexto completo del documento, envia el mensaje al LLM con las tools disponibles, ejecuta las acciones solicitadas por el LLM, y retorna la respuesta junto con la lista de acciones realizadas.

### Flujo principal

```go
func (uc *ChatWithDocumentUsecase) Execute(
    ctx context.Context,
    docID int,
    message string,
    history []ChatMessage,
) (*ChatResponse, error) {

    // 1. Cargar documento completo
    doc, err := uc.docRepo.GetWithPreloads(ctx, docID)
    if err != nil {
        return nil, err
    }

    // Validar que sea in_progress
    if doc.Status != "in_progress" {
        return nil, NewForbiddenError("solo se puede chatear con documentos en edicion")
    }

    // 2. Construir system prompt con contexto del documento
    systemPrompt := buildChatSystemPrompt(doc)

    // 3. Construir mensajes: system + history + nuevo mensaje
    messages := []openai.ChatCompletionMessage{
        {Role: "system", Content: systemPrompt},
    }
    for _, h := range history {
        messages = append(messages, openai.ChatCompletionMessage{
            Role:    h.Role,
            Content: h.Content,
        })
    }
    messages = append(messages, openai.ChatCompletionMessage{
        Role:    "user",
        Content: message,
    })

    // 4. Enviar a Azure OpenAI con tools
    resp, err := uc.aiClient.CreateChatCompletion(ctx, openai.ChatCompletionRequest{
        Model:                "gpt-5-mini",
        Messages:             messages,
        Tools:                ChatTools, // de T-4.6.1
        MaxCompletionTokens:  4096,
    })
    if err != nil {
        return nil, err
    }

    // 5. Procesar tool_calls si existen
    actions := []ActionResult{}
    choice := resp.Choices[0]

    if len(choice.Message.ToolCalls) > 0 {
        for _, toolCall := range choice.Message.ToolCalls {
            result, err := uc.executeToolCall(ctx, doc, toolCall)
            if err != nil {
                actions = append(actions, ActionResult{
                    Tool:   toolCall.Function.Name,
                    Result: fmt.Sprintf("Error: %s", err.Error()),
                })
                continue
            }
            actions = append(actions, ActionResult{
                Tool:   toolCall.Function.Name,
                Result: result,
            })
        }
    }

    // 6. Retornar respuesta
    return &ChatResponse{
        Message: choice.Message.Content,
        Actions: actions,
    }, nil
}
```

### System prompt

El system prompt debe incluir el contexto completo del documento para que el LLM pueda tomar decisiones informadas:

```go
func buildChatSystemPrompt(doc *CoordinationDocument) string {
    var sb strings.Builder

    sb.WriteString("Eres Alizia, asistente pedagogica de coordinacion educativa. ")
    sb.WriteString("Ayudas a coordinadores a modificar documentos de coordinacion.\n\n")

    sb.WriteString("DOCUMENTO ACTUAL:\n")
    sb.WriteString(fmt.Sprintf("- Titulo: %s\n", doc.Name))
    sb.WriteString(fmt.Sprintf("- Area: %s\n", doc.Area.Name))
    sb.WriteString(fmt.Sprintf("- Estado: %s\n\n", doc.Status))

    // Secciones
    sb.WriteString("SECCIONES:\n")
    for key, value := range doc.Sections {
        sb.WriteString(fmt.Sprintf("- %s (key: %s): %s\n", key, key, truncate(value, 500)))
    }

    // Disciplinas y clases
    sb.WriteString("\nMATERIAS Y CLASES:\n")
    for _, subject := range doc.Subjects {
        sb.WriteString(fmt.Sprintf("\n## %s\n", subject.SubjectName))
        sb.WriteString("Topics asignados:\n")
        for _, topic := range subject.Topics {
            sb.WriteString(fmt.Sprintf("  - ID:%d %s\n", topic.ID, topic.Name))
        }
        sb.WriteString("Clases:\n")
        for _, class := range subject.Classes {
            topicIDs := extractTopicIDs(class.Topics)
            sb.WriteString(fmt.Sprintf("  - ID:%d Clase %d: %s (topics: %v)\n",
                class.ID, class.ClassNumber, class.Title, topicIDs))
        }
    }

    sb.WriteString("\nUSA LAS HERRAMIENTAS DISPONIBLES para realizar cambios. ")
    sb.WriteString("Siempre confirma al usuario que cambios realizaste.")

    return sb.String()
}
```

### Ejecucion de tool calls

```go
func (uc *ChatWithDocumentUsecase) executeToolCall(
    ctx context.Context,
    doc *CoordinationDocument,
    toolCall openai.ToolCall,
) (string, error) {
    var args map[string]interface{}
    json.Unmarshal([]byte(toolCall.Function.Arguments), &args)

    switch toolCall.Function.Name {
    case "update_section":
        key := args["section_key"].(string)
        content := args["content"].(string)
        err := uc.updateSectionsUsecase.Execute(ctx, doc.ID, key, content)
        if err != nil {
            return "", err
        }
        return fmt.Sprintf("Seccion '%s' actualizada", key), nil

    case "append_to_section":
        key := args["section_key"].(string)
        content := args["content"].(string)
        err := uc.appendSectionUsecase.Execute(ctx, doc.ID, key, content)
        if err != nil {
            return "", err
        }
        return fmt.Sprintf("Contenido agregado a seccion '%s'", key), nil

    case "update_class_title":
        classID := int(args["class_id"].(float64))
        title := args["title"].(string)
        err := uc.updateClassUsecase.UpdateTitle(ctx, classID, title)
        if err != nil {
            return "", err
        }
        return fmt.Sprintf("Titulo de clase %d actualizado a '%s'", classID, title), nil

    case "update_class_topics":
        classID := int(args["class_id"].(float64))
        topicIDs := toIntSlice(args["topic_ids"])
        err := uc.updateClassUsecase.UpdateTopics(ctx, classID, topicIDs)
        if err != nil {
            return "", err
        }
        return fmt.Sprintf("Topics de clase %d actualizados", classID), nil

    default:
        return "", fmt.Errorf("tool desconocida: %s", toolCall.Function.Name)
    }
}
```

### Tipos

```go
type ChatMessage struct {
    Role    string `json:"role"`
    Content string `json:"content"`
}

type ChatResponse struct {
    Message string         `json:"message"`
    Actions []ActionResult `json:"actions"`
}

type ActionResult struct {
    Tool   string `json:"tool"`
    Result string `json:"result"`
}
```

## Notas

- Reutilizar los usecases existentes (`UpdateSections`, `UpdateClass`) para ejecutar las acciones. No duplicar logica de validacion.
- El system prompt incluye los IDs de clases y topics para que el LLM pueda referenciarlos correctamente en los tool calls.
- Usar `max_completion_tokens` (no `max_tokens`) por requerimiento de `gpt-5-mini`.
- Si un tool_call falla, continuar con los demas y reportar el error en `actions`. No abortar toda la operacion.
- El historial se envia completo desde el frontend en cada request (stateless en backend).
- Solo documentos en estado `in_progress` pueden ser editados via chat. Documentos `published` retornan 403.
