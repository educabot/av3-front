## Notas

- Mockear el cliente de Azure OpenAI para controlar las respuestas del LLM en los tests. No hacer llamadas reales a la API.
- Para tests de integracion, verificar que los cambios efectivamente se persistieron en la base de datos despues de los tool calls.
- El mock debe poder simular respuestas con y sin tool_calls segun el caso de test.
- Verificar que el system prompt incluye el contexto completo del documento (secciones, clases, topics con IDs).
- Los tests de historial (caso 6) son importantes para asegurar que el frontend puede mantener contexto de conversacion.
